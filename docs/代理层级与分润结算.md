# 代理层级与分润结算

> 与 `合作模式与分账.md` 衔接：**架构 B** 下骑手付至**合作运营商（或代理商）商户**；**平台不参与运营分成**。确认收入在**合作运营商**与**一/二级代理商**之间切分，周期完结后由收款商户**分账划出**上级应得。  
> **合作伙伴场地分润**不在本文范围，见 `合作模式与分账.md` §9、`PRD.md`。

## 1. 组织层级

```
品牌方
  └── 一级代理商（L1）
        ├── 二级代理商（L2）…
        │     └── 合作运营商（场地+运维）…  【L2 只能挂合作方】
        └── 合作运营商（场地+运维）…      【L1 可直接挂合作方】
```

| 角色 | 可发展下级 | 抽成来源 |
|------|------------|----------|
| **一级代理商** | 二级代理商、直属合作运营商 | ① 二级从其直属合作运营商分润中分给 L1；② 直属合作运营商经营收入中分给 L1 |
| **二级代理商** | 仅直属合作运营商 | 直属合作运营商经营收入中分给 L2 |
| **运营商（合作方）** | 无下级 | 可持有自有设备；上级在其设备经营上计提分润 |

术语：**经营收入** = 套餐在**收入确认口径**下归属于该站点的可分配净额（已扣**支付通道手续费**；**不含**品牌方 SaaS）。设备可为**自有或租赁**（租赁时产权在租赁公司，订单仍按 `device_owner_id` 归属运营主体）。

## 2. 入驻时设置分成比例

| 操作方 | 被添加方 | 配置字段 | 分成基数（合同表述） |
|--------|----------|----------|----------------------|
| 一级代理 | 二级代理 | `l1_from_l2_rate` | 二级从其**直属合作运营商**分润收入 × 比例 → 付给一级 |
| 一级代理 | 直属合作运营商 | `l1_from_op_rate` | **该站点经营收入** × 比例 → 付给一级 |
| 二级代理 | 直属合作运营商 | `l2_from_op_rate` | **该站点经营收入** × 比例 → 付给二级 |

说明：

- 二级商户的经营收入先按 `l2_from_op_rate` 切给二级；若该商户上级为二级，则一级按 `l1_from_l2_rate` 对**二级实收分润**再切一层，不对商户直连（避免重复基数）。
- 一级直属商户：仅 `l1_from_op_rate`，无二级参与。

示例（示意）：

```
经营收入 1000 元（已扣支付手续费；平台不抽成）
├─ 二级直属合作运营商：L2 抽 15% → 150；合作运营商净 850；L1 对 L2 分润抽 20% → 30
└─ L1 直属合作运营商：L1 抽 12% → 120；合作运营商净 880
```

## 3. 包月为主：换电单只记「应分」、周期完结才打款

与 `合作模式与分账.md` §3 一致，并叠加代理层：

### 3.1 套餐购买订单（PackageOrder）

- 支付成功：预收，**不**向商户/代理打款。
- 服务期按日（或按次）**确认收入**，形成「经营收入」台账。

### 3.2 换电订单（SwapOrder）

- 每笔成功换电关联 `package_order_id`。
- **不**按换电单笔向代理打款；仅计算该次换电对应的**应分配金额**（占当期套餐确认收入的分摊额），写入 `accrual_line`：
  - 品牌方应得
  - 商户应得
  - 二级应得（如有）
  - 一级应得（如有）
- 状态：`待结转` → 随套餐周期**正常完结**后并入可结算批次；退款/提前终止则冲正。

### 3.3 正式打款触发条件

| 条件 | 说明 |
|------|------|
| 包月周期完结 | `valid_to` 到达且未退款终止，或次卡额度用尽且订单关闭 |
| 无未决退款 | 该 `package_order_id` 无进行中退款 |
| 对账确认 | 运营商/代理在后台确认或 T+N 自动确认 |

打款对象：商户净额、二级应得累计、一级应得累计（按主体分批次）。

### 3.4 状态机（套餐维度）

```
服务中 → 确认收入/换电应分累计（待结转）
      → 周期完结 → 可结算 → 已打款
      → 退款中 → 冲正应分 → 重新结算或关闭
```

## 4. 后台能力映射

| 角色 | 组织管理（新增/编辑） | 不可管理 |
|------|----------------------|----------|
| 一级 | 二级代理商（`l1_from_l2_rate`）；**直属**运营商（`l1_from_op_rate`） | 二级名下的直属运营商 |
| 二级 | **直属**运营商（`l2_from_op_rate`） | 下级代理（不可发展） |
| 运营商 | — | 组织菜单（切换代理角色使用） |

| 角色 | 订单/流水 | 分润相关菜单 |
|------|-----------|--------------|
| 一级/二级 | 自有订单/流水 | **下级分润** ★（下级设备经营；用户脱敏） |
| 运营商 | 自有套餐/换电/流水 | **合作伙伴分润** ★（场地合作方；用户完整） |
| 租赁公司 | — | 租金收缴、协议与设备 |

| 角色 | 员工 | 登录 |
|------|------|------|
| 一级/二级/运营商/租赁 | **员工**：运营员工 + 权限；运营商另含**合作伙伴**档案 | 员工/合作伙伴可独立登录，菜单按权限 |
| 合作伙伴 | 由所属运营商在「员工」维护 | **个人账户**、分润明细、提现 |

## 5. 与合作伙伴分润的边界

- **下级分润**：上级代理对**下级主体**（二级、合作运营商）在其**自有设备**上的经营计提；与 `l1_from_l2_rate` / `l2_from_op_rate` 合同相关。
- **合作伙伴分润**：合作运营商对**场地合作方**在其**关联站点**上的经营计提；比例在「员工-合作伙伴」维护，与代理合同无关。
- 同一笔站点经营收入在演示中**先**按代理合同切上级，**再**按场地合同切合作伙伴的基数为「可分配经营收入」子集（产品实现以 PRD/原型为准，避免重复计提需在合同中约定基数口径）。
