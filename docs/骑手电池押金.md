# 骑手电池押金（运营商实收）

> 确认：2026-07-15 · [decision-005.md](../../decisions/decision-005.md) · 退押：[decision-006.md](../../decisions/decision-006.md) · 收押数额：[decision-007.md](../../decisions/decision-007.md)  
> 与「平台保证金」（运营商→平台清分账户）**不是同一概念**。

## 已确认规则

1. **缴押时机**：购买套餐**同笔支付**（套餐费 + 实付押金一笔进运营商子商户；信用免押则押金额为 0）。  
2. **后台落位**：挂在**用户管理 / 用户**；列表区分 **实付在押 / 信用免押 / 渠道担保**。  
3. **平台视角**：按运营商**只读汇总**在押总额与类型人数。  
4. **清分**：押金**不参与**平台抽成、站点合伙人分润；仅套餐实付参与切分。  
5. **退押**：仅退电池押金统一进**退款管理**；运营商在「退款设置」自选 **自动退款 / 手动确认**（与套餐退款模式独立）。  
6. **数额与门槛配置**：运营商「平台设置 → **押金设置**」配置：电池押金数额、**微信支付分免押门槛**、**芝麻信用免押门槛**；任一路达标免押，均未达标则实收押金。

## 类型定义

| 类型 | 说明 | 在押计入总额 |
|------|------|--------------|
| 实付在押 | 个人套餐同笔收押，未退完 | ✅ |
| 退押中 | 已发起退押、款未结清（退款管理待审或处理中） | ✅（仍占用在账） |
| 信用免押 | 芝麻/支付分 | ❌ |
| 渠道担保 | 人天池 / 设备租赁白名单，押金责任在渠道 | ❌（渠道信用模块另管） |
| 已退 / 无需 | 完结退还或本单无押 | ❌ |

## 退押（退款管理）

| 项 | 规则 |
|----|------|
| 前置 | 已还电；有在押实付押金 |
| 队列 | `type=押金退还`；与冷静期/中途完结同列表 |
| 自动 | `depositRefundMode=auto` → 系统原路退运营商子商户 |
| 手动 | `depositRefundMode=manual` → 待审核，确认后系统执行 |
| 影响 | **不**中途完结套餐；服务期继续 |
| 主单含押 | 冷静期/中途完结的押金子项随主单，不受押金模式单独开关 |

## 运营商「用户」

- KPI：实付在押总额、在押人数、信用免押人数、渠道担保人数  
- 列表筛选项：电池押金类型  
- 详情/套餐单：同笔支付与清分表继续标注「押金不切分」

## 平台「用户管理」

- 二级菜单：**用户信息** | **用户押金统计**  
- **用户信息**：用户表（电池押金列、筛选项）  
- **用户押金统计**：按运营商汇总表（在押总额、人数、免押、渠道担保、退押中）

## 骑手端

- 购套餐页：展示「套餐费 + 电池押金 = 应付合计」，同笔确认支付  
- 我的：展示押金状态；还电后「申请退押」→ 进入运营商退款管理
