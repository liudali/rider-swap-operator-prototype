# 骑手换电 · 代理商 / 运营商后台 PRD

## 1. 背景

面向**外卖员**换电业务，**物联网平台**与**骑手用户端**由品牌方建设；本后台提供给**代理商**、**合作运营商**及**设备租赁公司**使用（品牌方无此运营后台）。

## 2. 角色定义

| 角色 | 定位 | 主要职责 |
|------|------|----------|
| **代理商** | 区域渠道主体；**可持有自有设备** | 采购/持有换电柜与电池；发展二级代理与合作运营商；配置分润 |
| **运营商** | 代理商的合作方；**可持有自有设备** | 场地管理、设备维护；本站经营与收款（架构 B）；向**合作伙伴**分润 |
| **品牌方** | 平台方 | 物联网、用户端、技术后台；**不持有运营设备、不参与运营分成** |
| **租赁公司** | 设备资产出租方 | 出租柜机/电池；管理租赁协议与承租方还款 |

关系示意：

```
品牌方（不持设备）
  ├── 租赁公司（出租设备）
  └── 一级代理商（可持自有 / 租用设备）
        ├── 二级代理商（可持自有 / 租用设备）
        │     └── 合作运营商（可持自有 / 租用设备）
        └── 合作运营商（直属）
```

## 3. 目标用户

| 角色 | 使用场景 |
|------|----------|
| **合作运营商** | 管理运营设备（含租赁）；**员工**（含合作伙伴）；订单/流水；**合作伙伴分润**；**租赁管理**；**收款账户** |
| **一/二级代理商** | 组织管理；**员工**；设备/订单/流水；**下级分润**；租赁与**收款账户** |
| **租赁公司** | 出租设备、协议、收款计划与进度；**收款账户** |

## 4. 商业分工

| 主体 | 设备 | 数据可见性 |
|------|------|------------|
| **品牌方** | 不持有运营设备 | 不提供本后台（或仅技术监控，非本期） |
| **一/二级代理、合作运营商** | 可自有或**租赁**柜/电；运营订单按 `device_owner_id` 统计 | 仅本主体运营设备的数据；租赁履约见「租赁管理」 |
| **租赁公司** | 资产产权方 | 仅出租侧协议、设备与还款进度 |
| **上级代理** | 可不持设备仅抽成 | **下级分润**模块查看来自下级自有设备经营的计提 |

收款与分账见 `合作模式与分账.md`、`代理层级与分润结算.md`。

## 4.1 支付架构（架构 B）

- **收款主体**：设备归属主体对应的二级商户（合作运营商或代理商）。
- **平台**：不参与运营分成；SaaS 另计。
- **结算**：周期完结后，收款方向上级代理商户分账划出应得部分。

## 5. 功能模块

| 模块 | 一/二级代理 | 合作运营商 |
|------|-------------|------------|
| **总览** | 自有 KPI + **按站点经营分析**（用户数、换电数、订单总金额）+ 下级分润提示 | 同左 |
| **组织管理** | 二级/合作运营商 | — |
| **我的设备** | 仅自有资产 | 仅自有资产 |
| **我的订单** | 套餐/换电均以**表格**列表；换电详情抽屉含过程日志 | 同左 |
| **我的流水** | 仅自有设备关联 | 仅自有设备关联 |
| **下级分润 ★** | 订单明细（逐笔）+ 汇总 + 结算批次；**骑手信息脱敏** | — |
| **员工** | 运营员工 + 权限；合作伙伴类型 | 同左 |
| **合作伙伴分润 ★** | — | 主体：分润设置+明细；合作方登录：仅本站点明细 |
| **个人账户** | — | 合作伙伴：余额、提现、银行卡（登录后） |
| **站点 / 用户** | — | 自有站点、骑手 |
| **协议与设备** | 查看租用协议与绑定设备 | 出租方维护协议与设备清单 |
| **租金收缴** | — | 按账期管理收缴进度、方式与跟进 |
| **月租金** | 按月查看设备租金、收入覆盖；自动扣款/手动缴纳；收益不足须人工处理 | — |
| **收款账户** | 全部主体 | 进件商户与用途（骑手收款、分账、租金等） |

## 5.1 设备权属

| 权属 | 说明 |
|------|------|
| **自有** | 产权归运营主体，无租金义务 |
| **租赁** | 产权归租赁公司，运营主体按协议还款；订单仍计入运营主体 |

## 5.2 骑手套餐服务生命周期

| 能力 | 说明 |
|------|------|
| **中途完结** | 未到期主动结束服务；退还**未使用套餐费** + **电池押金**（已付押原路退；须还电） |
| **信用免押** | 满足门槛可用**芝麻信用** / **微信支付分**免押，无押金退款项 |
| **服务冻结** | 骑手中途申请冻结，期间不可换电 |
| **解冻续用** | 解冻后有效期顺延，继续消耗原套餐额度 |

后台「我的订单 → 服务变更」可查看申请与退款进度；套餐详情展示押金、冻结、退款拆分。

## 5.3 租金缴纳

- **自动扣款**：绑定「租金支付」账户，到期自动扣划至出租方。
- **手动缴纳**：自动失败或未开通时，承租方主动支付。
- **收益不足**：当月站点可分配收入 < 应付租金时，自动扣款失败，须**人工处理缺口**（补缴或登记跟进）。

## 6. 页面筛选

各页面顶部独立筛选区（无全局城市/站点条），随页面切换；「重置筛选」仅清空当前页条件。示例：

| 页面 | 筛选项 |
|------|--------|
| 总览 | 站点、统计范围 |
| 套餐订单 | 套餐单号、用户手机、支付日起止、服务/订单状态 |
| 换电订单 | 换电单号、用户手机、换电日起止、状态 |
| 服务变更 | 套餐单号、手机、变更类型、处理状态 |
| 资金实收 | 关联单号、流水类型、交易日起止 |
| 用户 | 用户 ID、手机、主站点、服务状态 |
| 员工 | 姓名/手机、类型（运营/合作方）、状态 |
| 合作伙伴分润 | 合作方、站点、订单类型、单号、手机、分润状态 |

## 7. 权限与数据隔离

- **设备**：`device_owner_id` = 当前登录主体；不可查看或操作他人设备。
- **订单/流水**：仅 `device_owner_id` 与当前主体一致的套餐、换电、资金、应分记录。
- **下级分润**：单独模块；提供**订单明细**（套餐单号、站点、柜机、金额、本账号计提）；**用户 ID、手机号脱敏**，不展示姓名、不可跳转骑手档案。
- **员工**：各经营主体可添加**运营员工**并配置功能权限；**合作伙伴**作为员工类型在「员工 → 合作伙伴」维护（含分润比例、关联站点）。
- **员工/合作伙伴登录**：独立账号登录后台；侧栏仅展示权限内模块；数据限定所属主体（合作伙伴仅限关联站点）。
- **合作伙伴个人账户**：合作伙伴可查看余额、绑定银行卡、发起**提现**及查看提现记录。
- **合作伙伴分润**（仅合作运营商）：**分润设置**从员工档案中的合作伙伴列表选取；按关联站点计提；订单明细用户 ID/手机完整展示；与下级分润独立。
- 一个站点同一时间仅关联一位合作伙伴。
- 汇总表订单笔数应与明细笔数可勾稽。
- 一级代理：不可管理二级名下的合作运营商档案（组织权限不变）。

## 9. 订单模型（摘要）

| 类型 | 说明 |
|------|------|
| `PackageOrder` | 须绑定主柜机 `cabinet_id` → 归属主体 |
| `SwapOrder` | 须关联套餐；含柜机/格口/电池信息；**过程日志**（用户操作、API 请求、设备指令与反馈） |
| `SwapOrderLog` | 按时间序；`kind`=user/req/dev；用于客诉与故障追溯 |
| `LeaseContract` | 出租方、承租方、设备清单、租期、租金、押金 |
| `LeaseRepayment` | 按期应还/实还；状态：已还/待还/未到期 |
| `PaymentAccount` | 主体进件账户；用途：骑手收款、分账、租金收付 |
| `LeaseRentBill` | 按月账单；含设备分摊、站点收入、覆盖缺口、缴纳方式 |
| `ServiceChange` | 中途完结 / 冻结 / 解冻申请及处理状态 |
| `PackageDeposit` | 电池押金；可关联信用免押凭证 |
