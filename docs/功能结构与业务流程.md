## 4. 核心业务流程

### 4.1 架构 B：骑手支付 → 收入确认 → 向上分账

```mermaid
sequenceDiagram
  participant R as 骑手
  participant App as 用户端
  participant Mch as 合作运营商二级商户
  participant Reg as 监管/清分
  participant BO as 运营后台
  participant L2 as 二级代理商户
  participant L1 as 一级代理商户

  R->>App: 购买包月套餐
  App->>Mch: 支付成功
  Mch->>BO: 资金实收流水
  Note over BO: 此时不向代理打款<br/>仅预收记账

  loop 服务期内按日
    BO->>BO: 确认收入 + 应分台账
    BO->>BO: 换电成功 → 换电应分待结转
  end

  alt 包月周期完结且无未决退款
    BO->>BO: 状态 → 可打款
    Mch->>Reg: 发起分账
    Reg->>L2: 划出二级应得
    Reg->>L1: 划出一级应得
    Note over Mch: 商户留存净额
    BO->>BO: 打款批次已分账
  else 中途退款
    BO->>BO: 冲正应分 / 退款出款
  end
```

### 4.2 换电订单与应分（不即时打款）

```mermaid
flowchart TD
  A[骑手扫码换电] --> B{有效套餐?}
  B -->|否| Z[失败]
  B -->|是| C[柜机开门换电]
  C --> D[换电订单成功]
  D --> E[写入换电过程日志]
  D --> F[按套餐确认收入分摊<br/>计算各方应分]
  F --> G[应分状态: 待结转]
  G --> H{套餐周期是否完结?}
  H -->|否| G
  H -->|是| I[可打款]
  I --> J[纳入打款批次 / 分账划出]
```

flowchart LR

### 4.5 员工 / 合作伙伴登录与权限

```mermaid
flowchart TD
  Start[选择登录身份] --> Type{身份类型}

  Type -->|经营主体| Full[完整角色菜单<br/>按 l1/l2/operator/leasing]
  Type -->|运营员工| Staff[读取 permissions 列表]
  Type -->|合作伙伴| Ptr[默认: 个人账户 + 分润查看]

  Staff --> Map[权限映射到菜单模块]
  Map --> Nav[渲染侧栏仅可见项]
  Nav --> Data[数据: device_owner_id = 所属主体]

  Ptr --> Nav2[侧栏: 个人账户 / 合作伙伴分润]
  Nav2 --> Data2[数据: 仅 partnerId + 关联站点]
  Data2 --> NoSet[不可操作分润设置/员工管理]
```

### 4.7 租赁设备与月租金

```mermaid
flowchart TD
  LeaseCo[租赁公司] -->|签订协议| Contract[租赁协议+设备清单]
  Contract --> Lessee[承租方 运营商/代理]
  Lessee --> Bill[按月租金账单]
  Bill --> Cover{站点收入 ≥ 月租金?}
  Cover -->|是| Auto[自动扣款至出租方]
  Cover -->|否| Gap[收益不足]
  Gap --> Manual[人工补缴 / 登记跟进]
  Auto --> Collect[租赁公司-租金收缴]
  Manual --> Collect
  Lessee --> View[承租方-月租金/协议与设备]
```
