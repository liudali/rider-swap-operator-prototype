# 运营商信用评估方案（准入档位制 · 方案 B）

> **版本**：1.0  
> **日期**：2026-06-12  
> **决策**：产品确认 **方案 B**（准入档位制）；见 [需求评审决策记录.md](./需求评审决策记录.md) **#13**  
> **状态**：一期 Mock；与 [渠道信用评估方案.md](./渠道信用评估方案.md) **独立**，勿混用

---

## 1. 背景与边界

| 能力 | 对象 | 作用 |
|------|------|------|
| **服务保证金 + 信用额度** | 运营商 | 跨网清分欠费记账；保证金为 0 才占用额度；用尽停跨网（PRD §5.6） |
| **渠道信用评估** | 渠道商 | 抵扣渠道应缴骑手押金；平台评 → 运营商可改额度 |
| **运营商信用评估（本文）** | 运营商主体 | 招商准入 + 年度复审；决定**档位政策包**（最低保证金、信用封顶、跨网默认、可签渠道数） |

**原则**

- 平台定档；运营商**只读**自己的档位与政策，**不能**自改平台授信上限（与渠道信用不同）。
- 保证金管理手工调额时，**不得高于**当前档位 `creditCap`。
- 不做连续评分；中间不调档，除非平台人工升/降档或年度复审。

---

## 2. 准入档位定义

| 档位 | 名称 | 最低保证金 | 信用额度封顶 | 跨网默认 | 可签渠道数 | 备注 |
|------|------|------------|--------------|----------|------------|------|
| **A** | 战略 | ¥50,000 | ¥100,000 | 开 | 不限 | 平台重点招商 |
| **B** | 标准 | ¥20,000 | ¥50,000 | 开 | ≤10 | **默认新入网** |
| **C** | 审慎 | ¥10,000 | ¥10,000 | 关 | ≤3 | 需加保或担保 |
| **D** | 观察 | 全额预存 | 0 | 关 | 1 | 仅本站，无授信 |

> **D 档最低保证金**：不对外展示固定金额，含义为跨网/B 端计提须**全额预存**，不设信用透支。

档位参数由平台管理员在「档位配置」维护（一期 Mock 本地配置；二期可审批流）。

## 3. 触发与流程

### 3.1 入网定档

```mermaid
flowchart LR
  A[商务提交入网] --> B[平台创建运营商主体]
  B --> C[运营商信用评估 · 待定档]
  C --> D{平台审批定档}
  D -->|通过| E[写入档位 + 初始化信用上限]
  D -->|驳回| F[停用 / 不开放业务]
  E --> G[保证金管理可对账充值]
```

1. 平台在「运营商管理」创建主体后，信用档案状态为 **待定档**。
2. 平台管理员在「运营商信用评估 → 运营商定档」选择档位、填写依据（如资质材料编号）。
3. 定档后：
   - 初始化 `creditLimit` ≤ 档位 `creditCap`（默认取封顶值，可在保证金管理微调但不可超封顶）。
   - 跨网默认策略写入运营商换电范围（A/B 默认可开跨网；C/D 默认关，运营商仍可在站点级关跨网）。
   - 签约渠道时校验「已签渠道数 < maxChannels」。

### 3.2 年度复审

| 项 | 规则 |
|----|------|
| 周期 | 定档日起每 **12 个月** |
| 动作 | 平台管理员复核；可维持 / 升档 / 降档 |
| 中间 | **不自动调档**；经营恶化依赖人工降档或保证金/跨网停服机制 |

### 3.3 升 / 降档

- 须填写变更原因；写入**变更记录**（操作人、时间、原档位、新档位）。
- 降档时：若当前 `creditLimit` > 新档 `creditCap`，系统**自动压顶**至新封顶；已占用欠费不清零，仍按 §5.6 清分。
- 升至 D 档：信用额度置 0，关闭跨网（与额度用尽逻辑一致）。

## 4. 与现有模块联动

| 模块 | 联动 |
|------|------|
| **运营商管理** | 列表/详情展示当前档位、封顶、下次复审日；未定档标红 |
| **保证金管理** | 调信用额度时校验 ≤ 档位封顶；展示「政策封顶 ¥X」 |
| **服务保证金账户（运营商）** | 只读：当前档位、最低保证金建议、信用封顶、跨网/渠道政策摘要 |
| **渠道销售** | 新增签约时校验可签渠道数；超限拦截并提示升档 |
| **跨网 / 清分** | 仍按 PRD §5.6：保证金优先 → 信用额度 → 停跨网；档位仅约束**上限与默认开关** |

## 8. 与渠道信用的区别

| 维度 | 运营商信用（本文） | 渠道信用 |
|------|-------------------|----------|
| 对象 | 运营商主体 | 渠道商 |
| 方法 | 准入档位制 | 评分 + 分级抵扣押金 |
| 输出 | 政策包（保证金/封顶/跨网/渠道数） | 信用分、抵扣比例、应押缺口 |
| 谁可改额度 | 仅平台（封顶约束） | 平台评 + **运营商可改**渠道额度 |
| 触发 | 入网 + 年度复审 | 新签手工 + 月度复评 |
