# 验收标准 · 四角色后台 + 骑手端（原型）

## 通用

- `python3 main.py` 端口 `8766` 可预览后台、骑手端、**产品文档浏览器**（`/prototype/docs/index.html`，渲染 `docs/*.md`）
- 侧栏展示当前登录主体名称与角色类型
- 可切换演示角色：**平台管理员** / **运营商** / **渠道商** / **设备租赁公司**（支持多家）；导航随角色变化
- 侧栏「切换登录身份」含：经营主体、员工登录
- 无全局筛选条；各页面独立筛选区；**无「重置筛选」按钮**
- 模块「说明」弹窗可用；无代理层级/下级分润表述
- **产品文档**：`/prototype/docs/index.html` 可浏览 `docs/*.md`（含 [设计评审报告.md](./设计评审报告.md)、[额度池PRD差异清单.md](./额度池PRD差异清单.md)；侧栏切换、Markdown 渲染、**Mermaid 流程图渲染为图形**）

## 平台管理员

| 模块 | 验收项 | 预期 |
|------|--------|------|
| 总览 | 业务汇总 KPI | 运营商数、渠道商数、用户数、站点数、设备数、订单数、业务流水、平台营收 |
| 总览 | 运营商账户汇总表 | 各运营商保证金/信用额度/跨网状态；可进详情抽屉 |
| 用户管理 | 服务运营商 | 列展示当前服务运营商（个人=套餐售卖方；渠道=额度售卖方） |
| 用户管理 | 套餐与人天 | 套餐状态+生效周期；渠道成员展示渠道商、人天额度、今日资格 |
| 用户管理 | 持有电池 | 服务中且已换电：展示 SN/SOC/归属；**已冻结**→未持有（须先还电）；**中途完结**→未持有（已还电）；**待首换开通**→未持有 |
| 用户管理 | 冻结校验 | 骑手端「申请冻结」：持有电池时提示「请先归还电池后再申请冻结」；还电后可提交 |
| 订单 | 服务冻结 | 个人套餐冻结/解冻满足条件**系统自动生效**；后台只读查询，无「通过解冻」审核 |
| 订单 | 解冻后首服 | 自行解冻后套餐详情标记「待领取电池」；骑手端首次扫码为**领取电池**，完成后恢复换电 |
| 订单管理 | 三 Tab | 套餐购买 / 换电 / 渠道商批发；全平台数据可筛选 |
| 订单管理 | 换电三元组 | 换电 Tab 含 user/柜机/电池运营商与跨网服务费 |
| 订单管理 | SOH | 换电 **详情**展示归还/换出电池 **SOH（%）**；列表不展示；运营商侧不展示 |
| 订单管理 | 渠道商订单 | **只读**监管；无「确认到账」/支付操作；线下 PO 由售卖运营商在渠道销售确认 |
| 设备管理 | 三 Tab | 设备台账 / 待绑定 / 批量导入 |
| 设备管理 | 批量导入 | 粘贴 SN 列表导入待绑定队列（演示） |
| 设备管理 | 绑定 | 待绑定设备可绑定至在营运营商 |
| 渠道商管理 | 列表 | 只读；账号、创建/维护方（运营商）、额度池、人天、人员规模 |
| 渠道商管理 | 人员明细 | 只读；登记骑手、**团队**、额度状态 |
| 渠道商管理 | 不可新增 | 平台侧无「新增渠道商」入口 |
| 流水管理 | 三 Tab | 用户支付 / 运营商之间 / 平台提成 |
| 流水管理 | 平台 1% | 用户支付与平台提成 Tab 高亮 1% 金额 |
| 流水管理 | 运营商之间 · 净额前/后保证金 | 日清账单展示「净额前保证金」「净额后保证金」；**允许负值**，负值表示占用信用额度；运营商后台不展示 |
| 平台账户 | 商户与营收 | 微信/支付宝商户、余额；**统计月份**筛选后展示该月提成 KPI 与营收构成（2026-04～06 演示） |
| 运营商管理 | 列表与筛选 | 名称/ID/城市/状态筛选；新增/编辑（演示 Mock） |
| 运营商管理 | 详情抽屉 | 基础信息、清分账户（保证金+信用额度）、**准入档位与政策**、进件摘要、经营汇总；可跳转「保证金管理」「运营商信用评估」 |
| 租赁公司 | 租赁公司列表 | 平台维护出租方主体；展示已绑定运营商；支持新增/编辑（演示） |
| 租赁公司 | 租赁关系绑定 | 弹窗内选择租赁公司+运营商建立/停用绑定；绑定后租赁公司方可签约 |
| **运营商信用评估** | 档位配置 Tab | 展示/编辑 A/B/C/D 政策包：最低保证金、信用封顶、跨网默认、可签渠道数 |
| 运营商信用评估 | 运营商定档 Tab | 已定档列表；待定档可「定档」；支持升/降档并填原因 |
| 运营商信用评估 | 变更记录 Tab | 升/降档、入网定档历史（操作人、时间、原/新档位、原因） |
| **保证金管理** | 充值确认 Tab | 待确认列表可「确认到账」「驳回」；确认后保证金余额增加并写入账本 |
| 保证金管理 | 账户总览 Tab | 展示清分收款专户；各运营商保证金/信用额度/**档位封顶**；调信用额度**不得高于封顶** |
| 保证金管理 | 变动明细 Tab | 全平台保证金账本（充值/日清/平台费/手工） |
| 保证金管理 | 待办角标 | 侧栏「保证金管理」有待确认充值时显示 `!` |
| **变更记录** | 平台查询 | 全平台变更时间线；可按类型/关键词筛选 |
| 跨网统一定价 | 编辑发布 | 柜机/电池服务费可改（本地 Mock）；**人天标准日值**可改（默认 ¥8.5/人天）；运营商只读展示 |
| 登录 | 切换身份 | 登录下拉首项「平台管理员 · 智格超能」 |

### 渠道分销（骑士卡） `CH-CARD`

| 定价 | 渠道差异 | 运营商可对不同分销渠道设不同专享价与佣金（按 SKU）；专享价 ≤ 正式零售价 |
| 无 | 卡库存/持卡分配 | **不应出现**批发入库、分配卡片能力 |
| 无 | 渠道信用额度 | 侧栏**无**「渠道信用额度」；运营商渠道销售信用表对该渠道显示「不适用」 |
