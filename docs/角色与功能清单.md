## 2. 商业关系

- **渠道商**向**一家或多家签约运营商**批量采购人天额度（**一运营商一池**；运营商定价）；成员**可跨网换电**，`userOwner` 为**团队消耗池的售卖方**。**渠道商主体由签约运营商在「渠道销售」添加与管理**（新增时默认建池）；平台仅查询监管。
- **运营商**向**资金方**租赁设备；服务 C 端个人 + 承接渠道成员换电；跨网设备使用费**经平台清分**，**平时划扣平台保证金**，保证金为 0 才启用信用额度（详见 [换电场景与运营商结算.md](./换电场景与运营商结算.md)）。
- **智格平台**：物联网、用户端、规则引擎、对账；收取 **1% 技术服务费**（C 端支付清分 + 渠道人天确认消耗向运营商计提）。

## 6. 跨运营商电池流转 · 结算枚举

> 完整场景、两类用户、保证金/信用额度清分见 **[换电场景与运营商结算.md](./换电场景与运营商结算.md)**。

### 两类用户与 userOwner

| 用户类型 | userOwner（用户归属运营商） |
|----------|----------------------------|
| ① 个人套餐 | 套餐售卖运营商（用户付款对象） |
| ② 渠道成员 | 人天额度售卖运营商（渠道采购合同的 sellerOperator） |

三元组：`userOwner` · `cabinetOwner` · `batteryOwner`（IoT 上报，后台只读）。

| # | userOwner | 柜机 | 电池 | 运营商间跨网设备服务费（经平台保证金/信用额度） |
|---|-----------|------|------|----------------------------|
| 1 | Op1 | Op1 | Op1 | 无 |
| 2 | Op1 | Op2 | Op2 | Op1→Op2 换电服务费 |
| 3 | Op1 | Op2 | Op3 | U 平台代付 ¥0.5 柜机 + ¥0.1 电池（U→B，已确认） |
| 4 | Op1 | Op1 | Op3 | Op1→Op3 电池使用费 |
| 5 | Op1 | Op2 | Op1 | Op1→Op2 柜机服务费 |
| 6 | Op1（渠道额度售卖方） | Op2 | Op3 | 是 | Op1 保证金代付 ¥0.5+¥0.1；渠道池 −1 人天 | 同 #3b |

**清分原则**：跨网设备服务费经平台清分；**每日 23:59:59 日清**；运营商只见**平台代收/代付**（不展示对手方）。单价：柜机 **¥0.5/次**、电池 **¥0.1/次**（暂定，平台管控）。

**保证金/信用额度**：平时划扣**平台保证金**；**仅当保证金为 0** 才启用信用额度（允许欠费）；**信用额度也用光** → 关闭该运营商所属用户**跨网换电**（个人+渠道）。

## 7. 平台 1% 服务费

| 场景 | 触发时点 | 商务状态 | 收取方式 |
|------|----------|----------|----------|
| C 端个人支付 | **支付成功** | 已确认 | 支付通道分账至平台商户（1%） |
| 渠道人天服务 | **确认消耗** | **已确认** | 按**平台标准人天价** × 1% 向**额度售卖方 U** 计提 |

**防线下绕过**：额度仅平台登记池生效；消耗必计提；渠道跨网与个人用户共用跨网/跨站准入规则。

## 8. 风险登记

| 风险 | 等级 | 缓解 |
|------|------|------|
| 渠道线下采购绕过平台 | 高 | 收费锚点改为确认消耗；额度/token 平台发放 |
| 跨运营商三元组错误 | 高 | IoT 强制上报，后台只读 |
| 运营商私下结算跨网费 | 高 | 强制平台代收/代付，不暴露对手方 |
| 保证金与信用额度均用尽仍跨网 | 高 | 自动关闭该运营商用户跨网换电 |
| 运营商待分账不足扣平台费 | 中 | 预存户 + 月结 + 欠费停预占 |
| 批发价过低逃费 | 中 | B 端 1% 计提基数与运营商批发价解耦，统一按平台标准人天价计提 |
| 采购运营商≠换电运营商 | 中 | 池绑定 seller + 规则声明可兑范围 |
| 多角色同账号混用 | 中 | 独立 NAV 与 entityId 隔离 |
| 未绑定设备被运营商自行接入 | 高 | 设备绑定仅平台可操作；运营商只见已绑定资产 |
| 文档状态矩阵与原型过期 | 高 | 大改后同步功能清单与验收（见 [设计评审报告.md](./设计评审报告.md)） |
| 人天池预占整批失败边界不清 | 中 | 天数池 FAQ；异常 Tab 演示 |
| 骑手端与后台 Mock 不一致 | 中 | 案例3 双端对齐；验收勾稽项 |
| 保证金充值/预警流程未定义 | 低 | 原型已补对公充值；最低余额预警待详设 |
| 合伙人分润与渠道月结并存误解 | 中 | 明确仅个人用户参与合伙人分润 |
| 操作审计与提现风控缺失 | 低 | 非本期；合规详设补充 |
