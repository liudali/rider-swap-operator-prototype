# 任务：订单变更审计（变更记录）

> **状态**：**已拍板 · 原型 Mock 已上线**  
> **决策来源**：[竞品借鉴决策记录.md](./竞品借鉴决策记录.md) C-02 · [竞品借鉴-待评审拍板.md](./竞品借鉴-待评审拍板.md)  
> **关联**：[PRD.md](./PRD.md) · [acceptance-criteria.md](./acceptance-criteria.md) · [角色与功能清单.md](./角色与功能清单.md)

---

## 1. 这个模块是干什么的？（一句话）

**变更记录 = 订单/服务的「生命历程账」**：记录从创建到结束，**谁在什么时间做了什么变更**——不是再做一个订单列表。

---

## 2. 为什么需要它？（业务目的）

| 场景 | 没有变更记录 | 有变更记录 |
|------|-------------|-----------|
| 骑手投诉「没换电却扣人天」 | 多 Tab 翻查，说不清 | 时间线：预占 → 确认消耗 → 换电单 |
| 渠道质疑消耗偏多 | 无法按人解释 | 按团队/用户筛分配、消耗、收回 |
| 退款/冻结纠纷 | 难还原时点 | 冻结申请 → 审核 → 退款单串联 |
| 平台监管 | 只见结果 | 跨运营商事件可聚合查询 |

**本质**：**可追溯、可解释、可对账**。

---

## 3. 拍板决议（已确认）

| ID | 决议 |
|----|------|
| **D-A1** | **统一侧栏「变更记录」**（平台 / 运营商 / 渠道各一入口，不按事件类型拆多个菜单） |
| **D-A2** | 渠道商可看，**仅限本渠道成员**相关事件 |
| **D-A3** | 保留时长与订单**同生命周期** |
| **D-A4** | 一期默认事件清单**全做**（见 §5） |

---

## 4. 入口与角色

| 角色 | 入口 | 数据范围 |
|------|------|----------|
| 平台管理员 | 侧栏 **变更记录** | 全平台（只读） |
| 运营商 | 侧栏 **变更记录** | 本运营商 |
| 渠道商 | 侧栏 **变更记录** | 本渠道成员（`channelId` 过滤） |

**原型**：`prototype/index.html` · Mock 数据 `orderAuditEvents`

---

## 5. 一期事件清单

| 类型 | 事件 | 一期 |
|------|------|:----:|
| 个人套餐 | 订单创建、冻结、恢复、中途完结 | ✅ |
| 个人套餐 | 换电关联（电池 SN / 换电单） | ✅ |
| 渠道 | 持卡分配、确认消耗、分配/收回 | ✅ |
| 跨网 | 引用换电单（不重复记账） | ✅ |
| 可选 | 手动延期、套餐升级 | ⭕ 二期 |
| 不做 | 纯 UI 点击、未影响计费的编辑 | ❌ |

---

## 6. 页面形态

- 筛选：关键词（手机/单号/摘要）、事件类型  
- 列表：时间倒序；变更前 → 变更后；操作人  
- 关联单号可跳转：换电单 / 套餐单 / 退款管理  

---

## 7. 与现有模块关系

- **写入**：业务动作发生时写审计事件（开发期同步/异步）  
- **读取**：变更记录页只读聚合，**不替代**套餐/换电/人天池各模块列表  
- **运营商信用评估**内的「变更记录」Tab = **档位定档历史**，与本模块不同域  

---
