# 员工相关能力标记二期 · 设计说明

## 目标

将原型中所有“员工账号与权限管理”能力统一标记为二期，避免评审人员将可浏览的 Mock 误认为一期交付范围，同时保留完整演示链路。

## 已确认方案

采用“完整标记、保留演示”：

1. 所有适用角色侧栏「员工」入口显示“二期”徽章。
2. 登录身份下拉中的员工账号统一增加「【二期】」前缀。
3. 员工页面继续保留增改、权限配置和员工登录演示，并展示“一期不交付”横幅。
4. PRD、验收标准、角色功能清单同步注明员工整块为二期。

## 备选方案与取舍

- 仅在员工页面标记：改动最小，但登录入口和侧栏仍会造成一期范围误解。
- 隐藏员工相关入口：范围最清晰，但会丢失二期方案评审和权限演示能力。
- 完整标记并保留演示（采用）：交付边界清晰，同时不损失原型价值。

## 原型改动

### 导航

- `employees` 继续保留在适用角色导航中。
- 复用现有 `phase2BadgeHtml()` 和二期视图元数据，不新增徽章样式。
- 平台、运营商、渠道商、资金方等出现员工入口时，均展示相同二期标记。

### 登录身份

- `employeeStore` 中生成的全部员工登录选项增加「【二期】」。
- 员工登录后的角色名、权限和数据隔离逻辑不变。
- 经营主体登录、站点合伙人登录不受影响。

### 员工页面

- 保留员工列表、新增/编辑员工、权限勾选、空态和无权限态。
- 页面顶部使用现有二期横幅，明确“员工账号与权限管理整块二期，一期不交付，原型仅演示”。

## 文档改动

- `docs/PRD.md`：角色能力、功能模块、权限章节中的员工描述统一追加“二期”。
- `docs/acceptance-criteria.md`：导航、员工页面、员工登录验收均标记二期；一期验收可不测。
- `docs/角色与功能清单.md`：所有角色员工功能状态统一为“二期 · 已实现（原型）”。
- 同步到 `prototype/docs/md/`、`docs/documentation/md/` 和 Pages 发布目录。

## 状态、权限与边界

- 二期标记只改变交付范围表达，不改变 Mock 数据、权限判断或路由。
- 无员工数据时仍显示“暂无员工”。
- 员工无 `employees.view` 权限时仍显示无权限提示。
- 渠道分销角色继续不提供员工功能，不因本次改动新增入口。

## 验证

1. 各适用角色侧栏员工入口均有二期徽章。
2. 登录下拉中的所有员工账号均带「【二期】」。
3. 员工页面显示二期横幅，增改与权限演示仍可操作。
4. 渠道分销登录无员工入口、无员工账号。
5. 三份核心文档与三端镜像内容一致。
6. 后台、骑手端、文档浏览器通过本地冒烟检查。

## Non-goals

- 不删除员工页面或 Mock 数据。
- 不调整权限模型、员工字段或数据范围。
- 不开发真实账号体系、邀请流程、密码管理或审计能力。
