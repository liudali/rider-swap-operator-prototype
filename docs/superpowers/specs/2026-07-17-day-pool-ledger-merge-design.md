# 人天额度池 · 额度明细并入额度池设计

## 目标

将渠道商人天池的顶层「额度明细」合并到「额度池」页面，减少顶层导航数量，同时保留额度账本的独立筛选和阅读空间。

## 已确认方案

1. 顶层 Tab 移除「额度明细」。
2. 「额度池」页面新增页内二级 Tab：
   - 额度池列表
   - 额度明细
3. 进入额度明细时默认展示全部额度池，不继承额度池列表的选中项。
4. 额度明细继续支持额度池、流水类型筛选。
5. 旧 `dayPoolTab=ledger` 自动兼容到「额度池 → 额度明细」。

## 信息架构

调整前：

```text
人天额度池
├── 额度池
├── 骑手团队
├── 骑手登记
├── 额度分配
├── 额度使用规则
├── 消耗记录
├── 零售价
├── 异常记录
└── 额度明细
```

调整后：

```text
人天额度池
├── 额度池
│   ├── 额度池列表
│   └── 额度明细
├── 骑手团队
├── 骑手登记
├── 额度分配
├── 额度使用规则
├── 消耗记录
├── 零售价
└── 异常记录
```

## 状态与路由

- 顶层状态继续使用 `state.dayPoolTab`，额度池固定为 `pools`。
- 新增 `state.dayPoolPoolsSubTab`：
  - `list`：额度池列表
  - `ledger`：额度明细
- 默认值为 `list`。
- 发现旧状态 `dayPoolTab === "ledger"` 时：
  - 设置 `dayPoolTab = "pools"`
  - 设置 `dayPoolPoolsSubTab = "ledger"`

## 筛选

- 额度池列表继续使用 `dayPool_pools` 筛选。
- 额度明细继续使用 `dayPool_ledger` 筛选。
- `pfKey()` 根据 `dayPoolPoolsSubTab` 返回对应筛选键。
- 首次进入额度明细时 `poolId` 默认为“全部”，不自动带入列表筛选或选中池。

## 页面与交互

### 额度池列表

- 保留现有 KPI、额度池表格、增购人天额度按钮。
- 页内 Tab 高亮“额度池列表”。
- 提供切换到“额度明细”的页内 Tab。

### 额度明细

- 保留现有账本字段：
  - 时间
  - 额度池
  - 类型
  - 变动（人天）
  - 余额后
  - 操作人
  - 关联
  - 原因
- 保留额度池、流水类型筛选。
- 页内 Tab 高亮“额度明细”。
- 默认展示全部额度池。

## 权限与边界

- 渠道主体账号和具备 `day_pool.view` 的员工可查看。
- 额度增购仍要求 `day_pool.edit` 或主体账号。
- 团队管理员仍固定进入消耗记录，不因本次合并改变。
- 不修改额度账本数据、流水类型或计算口径。

## 空态与兼容

- 无额度池：额度池列表显示“暂无额度池”。
- 无账本：额度明细显示“暂无额度变动”。
- 筛选无结果：显示当前筛选条件下暂无记录。
- 旧入口或旧状态自动跳转，不出现空白页面。

## Non-goals

- 不改额度账本实体和计算规则。
- 不把额度明细改为抽屉。
- 不联动列表选中的额度池。
- 不调整其他人天池 Tab 的顺序或功能。
