# 骑手用户端 · 双故事线主线设计

> 日期：2026-07-17  
> 阶段：MVP  
> 范围：用户端（微信小程序形态）文档清晰度 + 可交互原型主线  
> 真源原型：`prototype/mobile/index.html`  
> 关联：`docs/骑手端PRD.md` · `docs/个人套餐定价与退款.md` · `docs/天数池.md` · `docs/渠道骑手可换电校验.md` · `docs/用户-运营商归属模型.md`

## 1. 目标与成功标准

### 1.1 问题

当前骑手端原型身份过多（个人 / 人天池 / 设备租赁 / 骑士卡等并列），文档分散，缺少统一图例与步骤编号，评审与开发难以抓住一期主线。

### 1.2 目标

1. 用**双故事线导览**讲清两条完整生命周期：个人套餐用户、人天池用户。
2. 其他身份与异常收入「补充场景」，默认不打断主线。
3. 文档与原型共用同一套**步骤编号（Pxx / Cxx）**与**图例**。
4. 每个关键页面写清：字段、枚举、空态、错误态、个人 vs 人天池权限差异。

### 1.3 成功标准

| # | 标准 |
|---|------|
| S1 | 新同学可按导览连续走完个人主线与人天池主线各一遍，无需猜测下一步 |
| S2 | 演示控制台默认仅暴露两条主线身份；补充场景折叠可见 |
| S3 | `骑手端PRD` 成为用户端单一阅读入口，步骤编号与验收勾稽一致 |
| S4 | 服务卡 / 扫码结果 / 地图具备可读图例，状态语义不依赖口头解释 |

## 2. 已确认决策

| # | 决策 | 选择 |
|---|------|------|
| D1 | 生命周期终点 | **完整用户生命周期**（含续费/冻结/退款/退租；人天池含离职回池） |
| D2 | 信息组织 | **A · 双故事线导览**（非共享主干、非纯状态地图） |
| D3 | 其他身份处理 | **默认只展示两条主线**，其余收进「补充场景」 |
| D4 | 人天池起点 | **权益已到账**起；渠道采购/登记/分配为文档前置，不在骑手端主线演示 |

## 3. 整体结构

### 3.1 保留

- 底部三 Tab：首页 / 订单 / 我的
- 手机框 + 左侧控制台的双栏布局
- 现有扫码、购套餐、实名、选运营商等交互能力

### 3.2 改造重点：演示控制台 → 主线导览

```text
左侧控制台
├── 故事线切换：个人套餐 | 人天池
├── 步骤进度：P01–P09 或 C01–C08（高亮当前）
├── 当前步骤说明：前置条件 / 本步操作 / 成功结果
├── 快捷：上一步 / 下一步 / 跳到步骤
└── 补充场景（折叠）
    ├── 个人异常：定位失败、跨网、冷静期退款…
    ├── 人天池异常：预占失败、离职持电池…
    └── 二期：设备租赁、骑士卡、渠道停服专属站…
```

## 5. 主线一：个人套餐用户（P01–P09）

演示默认身份：`personal`（张骑手 U1028）；冷启动购套餐用 `none`。

| 步骤 | 名称 | 类型 | 用户端行为 | 成功结果 |
|------|------|------|------------|----------|
| P01 | 登录授权 | ● | 微信一键登录 + 协议勾选 | 进入已登录未开通/已有权益 |
| P02 | 实名认证 | ● | 姓名、身份证提交 | `kyc=true` |
| P03 | 确定服务运营商 | ●/◆ | 扫柜 > 会话上下文 > 定位；多 OP 弹选择 | `context_operator_id` 写入 |
| P04 | 购买套餐 | ● | SKU（方案 B）+ 新人价规则 + 押金同笔 Mock 支付 | `PackageOrder`；权益=个人套餐·服务中 |
| P05 | 首次领取电池 | ● | 无电池时扫码领取（非换电） | `holdsBattery=true` |
| P06 | 日常换电 | ● | 扫码 → 校验 → 四步柜门 → 成功单 | 换电订单；`swapUsed+1` |
| P07 | 续费 | ● | 续费原运营商，不重新定位 | 新购/续期套餐单 |
| P08 | 冻结与解冻 | ●/◆ | 须先还电；满足条件即时生效；解冻顺延 + 待领电池 | 状态：已冻结 ↔ 服务中·待领电 |
| P09 | 退款/退租/完结 | ● | 冷静期或中途退款；须还电；展示建议退款与审核态 | 服务终止；押金按规则退 |

### 5.1 个人关键状态枚举

`未登录` · `未实名` · `未开通` · `服务中` · `待领取电池` · `已冻结` · `退款申请中` · `退租中` · `已完结` · `套餐过期` · `次数耗尽`

### 5.2 个人权限（相对人天池）

- 可：购套餐、续费、冻结/解冻、冷静期退款、中途退款、退租退押。
- 持电池时：禁止冻结、退款、退租 → 统一提示先还电。
- 已有有效套餐：续费锁定原运营商。

## 6. 主线二：人天池用户（C01–C08）

演示默认身份：`channel`（孙骑手 U2110）；C06 切入 `channel_fail`（赵骑手 U2103）。

**前置（■，文档说明，骑手端不演示）**：渠道向运营商采购人天 → 登记骑手入团队 → 分配人天（首版 `on_allocate`）→ 生成/激活权益。

| 步骤 | 名称 | 类型 | 用户端行为 | 成功结果 |
|------|------|------|------------|----------|
| C01 | 登录授权 | ● | 同个人；登录后不引导购主套餐 | 进入渠道成员服务态 |
| C02 | 权益到账 | ◆/只读 | 展示渠道、团队、poolId、剩余人天 | 服务卡=渠道人天 |
| C03 | 今日预占 | ◆ | 展示今日已预占 / 已确认 / 失败 | 已预占则可换电 |
| C04 | 首次领取电池 | ● | 同扫码领取交互；校验走渠道权益 | 持电池 |
| C05 | 日常换电与确认消耗 | ●/◆ | 换电成功；结果页展示「今日已确认 · 不重复扣」 | 换电单 + 今日确认消耗 |
| C06 | 额度异常与自费兜底 | ! | 预占失败展示原因；购 1天/单次；**不扣池** | 临时可换电 |
| C07 | 离职/移除 | ■→用户只读 | 服务终止；持电池仅可还电 | 状态=渠道停用/已离职 |
| C08 | 还电与回池完结 | ●/◆ | 还电入柜；剩余人天回池（不向骑手退款） | 服务完结 |

### 6.1 人天池关键状态枚举

`待分配` · `已分配·待预占` · `今日已预占` · `今日已确认消耗` · `预占失败·权益不可用` · `自费兜底服务中` · `渠道停用·可还电` · `已离职·回池完结`

### 6.2 人天池权限（相对个人）

- 无主套餐自购入口（「购买套餐」隐藏或弱化；兜底 SKU 仅出现在 C06）。
- 不可：冻结、冷静期退款、中途退款（`source=day_pool`）。
- 与个人套餐互斥：补充场景说明，主线不演示 E004 冲突。
- 跨网规则与个人共用；`userOwner` = 额度售卖运营商。

## 7. 补充场景（默认折叠）

### 7.1 个人侧

- 定位拒绝 / 15km 无站 / 多运营商选择（`geo_ambiguous`）
- 支付失败、柜机超时/无空电
- 跨网案例3（`personal_cross3` + `ST-BJ-01`）
- 冷静期退款（`cooling_refund`）
- 中途完结退款

### 7.2 人天池侧

- 预占失败自费兜底（主线 C06 可一键进入）
- 持电池确认消耗（0 次换电仍确认 1 人天）— 结果页文案演示
- 离职持电池仅还电
- 个人与渠道互斥（文档 + 可选演示身份，默认隐藏）

### 7.3 二期收纳（⋯）

`lease_*`、`card_link_*`、渠道专属站白名单等：保留代码能力，演示台标「二期」，主线默认不选中。

## 8. 页面规格摘要（实施时写入骑手端 PRD 全文）

以下为每页必列清单；实施阶段补全字段表。

### 8.1 首页

| 项 | 说明 |
|----|------|
| 字段 | 站点 Marker、当前定位、客服条、扫一扫 FAB |
| 枚举 | 公开站 / 专属站（二期） |
| 空态 | 附近无站提示（定位场景） |
| 错误态 | 定位拒绝 |
| 权限差异 | 人天池与个人地图交互相同；专属站仅白名单（二期） |
| 图例 | 地图角标：公开站含义；补充场景说明专属站 |

### 8.2 扫码换电 / 领电池

| 项 | 说明 |
|----|------|
| 字段 | 柜机名、站点、流程步骤、电池 SN、SOC；成功页展示单号/站点/SOC（人天池另示「今日已确认」）；跨网三元组与费用仅后台记账，骑手端不展示 |
| 枚举 | 领取电池 / 换电 / 仅还电 |
| 空态 | — |
| 错误态 | 未登录、未开通、已冻结、预占失败、跨网关闭、柜机异常 |
| 权限差异 | 个人走套餐校验；人天池走 entitlement check；失败人天池出兜底 SKU |

### 8.3 我的 · 服务状态卡

| 项 | 说明 |
|----|------|
| 字段 | 权益类型、套餐名/人天剩余、今日资格、持电池、主按钮 |
| 枚举 | 见 §5.1 / §6.1 |
| 空态 | 未开通 / 待分配 |
| 错误态 | 权益不可用 + 原因文案 |
| 权限差异 | 主按钮：个人=服务购买/续费/解冻；人天池=联系渠道或自费换电 |

### 8.4 服务购买

| 项 | 说明 |
|----|------|
| 字段 | 收款运营商、SKU、价格、有效期口径、新人标、协议、押金 |
| 枚举 | 方案 B SKU；默认套餐仅渠道兜底路径 |
| 空态 | 无法解析运营商时阻断 |
| 错误态 | 未实名、未勾选协议、支付失败 |
| 权限差异 | 人天池主线不进本页；仅 C06 展示 1天/单次 |

### 8.5 订单三 Tab

| 子 Tab | 个人 | 人天池 |
|--------|------|--------|
| 换电订单 | 有；权益类型=个人套餐 | 有；权益类型=渠道人天/渠道兜底自费 |
| 服务购买 | 有套餐单 | 通常无；兜底支付后有 `channel_retail_fallback` |
| 退租订单 | 有 | 通常无；离职还电可记还电类记录 |

空态文案保持现有：「暂无换电/购买/退租记录」。

## 9. 原型改造要点

1. 左侧改为「故事线 + 步骤进度 + 补充场景折叠」。
2. 默认选中：个人 `personal` 或人天池 `channel`；去掉默认堆叠二期身份。
3. 「下一步」驱动：切换身份/字段到该步所需状态，并打开对应 Overlay 或 Tab。
4. 人天池换电成功页增加「今日人天：已确认」说明。
5. 设置页演示身份列表与控制台分组对齐（含 `pending_pickup` 等遗漏项）。
6. 同步：改完后跑 `scripts/sync-pages.sh`。

## 10. Non-goals

- 不重做运营商/渠道后台信息架构
- 不接真实微信登录、支付、地图 SDK、IoT
- 不把设备租赁 / 骑士卡升为一期主线
- 不在骑手端演示渠道采购→登记→分配全流程
- 不新增 OpenAPI 正式定义（规则仍引用现有专文）

## 11. 风险与补漏

| 风险 | 缓解 |
|------|------|
| 双故事线步骤重复（登录/领电/换电） | 文档交叉引用「同 P0x 交互，差异仅权益校验」；原型复用同一套 UI |
| 一期权益类型未来超过两种 | 失效条件：改回共享主干 + 分叉（见决策卡片） |
| 文档与原型再次漂移 | 验收按步骤编号；变更写 `原型变更记录` |
| 埋点 | 主线关键：登录成功、购套餐支付、换电成功、预占失败曝光、自费兜底支付、冻结/解冻、冷静期申请（写入骑手端 PRD 时对齐 `最小埋点清单`） |
