# 平台管理员管理 · 设计说明

## 目标

平台管理员侧新增一期模块「管理员」，用于维护平台内部人员及菜单权限。实现复用现有员工模块，不重复建设列表、表单和权限勾选能力。

## 已确认方案

1. 复用现有 `employees` 视图、`employeeStore`、员工表单和权限判断。
2. 平台角色下显示「管理员」，属于一期，不显示二期徽章。
3. 运营商、渠道商、资金方仍显示「员工」，整块二期。
4. 平台管理员账号登录不加「【二期】」；其他主体员工账号继续加「【二期】」。
5. 采用“角色模板 + 可自定义菜单权限”。
6. 超级管理员不可修改权限、不可停用。

## 信息架构

平台管理员侧栏增加：

```text
平台统价
管理员
```

沿用 `employees` 视图键，按当前角色动态显示：

- 平台角色：管理员（一期）
- 其他适用角色：员工（二期）

## 数据模型

平台管理员仍存放在 `employeeStore[ENT.platform.id]`，新增字段：

| 字段 | 说明 |
|------|------|
| `id` | 管理员 ID，如 `ADM-PLAT-001` |
| `roleType` | 固定 `staff` |
| `name` | 姓名 |
| `phone` | 手机号 |
| `jobTitle` | 角色模板名称 |
| `adminTemplate` | `super` / `operations` / `finance` / `custom` |
| `status` | `启用` / `停用` |
| `permissions` | 平台菜单权限 ID 数组 |
| `lastLoginAt` | 最后登录时间 |
| `protected` | 超级管理员保护标记 |

## 角色模板

| 模板 | 默认权限 |
|------|----------|
| 超级管理员 | 全部平台菜单；权限不可编辑、账号不可停用 |
| 运营管理员 | 总览、用户、订单、设备、渠道、运营商、统价、变更记录 |
| 财务管理员 | 总览、流水、平台账户、提现审核、保证金管理、平台服务费 |
| 自定义 | 创建人手动勾选平台菜单 |

模板负责初始化权限；除超级管理员外，选择模板后仍可调整勾选项，保存时记为自定义权限快照。

## 权限与导航

- 新增平台菜单权限映射，权限粒度为页面查看权。
- 平台主体登录视为超级管理员，可见全部菜单。
- 管理员登录仅展示 `permissions` 映射到的平台菜单。
- 管理员无权访问的深链自动回退到首个有权菜单。
- 当前登录管理员不可停用自己。
- `protected=true` 的超级管理员不展示停用和权限编辑操作。

## 登录身份

登录下拉新增“管理员登录”分组：

- 平台管理员账号：不加二期前缀。
- 其他主体员工账号：保留「【二期】」前缀并位于“员工登录（二期）”分组。
- 平台主体登录入口继续保留，代表超级管理员主体视角。

## 页面与交互

平台角色进入 `employees` 时：

- 页面标题：管理员
- KPI：管理员总数、启用数、角色模板数
- 列表字段：姓名/手机号、角色模板、状态、权限摘要、最后登录、操作
- 操作：新增管理员、编辑、启用/停用
- 表单：姓名、手机号、角色模板、状态、菜单权限

其他角色进入 `employees` 时维持现有员工页面与二期横幅。

## 状态与异常

- 空态：暂无管理员，提示“新增管理员并配置菜单权限”。
- 手机号重复：阻止保存并提示“该手机号已存在”。
- 权限为空：阻止保存并提示“至少选择一个菜单权限”。
- 停用自己：阻止并提示“不可停用当前登录账号”。
- 编辑超级管理员：权限与状态控件只读。
- 无权限深链：回退至有权菜单，不展示空白页。

## 验证

1. 平台侧栏显示“管理员”，无二期徽章。
2. 其他主体侧栏“员工”仍有二期徽章。
3. 平台管理员登录选项无二期前缀；其他员工仍有前缀。
4. 运营/财务模板初始化正确，允许调整权限。
5. 超级管理员不可编辑权限或停用。
6. 管理员登录后仅显示授权平台菜单。
7. 三份核心文档及三端镜像一致。

## Non-goals

- 不接真实密码、短信、邀请或找回密码。
- 不做数据行级权限，仅做平台菜单级权限。
- 不做审批流、权限版本历史或登录审计详情。
- 不把其他主体员工能力拉回一期。
