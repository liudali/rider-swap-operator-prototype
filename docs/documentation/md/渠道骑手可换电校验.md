# 渠道骑手可换电校验（换电前）

> 换电流程第一步：校验骑手当日是否可使用**渠道人天**或**有效个人套餐**。  
> 原型演示：`channel_fail`（U2103 预占失败·待还电）、`channel_no_quota`（U2112 个人无额度·待还电）  
> **decision-054**：已取消运营商自费兜底 SKU。

---

## 1. 接口概览

| 项目 | 说明 |
|------|------|
| 路径 | `POST /api/v1/entitlement/check` |
| 调用方 | 骑手 App / 小程序（扫码换电前） |
| 幂等 | 同一 user + 自然日可缓存结果；换电 start 须再次校验 |

---

## 2. 请求

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `user_id` | string | Y | 骑手 ID |
| `cabinet_sn` | string | Y | 柜机 SN |
| `site_id` | string | N | 站点 ID（推荐传） |
| `request_time` | datetime | N | 客户端时间；服务端以服务器时间为准 |

```json
{
  "user_id": "U2103",
  "cabinet_sn": "CAB-22018",
  "site_id": "ST-SH-01"
}
```

---

## 3. 响应

| 字段 | 类型 | 说明 |
|------|------|------|
| `allowed_swap` | boolean | 是否允许换电/出电 |
| `allowed_return` | boolean | 是否允许还电入柜 |
| `entitlement_type` | enum | `day_pool` / `personal_pkg` / `none` |
| `pool_id` | string | 渠道成员所属额度池 |
| `seller_operator_id` | string | 额度售卖方 U |
| `today_status` | enum | `reserved` / `confirmed` / `failed` / `pending_return` / `not_allocated` |
| `fail_reason` | string | 失败原因文案 |
| `gate_reason` | enum | `个人无额度` / `预占失败` / null |

**成功示例（渠道已预占）**

```json
{
  "allowed_swap": true,
  "allowed_return": true,
  "entitlement_type": "day_pool",
  "pool_id": "QP-2601",
  "seller_operator_id": "OP-SX",
  "today_status": "reserved",
  "fail_reason": null,
  "gate_reason": null
}
```

**失败示例（预占失败 · 持电池仅还电）**

```json
{
  "allowed_swap": false,
  "allowed_return": true,
  "entitlement_type": "day_pool",
  "pool_id": "QP-2601",
  "seller_operator_id": "OP-SX",
  "today_status": "pending_return",
  "fail_reason": "渠道额度池余额不足，今日预占失败",
  "gate_reason": "预占失败"
}
```

> **已废弃**：响应中的 `fallback` 对象（自费兜底 SKU）。一期不再返回，也不引导 C 端应急购套餐。

---

## 4. 错误码

| 码 | 含义 | 骑手端处理 |
|----|------|------------|
| `E001` | 未登记渠道且无有效个人套餐 | 引导服务购买（仅个人路径） |
| `E002` | 预占失败（池余额不足） | 展示原因；**禁止换电**；持电池→仅可还电（E007） |
| `E003` | 无匹配额度使用规则 | 联系渠道商 |
| `E004` | 个人与渠道身份冲突 | 提示退订/冻结个人套餐 |
| `E006` | 个人无额度·待还电 | `allowed_swap=false`，`allowed_return=true` |
| `E007` | 预占失败·待还电 | 同上；`gate_reason=预占失败` |

---

## 5. 与换电 start 的关系

```
扫码 → entitlement/check
  ├─ allowed_swap=true  → POST /api/v1/swap/start
  ├─ allowed_swap=false && allowed_return=true && 持电池 → 仅还电入柜
  └─ allowed_swap=false && 未持电池 → 拦截，展示 fail_reason（联系渠道续配）
```

跨网/跨站准入在 `swap/start` 前另调运营商策略（与本文档独立）。
