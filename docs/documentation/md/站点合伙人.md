# 站点合伙人

> 运营商在**站点管理**中为各站配置场地合伙人（个人或公司），从 C 端确认收入中向合伙人切分约定比例。**一站可多位合伙人；同一合伙人可关联多站，各站比例独立设置。**

关联：[合伙人站点分佣.md](./合伙人站点分佣.md) · [PRD.md](./PRD.md) §5.13 · [合作模式与分账.md](./合作模式与分账.md) §9 · [站点合伙人-Mock字段清单.md](./站点合伙人-Mock字段清单.md)

---

## 1. 角色与边界

| 角色 | 职责 |
|------|------|
| **运营商** | 维护合伙人档案（个人/公司）；在**站点管理**中为各站添加合伙人并设比例；查看分润明细 |
| **站点合伙人** | 场地方/物业等；按绑定站点查看分润明细、申请提现（**独立登录**） |
| **平台** | 不参与合伙人切分谈判；C 端仍先清分 **平台 1% + 运营商净额**，合伙人份额从运营商 99% 中切出 |

**合伙人类型**

| 类型 | 档案字段 |
|------|----------|
| **个人** | 姓名、手机、收款账户 |
| **公司** | 公司全称、联系人、电话、对公收款账户 |

**与渠道商区别**：渠道商是 B2B 采购方；站点合伙人是**场地分润方**。

**与站点支出区别**：站点支出 = 运营商**付费**给场地方；站点合伙人 = 从**经营收入分润**。可同人不同账。

## 2. 核心规则

### 2.1 关系模型（关键）

| 维度 | 规则 |
|------|------|
| **绑定粒度** | **站点 × 合伙人**；**一站可绑定多位**合伙人 |
| **同人多站** | 同一 `partner_id` 可绑定多个站点，**各站 R% 可不同** |
| **比例设置** | 运营商在**站点管理 → 合伙人**为每位合伙人单独设 `rate_pct` |
| **比例上限** | 同一站点所有合伙人比例**合计 ≤ 99%**（与平台 1% 合计 100%） |
| **配置入口** | **站点管理**（主）；站点合伙人菜单提供档案与明细 |

**示例**：浦东站 — 王场地方（个人）25% + 上海李物业（公司）5%；世博站 — 王场地方 20%（待生效 25%）。

### 2.2 清分公式（多合伙人）

计提基数 `B` = 该站点个人用户已确认可分配收入。

```
platform_amount   = round(B × 1%, 3)
partner_i_amount  = round(B × R_i / 100, 3)   // 每位合伙人按本站比例
operator_amount   = B − platform_amount − Σ partner_i_amount
```

**不计入**：渠道人天、设备租赁白名单、跨网费。

### 2.3 生效时间

绑定/解绑、比例变更 → **次日 0:00** 生效。

### 2.4 渠道专属站

`channel_dedicated=true` 默认不参与；若绑定须运营商显式配置。

## 3. 后台菜单

### 3.1 运营商

| 模块 | 说明 |
|------|------|
| **站点 → 合伙人** | **主配置入口**：添加/解绑合伙人、设本站比例 |
| **站点合伙人 → 档案** | 个人/公司合伙人档案维护 |
| **站点合伙人 → 分润绑定一览** | 只读汇总；跳转站点配置 |
| **站点合伙人 → 分润明细** | 按站点/合伙人/日期查询切分 |

### 3.2 站点合伙人（独立登录 · 只读配置）

| 模块 | 说明 |
|------|------|
| **总览** | 累计/本月分润 KPI；绑定站点摘要；最近分润 |
| **我的站点** | 各站当前比例与待生效规则（**不可自行修改**） |
| **分润明细** | 本人分润切分行；支持站点/日期筛选 |
| **提现结算** | 可提现余额；向**运营商**发起代付申请 |
| **收款账户** | 档案收款信息只读 |

> 提现由**运营商审核**后打款，非平台审核。演示登录：「站点合伙人 · 王场地方」。

## 4. 字段建议

| 实体 | 字段 |
|------|------|
| `SitePartner` | `id`, `operator_id`, `partner_type`（个人/公司）, `name`, `contact_name`, `phone`, `bank_account`, `status` |
| `SitePartnerBinding` | `id`, `site_id`, `partner_id`, `rate_pct`, `pending_rate_pct`, `effective_at`, `status` |
| `SplitLine` | `site_id`, `partner_id`, `partner_rate_pct`, `share_base`, `partner_amount`, … |

## 5. 验收要点

| # | 场景 | 预期 |
|---|------|------|
| 1 | 一站多合伙人 | 浦东站 王场地方 25% + 李物业 5%；合计 30% |
| 2 | 同人多站 | 王场地方浦东 25%、世博 20%；各站独立 |
| 3 | 个人/公司 | 档案区分类型；公司须填联系人 |
| 4 | 站点管理配置 | 站点列表「合伙人」按钮打开抽屉配置 |
| 5 | 比例变更 | 次日 0:00 生效；展示待生效 |
| 6 | 渠道人天 | 不产生合伙人分润 |
| 7 | 合伙人登录 | 王场地方可见浦东 25%+世博 20%；可提现 ¥12.25（演示） |
| 8 | 合伙人提现 | 向运营商申请；待审/已到账状态可演示 |
