# 渠道结算模式规则

> **状态**：已确认（2026-06-17，§3 修订 2026-06-12）  
> **决策来源**：[需求评审决策记录.md](./需求评审决策记录.md) #3B、C-06–C-08  
> **关联**：D03 单运营商签约、**D05 四种模式**、D16/D38 渠道信用与免押  
> **交付范围（2026-07-14 · `decision-003`）**：**人天池**、**渠道分销（卡差价）** 为一期；**设备租赁**（`CH-RENT`）、**激活码**（`CH-ACT`）整模式标为**二期**（原型可浏览，非一期交付）。

---

## 1. 共性规则

| 规则 | 说明 |
|------|------|
| **签约** | 一期每渠道**仅签一家运营商** |
| **userOwner** | 经渠道链接购卡的用户 `userOwner` = 签约运营商 U |
| **跨网** | **继承运营商换电范围**（4A） |
| **渠道免押** | **人天池 / 设备租赁 / 激活码**：押金算渠道 → [渠道信用评估方案.md](./渠道信用评估方案.md)。**渠道分销**（骑士卡等）：用户直购，按个人用户押金/免押规则，**不适用**渠道信用 |
| **C 端兜底** | 人天池/分销/设备租赁见各模式；**激活码**用户不经平台收款，无 C 端支付兜底 |
| **清分** | C 端实收：**支付成功实时清分**（1B） |

## 2. 模式一：人天额度池（人天池）

> 详规见 [天数池.md](./天数池.md)

| 维度 | 规则 |
|------|------|
| **采购** | 渠道向 U 批量购买人天；B2B 款采购时结清 |
| **消耗** | 预占确认：换电或持电池扣 1 人天/日 |
| **平台 1%** | **确认消耗时**按标准人天价 × 1% 向 U 计提 |
| **兜底** | 池不足时骑手 C 端在线支付（1天/单次 SKU） |

## 3. 模式二：渠道分销（骑士卡）

> **2026-06-12 修订**：骑士卡渠道为**销售渠道**，非 B2B 批发入库再分配。用户通过渠道**推广链接**直购套餐；系统标记渠道来源；运营商与渠道**佣金结算**。

### 3.1 商业模式

渠道商（如骑士卡渠道）是运营商的**分销/推广渠道**：

- 运营商维护套餐**正式零售价**（定价管理）。
- 运营商与渠道协商**渠道专享价**（低于正式价，按 SKU）。
- 渠道获得**专属推广链接**；用户点击链接**直接购买**，无需渠道后台分配卡片。
- 订单与用户打上**渠道来源标记**（`channel_id` / `link_code`）。
- 渠道收入 = 运营商按约定支付的**佣金**（按单固定金额或比例，签约时配置）。

**不存在**：渠道批发购卡、卡库存、持卡骑手分配、CO 批发订单。

### 3.2 计费与收银（C-06 / C-08 · 已确认）

| 时点 | 规则 |
|------|------|
| **用户购卡** | 经渠道链接以**渠道专享价**支付；**平台代收**（款进运营商子商户）；**支付成功实时清分** 1% → 平台，余款 → 运营商 |
| **渠道标记** | 订单、用户档案写入渠道 ID；换电权益随套餐订单即时开通 |
| **平台 1%** | **购卡支付成功时**按实付金额 × 1% 计提（C-06A） |
| **渠道佣金** | **默认**：按签约固定额/单，运营商与渠道**线下结算**。**佣金及时到付**（可选）：实付 × 运营商设定比例，支付成功**即时分账**至渠道子商户 |
| **换电权益** | 与个人套餐相同，有效期内按 SKU 规则换电；**不扣人天池** |
| **跨网费** | 同其他用户，U 承担日清跨网设备服务费 |
| **C 端兜底** | 平台 1% + 运营商净额（开启即时到付时另扣渠道佣金比例） |

### 3.2.1 佣金及时到付（2026-07-07）

仅**渠道分销（骑士卡）**签约可配置；与人天池/激活码无关。

| 维度 | 规则 |
|------|------|
| **开关** | 运营商在「渠道管理 → 签约渠道 → 编辑」开启 `instant_commission_payout` |
| **前置** | 渠道在「收款账户」绑定微信/支付宝子商户且状态**已开通**；未进件不可正式开启 |
| **佣金比例** | 运营商设置 `commission_rate`（如实付的 9%）；平台服务费仍默认 **1%** |
| **分账时点** | 用户经骑士卡链接支付成功 → 同步清分：平台 1% + 渠道佣金 + 运营商净额 |
| **对账** | 「佣金对账」展示已即时分账明细；未开启渠道仍显示「待线下结算」 |
| **演示** | `CH-CARD` 已开启（9%）；`CH-DELIV` 未开启（对比线下结） |

### 3.2.2 渠道差异化定价（2026-06-12）

同一运营商下，**不同分销渠道**可配置不同的渠道专享价与佣金。

| 维度 | 规则 |
|------|------|
| **定价粒度** | 签约渠道 × 套餐 SKU |
| **正式零售价** | 运营商「定价管理 → 个人套餐价」；全渠道基准价 |
| **渠道专享价** | 运营商与渠道协商；**须 ≤ 正式零售价**；用户经链接按此价支付 |
| **佣金** | 按单固定金额或实付比例；签约时配置，对账按单汇总 |
| **配置入口** | 运营商「**渠道管理 → 签约渠道**」；「**定价管理 → 渠道分销价**」汇总对比 |

### 3.2.3 多渠道分销 · 套餐授权与定价

同一运营商下可并存**多个分销渠道**（演示：`CH-CARD` 骑士卡、`CH-DELIV` 闪送骑士卡），管理原则：

| 步骤 | 操作 |
|------|------|
| 1. 签约 | 「渠道管理 → 签约渠道」为每个渠道商创建独立合同，结算模式 = 渠道分销 |
| 2. 授权 SKU | 编辑签约时为该渠道勾选可售套餐（如 30 天包月、7 天卡） |
| 3. 定价 | 每个渠道×SKU 分别配置**正式零售价**（只读基准）、**渠道专享价**、**佣金/单** |
| 4. 汇总 | 「定价管理 → 渠道分销价」Tab **横向对比**各渠道同 SKU 价格 |
| 5. 渠道侧 | 渠道登录后在「套餐与链接」基于授权 SKU 生成链接/二维码；**不可改价** |
| 6. 成交 | 用户经链接 24h 内购授权 SKU 享该渠道专享价；佣金按渠道规则结算 |

**渠道商权益（分销）**：专享价销售权、推广链接/二维码、成交与佣金对账；**不含**统管骑手押金/团队（与人天池区分）。

| **约束** | 渠道专享价 ≤ 正式零售价；调价对新单生效 |

**示例**（同一运营商 OP-SX）：

| 渠道 | SKU | 正式零售价 | 渠道专享价 | 佣金/单 |
|------|-----|------------|------------|---------|
| 骑士卡渠道 | 包月 30 天 | ¥299 | ¥279 | ¥25 |
| 闪送骑士卡 | 包月 30 天 | ¥299 | ¥269 | ¥30 |

### 3.3 推广链接、二维码与小程序落地

渠道在「**套餐与链接**」管理可售套餐，并为同一套餐生成**多条推广链接**（如包月 ¥279 可同时有 App Banner、短信、社群等链接）。每条链接可**一键生成二维码**，用于地推海报、社群、短信等场景。

| 字段 | 说明 |
|------|------|
| `package_id` / `sku_id` | 所属可售套餐（可选预选 SKU） |
| `link_code` | 渠道 + SKU + 链接唯一短码 |
| `purpose` | **链接用途**（渠道自定义，如「App 首页 Banner」） |
| `link_url` | 运营商**小程序路径**，带 `op`、`ch`、`lnk`、可选 `sku` |
| `qr_code` | 与 `link_url` 内容一致的二维码图片（PNG/SVG，可下载） |
| 统计 | **每条链接**独立统计点击、成交（链接购卡成功笔数） |
| 权限 | 渠道可新建/停用链接、下载二维码；**不可改价**（专享价由运营商签约配置） |

**用户路径（C 端）**：

1. 用户点击推广链接或**扫描二维码** → **直达签约运营商小程序**（非 H5 中转页）。
2. 完成**微信授权登录/注册** → **实名认证**（购套餐前置条件）。
3. 套餐列表展示价格：若处于渠道归因窗口内，展示**渠道专享价**；否则展示运营商**正式零售价**。
4. 支付成功 → 订单与用户写入渠道来源标记；换电权益即时开通。

**链接 URL 示例**（演示）：

```
wxmp://OP-SX/pages/landing/index?op=OP-SX&ch=CH-CARD&sku=SKU-30D&lnk=qsk-30d-home
```

### 3.3.1 24 小时渠道归因窗口

| 维度 | 规则 |
|------|------|
| **触发** | 用户**首次点击/扫码**某条**有效**推广链接时，记录 `attribution_start` |
| **窗口** | 自点击时刻起 **24 小时**（自然时间） |
| **价格范围** | 窗口内用户购买该渠道签约授权的**全部 SKU**，均按**渠道专享价**展示与结算（不限于链接绑定的单一 SKU） |
| **订单归属** | 归因期内成交订单写入 `channel_id`；`link_code` 取**首触链接**（用于链接级转化统计） |
| **窗口续期** | 归因期内再次点击**同一渠道**其他链接 → **不重置** 24h 窗口（首版默认） |
| **渠道切换** | 窗口内点击**其他渠道**链接 → 以**最后触达渠道**为准，**重置** 24h 窗口 |
| **过期** | 超过 24h 且未支付 → 恢复**正式零售价**；已支付订单不受影响 |

> **与非归因用户差异**：未点击渠道链接的用户，在运营商小程序内按**正式零售价**购套餐，订单**无**渠道标记、**不计**渠道佣金。

### 3.3.2 购卡记录

| 字段 | 说明 |
|------|------|
| 范围 | **仅**经本渠道推广链接成交的套餐购买 |
| 筛选 | 支付时间起止、套餐、链接用途、用户手机 |
| 关联 | 订单写入 `channel_id`、`link_code`、`link_purpose` |

### 3.4 佣金对账（平台提供）

| 字段 | 说明 |
|------|------|
| 周期 | **自然月**（非按单滚动） |
| 链接购卡笔数 | 当月带渠道标记的套餐订单 |
| 实付合计 | 当月用户经链接支付总额 |
| **应结佣金** | 按签约规则汇总 |
| 平台 1% | 已清分明细 |
| 明细 | 可选某月展开订单明细（渠道侧） |

### 3.5 后台菜单

总览、结算说明、**套餐与链接**、**购卡记录**、**佣金对账**、变更记录

> **不含**「渠道信用额度」：分销渠道无统管骑手/设备押金，不适用渠道信用评估。  
> **不含**「员工」：分销渠道无后台运营分工，仅渠道主体账号登录查看链接与对账。

> **说明**：原「CO 批发购卡 → 卡库存 → 持卡分配」路径**已废弃**。第三方 App SDK 对接（原美团骑士卡）**已取消**。一期仅走 **推广链接 → 用户直购 → 渠道标记 → 佣金结算**。

## 4. 模式三：设备租赁 · **二期**

> **交付**：整模式二期（原型标「二期」）。  
> **2026-07-01 修订**：原「设备租金池」更名为**设备租赁**。取消「预缴租金池余额 + 骑手兜底 SKU」模型；改为**运营商维护租赁设备清单与月租**，渠道**白名单用户**免 C 端购套餐、免按次计费。

### 4.1 商业模式

渠道向签约运营商**租赁换电柜/电池**，按设备清单缴纳**月租**（B2B）。运营商为渠道维护：

- **租赁设备清单**（柜机/电池 SN、部署站点；**不含单台月租**）
- **签约统一月租**（渠道与运营商 B2B 约定的一口价，MO 账单按此金额）
- **渠道专属站点**（专用载体：不对公众开放，仅白名单可见可用）
- **白名单用户**（由**渠道商自行维护**；仅名单内骑手可换电）

渠道 B2B 向运营商付月租；白名单骑手换电**不扣人天池、不按 C 端 SKU 计费**（付费类型须购白名单套餐）。服务通过**专属站点 + 白名单**闭环，与普通 C 端开放站点隔离。设备租赁渠道可视为**小型运营商**（userOwner=渠道），可选开通**跨网换电**。

### 4.2 运营商设置（签约与资产）

| 能力 | 规则 |
|------|------|
| **签约** | 结算模式 = 设备租赁；合同有效期、主体信息 |
| **设备清单** | 运营商维护：绑定 SN、类型、部署站点；**不对单台设备定价** |
| **月租** | 签约时与渠道约定**统一月租**（元/月）；MO 账单按签约价；**增删设备不改变月租**（除非重新签约） |
| **专属站点** | 签约时**新建**或绑定站点，标记 `channel_dedicated=true`、`public_open=false` |
| **欠费策略** | 月租未到账：**停服**（白名单骑手不可换电）；**不走** C 端兜底 SKU |
| **渠道停服** | 运营商将渠道商主体 `status=已停用`：**白名单不可取电/不出电**；扫码提示「当前站点已到期，请联系管理员续费」（`E_LEASE_CHANNEL_SUSPENDED`）；**仍允许还电入柜** |

### 4.2.1 渠道专属站点（专用 · 白名单可见）

| 维度 | 规则 |
|------|------|
| **定位** | 为签约渠道提供换电服务的**专用站点**；与普通「开放站点」区分 |
| **公众开放** | `public_open=false`：**不对普通 C 端用户开放**；不在公众地图/站点列表展示 |
| **小程序可见性** | 骑手端地图、附近站点、定位检索：**仅对该渠道白名单用户**返回专属站 POI |
| **非白名单** | 地图**不可见**；若通过柜机码等方式触达，扫码拦截 `E_WHITELIST` |
| **运营商** | 创建/绑定专属站、维护设备与月租；**不维护**渠道白名单 |
| **渠道商** | 查看专属站与租赁设备；**自行维护**白名单（增删骑手） |

### 4.3 白名单用户

| 维度 | 规则 |
|------|------|
| **准入** | 仅渠道**添加**的手机号/骑手可进入本模式；**由渠道商自行维护** |
| **组织** | **无团队/车队属性**（与人天池的团队登记不同）；扁平白名单 |
| **白名单类型** | 添加时区分：<br>· **白名单免费**：B2B 月租已覆盖，**免 C 端购套餐**即可换电<br>· **白名单付费**：须购买「白名单套餐」后方可换电 |
| **签约默认** | 运营商签约时可配置 `whitelist_default_access`（免费/付费），作为新增骑手默认类型 |
| **权益** | 免押（押金算渠道信用）；可在小程序看到并导航至**本渠道专属站** |
| **购套餐** | **白名单付费**可见「购买套餐」入口与渠道 SKU；**白名单免费**无购套餐要求 |
| **非白名单** | 专属站地图不可见；扫码拦截：`E_WHITELIST` |
| **维护方** | **渠道商**在「白名单用户」增删并选择类型；运营商**仅查看** |
| **离职/移除** | 移出白名单即停服；若持有电池须先归还；专属站对其不可见 |
| **渠道已停用** | 运营商停用渠道商主体后，名单内用户**暂停取电**；**还电入柜不受限** |

### 4.3.1 白名单套餐与收款（2026-07-03）

| 维度 | 规则 |
|------|------|
| **定价方** | **渠道商**在「白名单套餐」自行维护 SKU（名称、零售价、有效期）；**运营商不代设**白名单 C 端价 |
| **可见范围** | 小程序套餐列表**仅对白名单用户**展示；与普通 C 端开放 SKU **隔离** |
| **收款主体** | 用户支付进入**渠道商进件子商户**（`seller_channel_id`）；**非**运营商子商户 |
| **收款账户** | 渠道商须维护「**收款账户**」并完成进件；未进件不可上架白名单套餐 |
| **平台 1%** | **支付成功时**从渠道子商户实时分账 1%（与个人/分销 C 端同类） |
| **运营商 B2B** | 设备月租 MO 仍付运营商；**与白名单 C 端套餐款无关** |
| **订单归属** | `PackageOrder.seller_channel_id = CH`；`device_owner_id` 仍为签约运营商 U（换电三元组不变） |
| **退款** | 由**渠道子商户**原路退（若启用退款能力）；运营商退款管理**不处理**该笔 |

### 4.3.3 渠道跨网换电（小型运营商）

| 维度 | 规则 |
|------|------|
| **定位** | 设备租赁渠道视为**小型运营商**：名下骑手 `userOwner = channel_id` |
| **开通条件** | 向平台缴纳**跨网保证金**（演示 ¥20,000）；签约时可配置是否开通 |
| **跨网费用** | 骑手在他网换电产生**跨网设备服务费**（平台统价），由**渠道**保证金/信用额度支付 |
| **清分** | 经平台代收代付；渠道「跨网往来账」只见平台方向，不见对手方运营商 |
| **关闭** | 保证金用尽且信用额度不足 → 关闭渠道跨网换电（同运营商规则） |

### 4.3.2 电池持有（渠道可见）

| 维度 | 规则 |
|------|------|
| **数据来源** | 换电成功后 IoT/订单同步：当前**用户 ↔ 电池 SN** 绑定关系 |
| **展示** | 渠道商「**电池持有**」模块：白名单用户、手机号、当前持有电池 SN、SOC、取电时间、所在站点 |
| **未持有** | 显示「未持有」；仍可在白名单内换电取电 |
| **权限** | 渠道**只读**；不可手工改绑（以换电记录为准） |
| **运营商** | 可在渠道权益/订单侧查看同一关系，**不替代**渠道侧模块 |

### 4.4 计费与平台 1%

| 时点 | 规则 |
|------|------|
| **月租支付** | 渠道 → U 支付当月设备月租；到账后账期生效 |
| **换电** | **白名单免费**：免购套餐可换电；**白名单付费**：持有效白名单套餐可换电；**不扣人天池** |
| **平台 1%** | **月租 MO 支付成功** × 1%（B2B）；**白名单购套餐支付成功** × 1%（C 端，从**渠道子商户**分账） |
| **跨网费** | 渠道开通跨网后：由**渠道**承担跨网设备服务费（平台统价日清）；未开通则仅可在签约运营商站点换电 |

### 4.5 后台菜单

**运营商**：渠道管理（签约 · MO 月租单 · 租赁设备 · 专属站点）

**渠道商**：总览 / 结算说明 / **白名单套餐** / **收款账户** / **月租账单** / **租赁设备** / **电池持有** / **白名单用户** / **跨网往来账** / 变更记录 / 渠道信用 / 员工

> **废弃**（相对旧「设备租金池」）：租金池余额、预缴充值、到期兜底 SKU（`fallback_sku`）、C-07 到期策略。

## 5. 模式四：激活码 · **二期**

> **交付**：整模式二期（原型标「二期」）。  
> **2026-07-02 新增**：第四种渠道结算模式。用户通过**激活码**在运营商小程序开通套餐；**不经平台收款**；平台 1% 计提规则**参考人天池 B 端**。

### 5.1 商业模式

渠道向签约运营商**批发激活码**（B2B 服务订单 AC-，运营商手动确认到账）。渠道管理码库存并发放给骑手；骑手在小程序输入激活码开通对应套餐权益。

| 维度 | 规则 |
|------|------|
| **一码一用** | 每个激活码全局唯一，核销成功后不可再次使用 |
| **用户付款** | **不经平台/运营商在线收款**；渠道与骑手线下或自有 App 结算（平台不参与） |
| **权益** | 核销成功即开通套餐（如 30 天包月），换电规则同个人套餐 |
| **userOwner** | 签约运营商 U |

### 5.2 运营商设置

| 能力 | 规则 |
|------|------|
| **签约** | 结算模式 = 激活码；约定单码批发价、默认套餐码、服务人天（计提用）、最低起购码数 |
| **服务订单** | 渠道提交 AC- 批发单；运营商**确认到账**后码库存入库 |
| **渠道权益** | 查看库存、核销统计；不维护额度池/推广链接 |

### 5.3 渠道商后台

| 模块 | 说明 |
|------|------|
| **激活码** | 码库存：未发放 / 已发放 / 已核销 / 已作废；支持批量导入 |
| **核销记录** | 核销时间、用户、开通套餐、平台计提基数与 1% 金额（只读） |
| **渠道信用** | 与人天池同类，适用渠道信用评估 |

### 5.4 计费与平台 1%

| 时点 | 规则 |
|------|------|
| **批发采购** | 渠道 → U 支付码批发款；到账后入库；**无**平台 1%（B2B 采购款已是运营商收入） |
| **码核销** | 骑手输入有效码 → 开通套餐；**平台 1% 触发** |
| **计提公式** | **平台标准人天价 × 码对应服务人天 × 运营商 B 端费率**（与人天池「确认消耗」同类费率；**与批发单价无关**） |
| **代扣** | 向运营商 U 计提；优先划扣保证金，保证金为 0 才占用信用额度 |
| **跨网费** | 同其他用户，U 承担日清跨网设备服务费 |

**示例**（标准人天价 ¥8.5，B 端费率 1%，30 天码）：

```
计提基数 = 8.5 × 30 = ¥255
平台 1% = 255 × 1% = ¥2.55
```

### 5.5 与人天池对照

| 维度 | 人天池 | 激活码 |
|------|--------|--------|
| 渠道采购 | 人天额度 PO | 激活码 AC |
| 用户获权益 | 登记 + 分配人天 | 输入激活码 |
| 用户向平台付款 | 池不足可兜底 SKU | **无** |
| 平台 1% 触发 | **确认消耗**（按日） | **码核销成功**（按码一次性，基数=标准价×服务人天） |
| 费率 | 运营商 B 端费率 | 同左 |

## 7. 平台营销（架构 C · PLATFORM）

> **2026-07-06 新增** · 专文：[平台营销-PRD.md](./平台营销-PRD.md)

平台作为统一获客方向 **opt-in** 运营商开展营销；与 §3 渠道分销**并行**，**不**新增第五种渠道结算模式。

### 7.1 与渠道分销差异

| 维度 | §3 渠道分销 | §7 平台营销 |
|------|-------------|-------------|
| 推广方 | 第三方渠道商 | **平台** |
| 收款 | 运营商子商户（架构 B） | **平台子商户（架构 C）** |
| 链接 | 带 `op=` | **不带 `op=`** |
| `userOwner` | 支付成功 | **首服取电** |
| 活动价 | 渠道专享价 | **Campaign 统一定价** |
| 平台收入 | 1% | 1% + **营销服务费** |

### 7.2 计费与收银

| 时点 | 规则 |
|------|------|
| **用户购卡** | 经平台链接以 **Campaign 活动价** 支付；款进 **平台子商户**；支付成功计提 **1% 技术服务费** |
| **归因** | `channel_id=PLATFORM`；24h 窗口；**最后触达赢**（与骑士卡同级） |
| **激活** | 支付后 `userOwner=null`；**首服取电**以柜机 `device_owner_id` 锁定运营商 |
| **拨付** | 激活后平台向运营商拨付：`实付 − 营销服务费`（按运营商签约协议） |
| **退款** | 激活前：**平台子商户**原路退；平台 1% 不退 |

### 7.3 运营商参与

- **自愿签约**（opt-in）；签约确认营销服务费/单。
- 未参与运营商不可承接平台营销订单激活。
