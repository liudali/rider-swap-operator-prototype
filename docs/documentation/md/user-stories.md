# 用户故事 · 五角色后台 + 骑手端

> 与 [PRD.md](./PRD.md)、[acceptance-criteria.md](./acceptance-criteria.md)、[骑手端PRD.md](./骑手端PRD.md) 配套。  
> 格式：`作为 <角色>，我希望 <能力>，以便 <价值>`。  
> 状态：**已实现** = 原型可演示；**拍板待落地** = 口径已定、原型/详设待补。

## 1. 平台管理员（经营主体）

- 作为平台管理员，我希望在**总览**查看全平台 KPI（运营商、渠道、用户、站点、设备、订单、流水、平台营收），以便掌握业务大盘。
- 作为平台管理员，我希望在**总览**查看各运营商保证金、信用额度与跨网状态汇总，以便识别清分风险运营商。
- 作为平台管理员，我希望在**用户管理 → 用户信息**看到**电池押金方式**与**押金状态**分列，以及统一的持有电池（编码-SOC-SOH），以便客服与风控溯源。
- 作为平台管理员，我希望在**订单管理**查询全平台套餐单、换电单与渠道批发单，并看到换电三元组与跨网设备服务费，以便纠纷核查。（渠道批发单**仅只读**，不参与确认到账）
- 作为平台管理员，我希望在**设备管理**用弹窗批量导入设备（先选运营商，再填 SN 或上传 SN 表；类型与参数由 IoT 自动获取），以便设备直接归属运营商并可分配站点。
- 作为平台管理员，我希望**不能**在平台侧新增/编辑渠道商主体，仅只读监管渠道额度与人天，以便权责边界清晰（渠道由运营商创建）。
- 作为平台管理员，我希望在**流水管理**查看用户支付、运营商间跨网日清与平台 1% 提成明细，以便对账与审计。
- 作为平台管理员，我希望在**平台账户**查看平台商户号与本月 C 端分账 + B 端计提构成，以便财务汇总。
- 作为平台管理员，我希望在**运营商管理**新增/维护运营商主体、查看清分账户与经营摘要，以便入网治理。
- 作为平台管理员，我希望在**保证金管理**核对运营商对公充值申请并确认入账或驳回，以便恢复保证金优先扣款路径。（**已实现**）
- 作为平台管理员，我希望在**保证金管理**查看全平台保证金账本并手工调额/调信用额度（演示），以便异常补录与额度治理。（**已实现**）
- 作为平台管理员，我希望在**跨网统一定价**发布柜机/电池服务费与人天标准日值，以便全网规则统一、运营商只读遵循。
- 作为平台管理员，我希望审核资方提交的租赁协议（校验承租方、设备 SN 与台账一致），通过或退回修改，以便租赁设备绑定前条款合规。（**拍板待落地**）

## 2. 运营商（经营主体）

- 作为运营商，我希望只管理**经平台绑定**的换电柜与电池（含租赁设备），以便权责清晰。
- 作为运营商，我希望只看到**本主体设备产生**的套餐/换电订单与流水，以便对账准确。
- 作为运营商，我希望管理自有站点及**换电范围**（跨网/跨站开关），以便控制用户换电地理边界。
- 作为运营商，我希望维护**城市底价 + 价格分区**，将站点挂到分区并按任意 SKU 覆盖价格（可高/低于底价），未配时继承城市底价，以便偏远站补贴或差异化定价。
- 作为运营商，我希望在**渠道销售**新增/编辑签约渠道商、维护**一运营商一池**的卖方额度池，以便 B2B 人天销售。
- 作为运营商，我希望在**渠道销售**对线下「待确认到账」批发订单**确认到账**后额度池入账，以便采购款与额度对齐。
- 作为运营商，我希望在**平台服务 → 运营商往来**同页切换概况、服务费明细、日清、周月，并在 **Tab 下方**看到对应说明与筛选，以便连续完成对账。
- 作为运营商，我希望在**平台服务费**分别查看 C 端支付分账与 B 端确认消耗/激活码计提，并按类型+时间筛选，以便区分两类成本。
- 作为运营商，我希望在**保证金账户**查看清分专户、提交对公充值申请并跟踪确认状态，以便补足跨网清分保证金。（**已实现**）
- 作为运营商，我希望在**员工**中维护运营员工权限，以便分工协作。
- 作为运营商，我希望在**月租金/融资相关账单**查看各期应还；通过**微信/支付宝扫码**或**对公打款提交工单**整单缴纳（**不做自动代扣**），以便履行租赁/融资还款义务。
- 作为运营商，我希望在**站点管理**为各站配置场地合伙人及分润比例（一站可多位、同人多站不同比例），以便场地方按约定分享经营收入。（**已实现**）
- 作为运营商，我希望在**站点合伙人**维护个人/公司合伙人档案，并查看分润绑定一览与分润明细，以便对账与审计。（**已实现**）
- 作为运营商，我希望合伙人建档时锁定个人/公司类型，完成对应开户后才能绑站/提现，以便收款主体合规。（**已实现**）
- 作为运营商，我希望合伙人添加、调比例、解绑**立即生效并留痕**，历史分润不回溯，以便配置结果即时可见且可审计。（**已实现**）
- 作为运营商，我希望在**站点管理 → 场费电费**维护站点费用，并在总览按月/站点查看场费分摊与当月电费，以便掌握站点成本。（**已实现**）
- 作为运营商，我希望收款子商户绑定对公结算账户，未绑定时不可提现，以便平台审核后打款目标明确。（**已实现**）

## 3. 渠道商（经营主体）

- 作为渠道商，我希望向**签约运营商**采购人天额度（一运营商一池），增购不新建第二池，以便额度账本连续。
- 作为渠道商，我希望创建骑手团队、登记骑手并分配/收回人天额度，离职强制回池，以便按骑手控制额度。
- ~~作为渠道商，我希望在额度使用规则为团队配置周期上限~~（**已取消** · decision-062）
- 作为渠道商，我希望查看消耗明细、预占/确认状态，并在余额不足时看到可点击的顶栏预警，以便及时进入额度池续购。
- 作为激活码渠道商（二期），我希望申请 AC 批发单，待运营商确认到账后自动按数量生成批次内码；可标记发放、申请作废，以便形成完整码生命周期。
- 作为渠道商，我希望查看运营商个人套餐零售价（只读），以便了解个人用户定价；**渠道骑手零额度不可自费换电**（decision-054）。
- 作为渠道商，我希望**没有**收款账户菜单，向运营商付款走 B2B 采购，以便与商业模式一致。

## 4. 资金方（经营主体）

- 作为资金方，我希望在**设备清单**独立维护设备组合（新建/CSV 导入），并在清单内**替换故障设备**（如电池 A 换 B），以便与租赁协议解耦管理。
- 作为资金方，我希望创建租赁协议时**关联设备清单**而非逐台勾选，且同一运营商可签**多份协议**对应多份清单。
- 作为资金方，我希望创建租赁/融资相关协议后由**运营商确认**生效，**平台仅展示不参与审核**，以便电子约定与线下合同对齐。（**2026-07-13 确认**）
- 作为资金方，我希望在**租金/还款收缴**按账期跟踪扫码支付与对公工单进度，并**审核运营商对公打款工单**，以便确认到账后核销账单（**不做自动代扣**）。
- 作为资金方，我希望调价、续期、改截止日、提前解约时与**运营商重新确认**（平台只读），以便承租方知情同意。（**2026-07-13 确认**）

## 5. 运营员工（员工登录）

- 作为运营员工，我希望用独立账号登录后台，**只看到被授权的菜单**（如设备、订单、用户），以便最小权限协作。
- 作为运营员工，我希望数据范围限定所属经营主体（`device_owner_id`），不能查看其他主体数据，以便数据隔离。
- 作为财务类运营员工，我希望在授权下查看流水、往来账、平台服务费与**保证金账户**，以便协助对公充值与对账。（**已实现** `deposit.view`）

---

## 6. 团队管理员（渠道员工 · `team_admin`）

- 作为团队管理员，我希望用独立账号登录渠道商后台，**仅看到本团队消耗明细** Tab，以便管理本站点/本车队骑手换电情况。
- 作为团队管理员，我希望导出本团队消耗明细 CSV，以便与渠道运营对账。
- 作为团队管理员，我希望**不能**登记骑手、分配额度或修改额度规则，以免越权操作全渠道池。

---

## 7. 骑手（用户端 · 微信小程序）

- 作为骑手，我希望在**首页**查看附近换电站并一键**扫码换电**，以便快速取电。
- 作为个人套餐用户，我希望在**服务购买**选择方案 B SKU（7天/30天畅换/季卡/次卡）并完成支付 Mock，以便开通换电权益。
- 作为个人套餐用户，我希望服务中查看剩余天数/次数，冻结时先还电再申请冻结，以便符合服务规则。
- 作为渠道成员，我希望在**我的**看到渠道名、今日人天资格（预占/已确认），以便知晓能否换电。
- 作为渠道成员，我希望在额度预占失败时看到**失败原因**，并明确**不可换电**（持电池仅可还电、须渠道续配），以便联系渠道补额度。
- 作为骑手，我希望在运营商关闭跨网/跨站或信用额度停跨网时被明确拦截并提示原因，以便理解无法换电的原因。
- 作为骑手，我希望跨运营商换电成功后，后台换电单与我的订单记录一致（含案例3 三元组与跨网服务费），以便纠纷可追溯。
- 作为骑手，我希望在**订单**查看换电、服务购买与退租/押金记录及空态，以便自助查询历史。
- 作为骑手，我希望完成实名认证、电池退还预约与客服拨号，以便合规使用与结束服务。

## 8. 站点合伙人（独立登录）

- 作为站点合伙人，我希望用独立账号登录后台，**只读**查看运营商为我配置的各站当前生效比例，以便知悉合作条款。（**已实现**）
- 作为站点合伙人，我希望查看本人分润明细与累计可提现余额，并向运营商发起提现申请，以便收取场地分润。（**已实现**）
