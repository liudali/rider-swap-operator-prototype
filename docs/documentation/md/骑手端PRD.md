# 骑手换电 · 用户端（移动端）PRD

> 微信小程序形态 · 与 [PRD.md](./PRD.md)（后台）、[换电场景与运营商结算.md](./换电场景与运营商结算.md) 配套  
> 可交互原型：`prototype/mobile/index.html`

## 1. 定位

面向**外卖骑手**的换电服务入口：找站、扫码换电、购买套餐、查看订单与账户。权益来源分两类，与后台 `entitlementType` 一致：

| 类型 | 称谓 | 权益来源 | 购买入口 |
|------|------|----------|----------|
| **用户①** | 个人套餐用户 | 向运营商购买包月/次卡 | 「服务购买」「购买套餐」 |
| **用户②** | 渠道成员 | 渠道商分配人天额度 | 无自购；展示「渠道额度」状态 |

## 2. 信息架构

底部三 Tab（参考竞品）：

| Tab | 页面 | 核心能力 |
|-----|------|----------|
| **首页** | 地图 + 扫一扫 | 站点地图、定位、扫码换电主入口 |
| **订单** | 订单列表 | 换电订单、服务购买订单、退租/押金订单 |
| **我的** | 个人中心 | 登录、服务状态卡、常用功能、设置 |

## 3. 页面说明

### 3.1 首页

| 元素 | 说明 |
|------|------|
| 顶栏 | 应用名「智格换电」；小程序胶囊占位 |
| 客服条 | 客服电话、服务时间（只读） |
| 地图区 | 腾讯地图风格底图（原型静态）；站点 Marker；当前定位 |
| 扫一扫 | 主 FAB；触发换电流程 |
| 辅助按钮 | 帮助、重新定位 |

**扫码换电流程**（弹层/全屏）：

1. 模拟扫描柜机二维码  
2. 校验：是否登录、是否有有效权益（个人套餐服务中 / 渠道人天有余量）  
3. 校验：**渠道骑手可换电校验** — 渠道成员预占失败时拦截渠道扣池，展示 `fail_reason`，引导 1天/单次自费兜底（见 [渠道骑手可换电校验.md](./渠道骑手可换电校验.md)）  
4. 校验：**跨网换电** — 个人与渠道成员共用规则：用户运营商或柜机运营商关闭跨网时双向拦截；信用额度用尽时禁止出站跨网  
5. 成功：展示换电结果（柜机、格口、电池 SOC）；写入换电订单 Mock  
6. 失败：超时/无空电/预占失败等错误态；预占失败可转自费  

**同运营商**：任意站点可购套餐、可换电；**不设用户绑定站点**。

### 3.1.1 运营商归属 · 路径 A（方案乙）

冷启动无链接/扫柜时，点「服务购买」前解析 `context_operator_id`：

| 步骤 | 行为 |
|------|------|
| 1 | 若已有生效个人套餐 → 续费原 OP，不弹选择 |
| 2 | 若有启动 `op=` 或本会话扫柜 → 用该 OP |
| 3 | 否则请求定位 → 15km 内最近在营公开站反推 OP |
| 4 | 唯一且 ≤5km → 自动写入会话；多 OP 接近 → **选择服务运营商** sheet |
| 5 | 无站/拒定位 → 阻断购套餐，引导扫柜或开启定位 |
| 6 | 支付单绑该 OP 子商户；成功后 `userOwner = seller_operator_id` |

「我的」页展示服务条：「当前服务：{运营商名} · 距您最近 {站点} {距离}」。首页 ⌖ 定位按钮触发 geo 解析（不重复展示服务条）。详见 [用户-运营商归属模型.md](./用户-运营商归属模型.md)。

### 3.2 订单

| 子 Tab | 字段 | 空态 |
|--------|------|------|
| 换电订单 | 单号、站点、时间、状态、权益类型 | 「暂无换电记录」 |
| 服务购买 | 套餐单号、套餐名、金额、支付状态 | 「暂无购买记录」 |
| 退租订单 | 退租单号、押金、状态 | 「暂无退租记录」 |

### 3.3 我的

#### 未登录

| 元素 | 说明 |
|------|------|
| Logo + 品牌 | 智格换电 / 安全出行 放心骑行 |
| 一键登录 | 微信授权登录（Mock） |
| 协议勾选 | 隐私政策、数字证书、换电服务协议（必选） |

#### 已登录

| 元素 | 说明 |
|------|------|
| 手机号 | 脱敏展示；实名状态标签（未认证/已认证） |
| 设置 | 齿轮入口 |
| **服务状态卡** | 见下表 |
| 常用功能 | 6 宫格：扫码换电、购买套餐、优惠券、客服、电池退还、实名认证 |
| 我的订单 | 退租订单、服务购买订单（快捷入口） |

**服务状态卡**（按权益类型）：

| 状态 | 展示 | 主按钮 |
|------|------|--------|
| 未开通 | 暂未开通服务 | 服务购买 |
| 个人套餐·服务中 | 30天畅换剩余 X 天 / 次卡剩 N 次；已换电 Y 次 | 续费 / 查看套餐 |
| 个人套餐·已冻结 | 服务已冻结 | **解除冻结**（满足条件即生效） |
| 个人套餐·待领电池 | 解冻后待领取电池 | **扫码领取** |
| 渠道人天 | 渠道：顺丰同城；今日人天已确认；可跨网换电 | 联系渠道 |
| 渠道人天·不可用 | 预占失败原因；引导 1天畅换/单次自费 | 自费换电 |

## 4. 弹窗 / 子页

| 模块 | 字段 / 交互 | 错误态 |
|------|-------------|--------|
| 服务购买 | 方案 B SKU + **新人套餐**（首购展示，站点新人 &gt; 通用新人 &gt; 常规价）；支付 Mock | 非首购不展示新人价；未勾选协议不可购 |

| 设置 | 消息通知、注销、切换演示身份 | — |
| 客服电话 | 拨号确认 | — |

## 5. 与后台对齐

| 用户端 | 后台 |
|--------|------|
| `entitlementType=个人套餐` | 运营商订单 · 套餐单 `SUB*` |
| `entitlementType=渠道人天` | 渠道额度池消耗；`poolId` |
| 换电单 | 运营商「订单与服务 → 换电订单」三元组 |
| 换电拦截 | 运营商关跨网/跨站、信用额度停跨网 |
