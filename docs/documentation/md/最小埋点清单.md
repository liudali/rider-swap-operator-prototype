# 最小埋点清单（一期）

> **依据**：2026-07-13 确认清单第 35 项（会前单独文档）  
> **用途**：技术评审与研发埋点对照；非完整数据仓库方案。

## 1. 事件表

| 事件 ID | 触发时机 | 关键属性 | 角色 |
|---------|----------|----------|------|
| `pay_success` | C 端/B 端支付成功 | order_id, amount, pay_channel, operator_id, sku_id | 骑手/渠道 |
| `clearing_realtime` | 实时清分入账 | order_id, platform_fee, operator_net | 系统 |
| `clearing_custody_hold` | 监管冻结入账（一期） | order_id, hold_amount, release_plan | 系统 |
| `day_pool_reserve` | 00:00 预占 | pool_id, rider_count, success/fail | 系统 |
| `day_pool_confirm` | 确认消耗 | pool_id, user_id, reason=swap\|hold_battery | 系统 |
| `day_pool_release` | 日终释放 | pool_id, user_id | 系统 |
| `day_pool_batch_fail` | 整批预占失败 | pool_id, need, balance | 系统 |
| `day_pool_retry` | 手动/续费自动重试 | pool_id, trigger=manual\|topup | 运营 |
| `platform_fee_accrue` | B 端 1% 计提 | pool_id, day_value, fee | 系统 |
| `cross_net_fee` | 跨网设备费产生 | swap_id, cabinet_fee, battery_fee, city_price_id? | 系统 |
| `withdraw_submit` | 运营商提现申请 | amount, operator_id | 运营商 |
| `withdraw_audit` | 平台审核通过/驳回 | apply_id, result, reason? | 平台 |
| `deposit_warn` | 保证金/人天池低于阈值 | entity_id, threshold, channel=sms\|inapp | 系统 |
| `sms_sent` | 短信发出 | template_id, to_role, biz_ref | 系统 |
| `iot_alert` | IoT Webhook 告警 | device_sn, alert_type, severity | 系统 |
| `iot_remote_cmd` | 远程运维指令 | device_sn, cmd, result | 运营 |
| `lease_penalty_calc` | 违约金计算 | contract_id, penalty_amount, rule_id | 系统 |
| `newcomer_used` | 新人价资格消耗 | user_id, operator_id, order_id, channel=direct | 系统 |

## 2. 成功指标（产品）

| 指标 | 定义 |
|------|------|
| 支付成功率 | pay_success / pay_attempt |
| 人天预占成功率 | reserve success / attempts |
| 提现审核时效 | submit → audit 中位时长 |
| 跨网日清完成率 | 日清批次成功天数 / 日历天 |
| 告警触达率 | sms_sent(success) / deposit_warn |

## 3. 非目标

- 不作全量行为分析埋点（二期）
- 骑手端平台营销转化漏斗（骑手营销二期）
