## 1. 背景与定位

平台作为统一获客方，向全网 **opt-in** 运营商开展营销活动。用户在品牌小程序经平台推广链接购套餐，**款进平台子商户**；**首服取电**时以柜机所属运营商锁定 `userOwner`，平台再向该运营商拨付（扣除营销服务费）。

与第三方「渠道分销」（骑士卡 → 单运营商、运营商子商户收款）**并行**，不新增第五种渠道结算模式。

**已确认决策（2026-07-06）**：

| # | 决策 |
|---|------|
| 1 | **营销服务费**与 **1% 技术服务费**分开计提、分开展示 |
| 2 | 用户款由 **平台收取**；激活后平台向运营商拨付（扣营销服务费） |
| 3 | 链接 **不指定运营商**；**首服取电**锁定运营商 |
| 4 | 运营商 **自愿签约**参与，非全量开通 |
| 5 | 渠道归因 **最后触达赢**（24h 窗口，与骑士卡同级） |
| 6 | 活动级 **统一平台活动价**（Campaign 维度）；运营商签约确认营销服务费/单 |

## 2. 架构对照

| 维度 | 渠道分销 CH-CARD（架构 B） | 平台营销 PLATFORM（架构 C） |
|------|------------------------------|-------------------------------|
| 收款主体 | 运营商子商户 | **平台子商户** |
| 推广链接 | 带 `op=`，落地运营商小程序 | **不带 `op=`**，落地品牌小程序 |
| `userOwner` 时点 | 支付成功 | **首服取电** |
| 运营商参与 | 单渠道签一家 OP | **多 OP opt-in** |
| 活动价 | 渠道专享价（按渠道×SKU） | **Campaign 统一定价** |
| 平台收入 | 1% 技术服务费 | 1% 技术服务费 + **营销服务费** |
| 退款主体 | 运营商子商户 | **平台子商户** |

---

## 3. 端到端流程

```text
平台创建 Campaign → 生成推广链接（无 op）
  → 用户点击/扫码（24h 归因，last touch）
  → 购套餐（平台活动价）→ 支付进平台子商户
  → 订单 activation_status=pending，userOwner=null
  → 用户首服扫码取电 → 校验 OP 已签约且参与 Campaign
  → 锁定 userOwner = cabinet.device_owner_id
  → 平台拨付运营商：实付 − 营销服务费
```

### 3.1 资金公式（单笔）

| 时点 | 动作 | 金额 |
|------|------|------|
| 支付成功 | 平台子商户实收 | `paid_price` |
| 支付成功 | 1% 技术服务费（平台收入） | `paid_price × 1%` |
| 激活结算 | 向激活运营商拨付 | `paid_price − marketing_service_fee` |
| 激活结算 | 营销服务费（平台收入） | 按运营商×SKU 协议 |

平台留存 ≈ `1% + marketing_service_fee`；**1% 不从运营商拨付中二次扣减**。

## 4. 角色与菜单

| 角色 | 菜单 | 能力 |
|------|------|------|
| **平台管理员** | 平台营销 | 活动、运营商签约监管、链接/码、待激活订单、拨付流水、营销对账 |
| **运营商** | 渠道管理 → 平台营销 | 查看/确认参与活动、营销服务费、本 OP 激活成交、对账确认 |
| **骑手** | 品牌小程序 | 平台链接触达、平台活动价支付、待激活取电、激活后换电 |

未签约运营商（演示 `OP-BJ`）：后台无有效 Tab 或显示「未参与」；骑手扫其柜机取电拦截。

## 5. 核心对象

### 5.1 营销活动 Campaign

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 如 `CMP-2607-01` |
| `name` | string | 活动名称 |
| `sku_prices` | array | `{ skuId, skuName, officialPrice, activityPrice }` |
| `start_at` / `end_at` | datetime | 投放周期 |
| `status` | enum | 草稿 / 进行中 / 已结束 / 已停用 |

**约束**：`activityPrice ≤ officialPrice`（正式零售价只读引用运营商定价基准）。

### 5.2 平台营销协议 PlatformMarketingAgreement

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 如 `PMA-SX-01` |
| `operator_id` | string | 运营商 |
| `campaign_id` | string | 关联活动 |
| `sku_id` | string | 授权 SKU |
| `marketing_service_fee` | number | 营销服务费/单（付平台） |
| `status` | enum | 待确认 / 已启用 / 已停用 |

运营商在后台 **确认参与** 后 `status=已启用`；停用后新单不可激活，已 `pending` 订单仍可按退出前协议激活。

### 5.3 营销链接 MarketingLink

| 字段 | 说明 |
|------|------|
| `channel_id` | 固定 `PLATFORM` |
| `campaign_id` | 所属活动 |
| `link_code` | 短码 |
| `link_url` | `wxmp://brand/pages/landing?ch=PLATFORM&campaign=…&lnk=…`（**无 op**） |
| `purpose` | 链接用途 |
| `clicks` / `conversions` | 统计 |

### 5.4 套餐订单扩展（平台营销轨）

| 字段 | 说明 |
|------|------|
| `payment_architecture` | `platform_collect` |
| `channel_id` | `PLATFORM` |
| `campaign_id` | 活动 ID |
| `link_code` | 首触链接 |
| `activation_status` | `pending` / `activated` / `expired` / `refunded` |
| `seller_operator_id` | 支付时为 `null`；激活后 = `activated_operator_id` |
| `activated_operator_id` | 首服取电柜机所属 OP |
| `activated_at` | 激活时间 |
| `activated_cabinet_sn` | 取电柜机 |
| `marketing_service_fee` | 结算时写入 |
| `operator_payout` | 拨付金额 |
| `platform_fee` | 1% 技术服务费 |

### 5.5 激活拨付 Settlement / 月度对账 Statement

- **Settlement**：单笔激活后生成拨付记录。
- **Statement**：自然月汇总某运营商应付拨付；运营商确认（演示）。

## 6. 订单状态机

| `activation_status` | 条件 | `userOwner` |
|---------------------|------|-------------|
| `pending` | 已支付，未取电 | `null` |
| `activated` | 首服取电成功 | 柜机 `device_owner_id` |
| `expired` | 超过激活时限（默认 30 天）未取电 | — |
| `refunded` | 激活前退款成功 | — |

**激活校验**：

1. 柜机所属运营商 `status=在营` 且协议 `已启用`；
2. 运营商参与该 `campaign_id` 且授权该 `sku_id`；
3. 不满足 → 错误码 `E_PLATFORM_OP_NOT_ENROLLED`，文案「请前往参与平台活动的换电站激活」。

**激活时限**：支付后 **30 天**内须完成首服取电；逾期进入退款/客服队列（二期自动退）。

## 7. 归因规则

复用渠道分销 §3.3.1，扩展 `channel_id=PLATFORM`：

| 场景 | 规则 |
|------|------|
| 平台链接触达 | 记录 `attribution_start`，24h 内享 Campaign 活动价 |
| 窗口内骑士卡 → 平台 | **平台赢**，重置 24h |
| 窗口内平台 → 骑士卡 | **骑士卡赢**，走架构 B |
| 过期未支付 | 恢复正式零售价 |
| `link_code` | 首触链接（转化统计） |

---

## 8. 退款

| 阶段 | 规则 |
|------|------|
| 激活前 | **平台子商户**原路退全额；平台 1% 不退（与架构 B 一致） |
| 激活后 | 走服务变更/中途完结；已拨付部分由运营商与平台线下调账（首期人工） |

---

## 9. 空态 / 错误态

| 场景 | 空态/错误 |
|------|-----------|
| 平台无进行中活动 | 活动列表「暂无活动」+ 创建引导 |
| 运营商未签约 | Tab 展示「未参与平台营销」+ 可申请活动列表 |
| 骑手待激活扫非参与 OP 柜机 | `E_PLATFORM_OP_NOT_ENROLLED` |
| 活动已结束仍 pending | 提示联系客服退款 |
| 对账无数据 | 「本月无平台营销激活成交」 |

## 10. 权限

| 操作 | 平台管理员 | 运营商 |
|------|------------|--------|
| 创建/编辑 Campaign | ✓ | — |
| 生成推广链接 | ✓ | — |
| 确认营销协议 | 监管 | ✓（本 OP） |
| 查看待激活订单 | 全平台 | 仅本 OP 激活后 |
| 确认月度对账 | ✓ | ✓（本 OP） |
| 激活前退款审核 | ✓ | —（首期平台处理） |

---

## 11. 分期范围

**本期（原型）**：全链路 Mock、归因冲突一条、激活前退款一条。

**二期**：逾期自动退、地图高亮可激活站、拨付与保证金勾稽。

**三期**：真实支付联调（平台主商户 + 延迟分账至运营商）。
