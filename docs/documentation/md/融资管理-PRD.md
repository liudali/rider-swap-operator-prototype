> 版本 1.1 · 2026-07-02  
> 关联原型：`prototype/index.html` · 运营商菜单 **融资管理** · 资方菜单 **放款申请**  
> 设计参考：两轮换电资产融资租赁管理系统（fuzzy-guide 2026-05-29）

---

## 1. 背景与定位

运营商以换电资产运营并收取服务收入，设备采购需大量资金。运营商向融资租赁方（设备租赁公司）申请**资产融资放款**，并跟踪借据与还款。

**首期定位**：内部融资台账，替代 Excel 与人工追踪；**非**资方审批系统全量替代。资方侧首期提供**放款申请确认**与预还款计划审核（Mock 演示）。

**明确排除（I6-4）**：ERP / 运营系统自动对接、银行流水自动核销、资方在线审批流、提前还款/展期/罚息、发票与总账。

**架构决策（2026-07-02）**：移除原「设备直租」轨道（协议与设备 / 月租金 / 租金收缴）；运营商 ↔ 租赁公司**仅保留本融资台账**一条协作链路。

---

## 2. 系统架构

| 参与方 | 系统入口 | 核心对象 |
|--------|----------|----------|
| **运营商** | **融资管理** | 授信项目、资产池、放款批次、预/正式还款计划、借据 |
| **设备租赁公司（资方）** | **放款申请** | 批次审核、预还款计划确认、借据录入（演示） |
| **平台管理员** | 租赁公司绑定 | 建立「租赁公司 ↔ 运营商」关系；首期不单独建设融资审批台 |

| 角色 | 菜单 | 能力 |
|------|------|------|

## 4. 核心对象与状态

### 4.1 授信项目（Finance Project）

- 挂在资方下，字段：项目名称、授信总额、是否循环额度、单台参考融资额、基础放款条件、项目状态。
- 额度口径：**已占用**（已放款）、**拟占用**（已提交/资方已确认未放款）、**可用** = 总额 − 已占用 − 拟占用。
- 循环额度：还款确认后可释放；非循环：还款不释放。

### 4.2 可融资资产（Asset）

| 状态 | 说明 |
|------|------|
| 未入池 | 未进入可融资池 |
| 可融资 | 人工确认，可加入申请批次 |
| 申请锁定 | 已加入已提交资方的申请 |
| 已融资 | 已随某笔放款绑定借据 |
| 已替换 | 曾为标的物，已被替换 |
| 异常 | 故障/丢失/资方不认可 |

首期：**Excel/手工录入**入池；不对接运营系统自动判断。可选展示用户数、柜效、近 30 日收入/换电（预留自动化）。

### 4.3 放款申请批次（Drawdown Application）

组织维度：资方 + 融资项目 + **申请月份** + **批次号**（同月可多批）。

| 状态 | 运营商 | 资方 | 资产 |
|------|--------|------|------|
| 草稿 | 选资产、填金额、维护预还款计划 | — | 可增删 |
| 已提交资方 | 只读 | 待确认 | 申请锁定 |
| 资方已确认 | 待录入借据 | 已确认 | 锁定 |
| 已放款 | 持续还款 | — | 已融资 |
| 已归档 | 只读 | — | — |

### 4.4 预还款计划 / 正式还款计划

- **预还款计划**：挂在申请批次下，借据生成前；状态：草稿 → 待确认 → 已确认 → 已固化。
- **正式还款计划**：放款后由确认版预计划固化，绑定借据；进入还款日历。

### 4.5 借据（Loan Note）

放款完成后录入：借据编号、实际放款金额、放款日、起租日、期限、合同编号等。首期默认**一批次一借据**。

## 5. 页面结构（运营商 · 融资管理）

### 5.1 工作台

- KPI：本月待还、逾期、待提交申请、待资方确认、待录入借据、授信占用、可融资资产数。
- 待办列表：跳转台账 / 还款日历。

### 5.2 融资台账

- 列表：批次号、申请月、项目、设备数、申请金额、状态、提交时间。
- 详情：资产清单、区域汇总、申请/确认金额、预还款计划、借据、正式还款计划。
- 操作：草稿 → **提交资方**；资方已确认 → **录入借据 · 放款完成**。

### 5.3 授信项目

- 资方、项目、授信总额、已占用、拟占用、可用、循环/非循环、单台参考额。

### 5.4 资产池

- 筛选：融资状态；列：SN、类型、区域/城市/站点、用户数、柜效、关联批次。

### 5.5 还款日历

- 按应还日列表：应还、实还、未还、状态（含逾期）；**登记还款**（演示）。

---

## 6. 页面结构（资方 · 放款申请）

- 列表：批次号、运营商、申请月、设备数、申请金额、状态、提交时间。
- 详情：预还款计划期次表。
- 操作：**已提交资方** → 确认申请与预还款计划（录入确认金额）；**资方已确认** → 录入借据（演示，可与运营商侧重复演示）。

---

## 7. 关键流程

```mermaid
flowchart LR
  A[资产入池] --> B[创建放款申请草稿]
  B --> C[维护预还款计划]
  C --> D[提交资方]
  D --> E[资方确认]
  E --> F[录入借据]
  F --> G[正式还款计划]
  G --> H[还款日历 / 登记还款]
```
