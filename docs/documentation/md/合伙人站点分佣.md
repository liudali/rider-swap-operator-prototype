# 合伙人站点分佣规则

> 运营商在**站点维度**绑定合伙人并设置分润比例；**同一合伙人可在不同站点获得不同比例**。渠道人天用户**不适用**本文。

关联：[站点合伙人.md](./站点合伙人.md) · [合作模式与分账.md](./合作模式与分账.md) §9

## 1. 前置条件

| 步骤 | 操作方 | 说明 |
|------|--------|------|
| 建站 | 运营商 | 维护站点信息 |
| 绑设备 | 运营商 | 站点绑定柜机、电池（自有或租赁） |
| 加合伙人 | 运营商 | **站点合伙人 → 合伙人档案**：类型（个人/公司）、名称、联系人、收款账户 |
| 站点绑定 | 运营商 | **站点管理 → 合伙人**：每站可添加**多位**合伙人，各设本站 R% |

## 2. 适用范围

- **计入**：个人用户（C 端自购）套餐确认收入、关联换电确认收入。
- **不计入**：渠道商人天额度用户、跨运营商往来 跨网设备服务费、平台 B 端 1% 服务费台账。

## 3. 分佣比例

### 3.1 无合伙人

| 方 | 比例 |
|----|------|
| 平台 | 1% |
| 运营商 | 99% |

### 3.2 有合伙人（按站点比例）

运营商在**站点管理**为每个站点配置 **1～N 位**合伙人，各设 **R_i%**（0 < R ≤ 99，**Σ R_i ≤ 99**）。同一合伙人绑定多站时，**各站 R% 独立**：

| 方 | 比例 |
|----|------|
| 平台 | 1%（固定） |
| 合伙人 i | **R_i%**（该站当日生效值） |
| 运营商 | 99% − Σ R_i% |

**示例**：浦东站 王场地方 25% + 李物业 5% → **平台 1% + 合伙人合计 30% + 运营商 69%**；世博站仅王场地方 20% → **1% + 20% + 79%**。

合伙人比例从运营商原 99% 份额中切出，不改变平台 1%。

## 4. 计提基数

字段建议：`share_base` / `allocatable`

- 来源：该站点、该运营商 `device_owner_id` 设备上，个人用户订单**已确认**的可分配收入。
- 包月：按日摊销或按次摊销后的当日/当笔确认额。
- 换电：关联个人套餐的换电确认片段。
- 退款：按原 SplitLine 冲正。

## 5. 金额公式（单笔 · 多合伙人）

计提基数为 `B`；本站有 N 位合伙人，比例分别为 `R_1…R_N`（无合伙人则 N=0）：

```
platform_amount     = round(B × 0.01, 3)
partner_i_amount    = round(B × R_i / 100, 3)
operator_amount     = B − platform_amount − Σ partner_i_amount
```

### 尾差规则

1. **先算平台**，四舍五入至小数点后 **3** 位。
2. **再按合伙人逐位计算**，各四舍五入至 **3** 位。
3. **运营商** = 基数 − 平台 − 全部合伙人份额，**不再四舍五入**，吸收全部尾差。

## 6. 规则生效时间

| 项 | 说明 |
|----|------|
| **生效时点** | 运营商提交添加/调比例/解绑后**立即生效** |
| **适用范围** | 合伙人分佣比例调整、站点绑定/解绑合伙人 |
| **计提口径** | 变更提交后的新确认收入按新规则；提交前已确认收入按旧规则，不追溯 |
| **审计留痕** | 追加绑定变更记录：操作时间、操作人、动作、变更前后比例、备注 |

**示例**：6 月 10 日 16:00 将合伙人比例从 30% 改为 25% → 16:00 前已确认收入不变；16:00 后新确认收入按 25%。

## 7. 字段建议（开发）

| 字段 | 说明 |
|------|------|
| `site_id` | 站点 |
| `partner_id` | 合伙人（可空） |
| `partner_rate_pct` | **该站点**收入确认日已生效的合伙人比例 |
| `effective_at` | 规则实际生效时间（提交时间） |
| `binding_log_id` | 本次添加/调比例/解绑对应的变更记录 |
| `share_base` | 计提基数 B |
| `platform_amount` | 平台份额 |
| `partner_amount` | 合伙人份额 |
| `operator_amount` | 运营商份额 |
| `split_label` | 展示用，如「平台1%+合伙人30%+运营商69%」 |
| `user_type` | `personal` / `channel_day`；仅 personal 走本规则 |

## 8. 验收用例

| # | 场景 | B | R% | 平台 | 合伙人 | 运营商 |
|---|------|---|-----|------|--------|--------|
| 1 | 无合伙人 | 100.000 | 0 | 1.000 | 0.000 | 99.000 |
| 2 | 浦东站合伙人 30% | 245.000 | 30 | 2.450 | 73.500 | 169.050 |
| 2b | 同合伙人·世博站 20% | 245.000 | 20 | 2.450 | 49.000 | 193.550 |
| 3 | 尾差 | 9.960 | 30 | 0.100 | 2.988 | 6.872 |
| 4 | 合伙人 12% | 245.000 | 12 | 2.450 | 29.400 | 213.150 |

**生效时间**

| # | 场景 | 操作 | 预期 |
|---|------|------|------|
| 5 | 比例变更 | 6/10 16:00 将浦东站 30% 改为 25% | 立即生效并写变更记录；历史分润不回溯 |
| 6 | 同人多站 | 王场地方浦东 30%、世博 20% | 各站按本站 R% 独立切分 |
