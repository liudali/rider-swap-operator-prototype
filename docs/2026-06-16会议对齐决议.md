## 2. 渠道商

| 决议 ID | 结论 |
|---------|------|
| D03 | **一期强制单运营商签约**；取消「同时签多家」演示 |
| D05 | 渠道结算三种模式均要有 Mock：**卡差价 / 人天额度池 / 设备租金池** |
| D06 | 人天计费维持：**换电或持电池**均确认消耗 1 人天 |
| D18 | 渠道总览展示**签约运营商 Logo** |
| D19 | 额度**双向增减**维持（渠道增购 + 运营商额度调整） |
| D16 | 新增 **「渠道信用额度管理」**：渠道信用评估抵扣电池押金；信用好→渠道免押；信用差→渠道线下补押；运营商设单台柜/电押金；按渠道用户规模比对，不足提醒运营商 |
| D38 | **C 端**：芝麻/支付分免押；**渠道用户**默认免押，押金算在渠道身上 |

## 3. 租赁与租金

| 决议 ID | 结论 |
|---------|------|
| D04 | **公对公代扣不做**；主路径：**微信/支付宝扫码** + **对公打款工单**（资方/运营商审核） |
| D11 | 设备替换：**仅保修期内**可换；保修期跟随设备（**平台管理员可编辑**） |
| D12 | 授信 + 电子合同：原型加 Mock 入口，标 **「未接入」** |

## 4. 订单与业务记录命名

| 决议 ID | 结论 |
|---------|------|
| D07 | 运营商侧拆为 **4 个独立菜单**（不再用「我的订单」聚合 Tab） |
| D39 | 第三、四类：**服务冻结** / **中途完结**（原「服务变更」拆分） |

| 菜单 | 说明 |
|------|------|
| 套餐购买订单 | 个人用户购买包月/次卡 |
| 换电订单 | 换电行为记录（个人/渠道） |
| 服务冻结 | 冻结/解冻申请 |
| 中途完结 | 提前结束服务与退款 |

## 5. 分润、流水与退款

| 决议 ID | 结论 |
|---------|------|
| D08 | **C 端实时清分（1B）**（非周期完结后结转）；站点合伙人分佣**移出本期** → [站点合伙人-待定.md](./站点合伙人-待定.md) |
| D24 | 平台 **1% 不退**；**用户退款由运营商承担** |
| D30 | 流水≠收入；**支付后实时清分**至运营商，运营商可提现 |
| D36 | 若运营商已提现后遇用户退款 → 订单进入 **「待退款」** 状态；退款金额由运营商管理，**≤ 关联订单已付金额** |
