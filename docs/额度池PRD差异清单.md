**D1 已确认口径（组合方案）**

1. 渠道商 **在线支付增购** → 支付成功自动入账同一池（不新建第二池）。
2. 渠道商 **线下支付增购** → 生成 PO，**运营商确认到账**后入账。
3. 运营商在「签约渠道」**新增渠道商**时，**默认创建空池/待配置池 + 默认团队** — **✓ 已实现**（`createPendingPoolForChannel`）。

**D2 已确认口径**

- **平台**：维护人天标准日值等 **参考价**（只读展示给运营商）。
- **运营商**：自行维护 C 端零售价 / 站点差价；渠道商侧 **只读**展示，不可编辑。
