## 1. 背景与定位

平台作为统一获客方，向 **opt-in** 运营商投放营销活动。优惠以 **立减优惠券** 形式给到用户：购套餐时展示原价并立减，**必须先锁定运营商**；用户实付款进入 **该运营商子商户**（与直购同分账主干）。

**平台不代收用户套餐款，也不再「激活后向运营商拨付用户款」。**

与第三方「渠道分销」并行，不新增第五种渠道结算模式。

### 已确认决策

| # | 决策 | 版本 |
|---|------|------|
| 1 | 营销服务费与 1% 技术服务费分开计提、分开展示 | 仍有效 |
| 2 | ~~用户款由平台收取、激活后拨付~~ → **款进锁定运营商；不代收过桥** | **012 更正** |
| 3 | ~~链接不指定 OP、首服取电锁定~~ → **购买时锁定 OP（链接须带 `op=` 或下单前强制选择）** | **012 更正** |
| 4 | 运营商自愿 opt-in | 仍有效 |
| 5 | 渠道归因最后触达赢（24h） | 仍有效 |
| 6 | 活动级优惠统一配置（现为 **立减券面额**，非改写价签唯一路径） | **012 调整表述** |

## 2. 架构对照

| 维度 | 渠道分销 CH-CARD（架构 B） | 平台营销 PLATFORM（**券+锁 OP · 012**） |
|------|------------------------------|----------------------------------------|
| 优惠形态 | 渠道专享价 | **立减优惠券**（原价 − 券 = 实付） |
| 收款主体 | 运营商子商户 | **锁定运营商**子商户（同 B） |
| 推广链接 | 带 `op=` | **必须带 `op=`**（参与活动的 OP） |
| `userOwner` 时点 | 支付成功 | **支付成功** |
| 运营商参与 | 单渠道签一家 OP | **多 OP opt-in**（每 OP 一条投放链或可选 OP） |
| 平台收入 | 1% 技术服务费 | 1% 技术服务费 + **营销服务费（协议月结）** |
| 用户款过桥 | 无 | **无** |

## 3. 端到端流程

```text
平台创建 Campaign（配置 SKU 原价 + 立减券面额）
  → 与 opt-in 运营商签约（营销服务费/单）
  → 生成推广链接（ch=PLATFORM + campaign + sku + op=已签约OP）
  → 用户点击（24h 归因，last touch）
  → 收银台：原价、立减券、实付；收款方=锁定运营商
  → 支付成功 → userOwner=op；套餐即服务中
  → 1% 按实付进入支付分账；营销服务费记协议应付（不对用户过款截留）
```

### 3.1 资金公式（单笔）

| 时点 | 动作 | 金额 |
|------|------|------|
| — | **交付范围** | **二期**（decision-014）；一期不交付 |
| 展示 | 原价 / 立减 / 实付 | `list_price` / `coupon_amount` / `paid = list − coupon` |
| 支付成功 | 运营商子商户实收 | `paid` |
| 支付成功 | 1% 技术服务费 | `paid × 运营商 C 端费率` |
| 支付成功 | 券核销记账 | `coupon_amount`（**运营商让利**，非平台补贴） |
| 协议结算 | 营销服务费应付平台 | 按 `marketing_service_fee`/单 |

**已确认（决策 013）**：券成本默认由 **锁定运营商承担**（少收面价差）；平台不做营销预算补贴。签约文案须明示。

## 4. 角色与菜单

| 角色 | 菜单 | 能力 |
|------|------|------|
| **平台管理员** | 平台营销（二期） | 活动（券）、运营商签约、链接（含 op）、成交订单、券核销/营销费结算、对账 |
| **运营商** | 渠道管理 → 平台营销（二期） | 确认参与、查看本 OP 成交与月对账（营销费应付） |
| **骑手** | 品牌小程序（二期） | 领券立减、购时见锁定运营商、支付即服务 |

## 5. 核心对象（摘要）

### 5.1 Campaign

`sku_prices[]`：`officialPrice`（原价）、`couponAmount`（立减）、`payPrice = official − coupon`。

### 5.2 链接

`link_url` **必须**含 `op=`；`op` 须为该 Campaign×SKU 下协议已启用的运营商。

### 5.3 订单

| 字段 | 说明 |
|------|------|
| `payment_architecture` | `operator_collect`（运营商收款） |
| `coupon_id` / `coupon_amount` | 核销券 |
| `locked_operator_id` | 支付前锁定，= `userOwner` |
| `paid_price` | 实付 |
| `platform_fee` | 1% |
| `marketing_service_fee` | 应付平台营销费 |
| **无** | `activation_status=pending`、平台拨付用户款 |

---

## 6. 状态机（简化）

支付成功 → **服务中**（`userOwner` 已锁定）。  
退款走运营商子商户原路退 + 券冲正 / 营销费冲正（细则同架构 B + 券规则）。

---

## 7. 归因

24h 最后触达；平台链赢则享受该 Campaign×该 `op` 的立减券；窗口内被骑士卡覆盖则走渠道专享价（架构 B）。

---

## 8. 退款

| 阶段 | 规则 |
|------|------|
| 服务中退款 | **运营商子商户**原路退实付；1% / 营销服务费按决策 008 冲正；**无平台券补贴可冲**（让利已由运营商承担） |
| 不存在 | 「激活前平台退款」路径（已无平台代收） |

---

## 9. 空态 / 错误

| 场景 | 处理 |
|------|------|
| 链接缺 `op` 或 OP 未签约 | 拦截下单：「请选择参与活动的服务商」 |
| 券已核销 / 活动停用 | 恢复原价或提示活动结束 |
| 对账无数据 | 「本月无平台营销成交」 |
