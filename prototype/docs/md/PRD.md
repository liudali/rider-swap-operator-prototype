## 1. 背景

面向**外卖员**换电业务，**物联网平台**与**骑手用户端**由智格平台建设；本后台提供给**平台管理员**、**运营商**、**渠道商**及**资金方**使用。

## 2. 角色定义

| 角色 | 定位 | 主要职责 |
|------|------|----------|
| **平台管理员** | 智格超能平台运营方 | **用户管理**（服务运营商、套餐/人天权益、渠道归属、持有电池）；**订单管理**（套餐/换电/渠道批发）；**设备管理**（全量台账、批量导入、绑定运营商）；**渠道商管理**（全平台只读查询）；**流水管理**（用户支付、运营商间跨网设备服务费、平台提成）；**平台账户**；**运营商管理**；**设备租赁公司管理**与**租赁关系绑定**；**平台统价**（跨网设备费 + **人天标准日值**）；全业务汇总 |
| **运营商** | 换电服务运营主体；**可持有自有或租赁设备** | 向资金方承租设备；运营站点与柜机；C 端个人套餐定价；向渠道商批发人天额度；跨运营商电池往来核算 |
| **渠道商** | B 端换电服务采购方 | 向**一家签约运营商**签约（一期单运营商）；结算模式：**人天池 / 卡差价 / 设备租赁**（见 [渠道结算模式规则.md](./渠道结算模式规则.md)）；登记团队骑手或**白名单用户**；**跨网能力继承运营商换电范围配置**；查看消耗明细 |
| **智格平台** | 技术平台方 | 物联网、用户端、规则引擎、对账；收取 **1% 技术服务费**（C 端支付分账 + 渠道人天确认消耗向运营商计提） |
| **资金方（设备租赁公司）** | 设备资产出租方 | 出租柜机/电池；管理租赁协议与承租方还款；**承租运营商须为平台已绑定主体**；系统支持**多家租赁公司并存**，运营商可与不同出租方分别签约 |

关系示意：

```
智格平台（技术 + 1% 服务费 + 平台治理后台）
  ├── 平台管理员（用户/订单/设备/渠道/流水 · 运营商治理 · 租赁公司绑定 · 跨网统价）
  ├── 设备租赁公司（出租设备 · 须平台绑定运营商）
  │     └── 运营商（承租 · 可向多家租赁公司分别签约 · 换电运营 · 定价 · 渠道批发）
  │           └── 渠道商（采购人天 · 团队骑手 · 分配额度）
  └── C 端骑手（个人套餐 / 渠道成员换电）
```

### 2.1 术语：跨网设备服务费（不再使用代号「L1」）

当骑手在 **A 运营商** 购买套餐或持有额度，却在 **B 运营商的柜机/电池** 上换电时，A 需向设备持有方支付 **跨网设备服务费**（经平台代收代付、日清）：

| 费用项 | 触发条件 | 默认单价 |
|--------|----------|----------|
| **柜机服务费** | 柜机运营商 C ≠ 用户运营商 U | ¥0.5/次 |
| **电池服务费** | 电池运营商 B ≠ 用户运营商 U | ¥0.1/次 |

U、C、B 三方相同时 **不产生** 该项费用。与 **平台 1% 技术服务费**（支付分账/消耗计提）、**套餐应分/人天消耗** 是并行、互不替代的三类账目。详见 [换电场景与运营商结算.md](./换电场景与运营商结算.md)。

### 2.2 术语：人天标准日值

平台统一设置的人天单价（默认 **¥8.5/人天**），具有双重含义：

| 用途 | 说明 |
|------|------|
| **B 端 1% 计提基数** | 渠道人天**确认消耗**时，平台按此价格 × 1% 向额度售卖方 U 计提，**与运营商实际批发价无关** |
| **运营商默认批发价** | 运营商面向渠道商签约/采购时的**初始批发单价**；运营商可在「定价管理」中修改 |

平台在后台维护并向运营商**只读展示**标准日值；运营商不可修改平台计提基数。

## 4.1 支付架构（架构 B · 一期 Mock）

- **C 端收款主体**：运营商进件子商户；**支付成功时实时清分**：1% → 平台、运营商净额**均可提现**（决策 1B；不做按服务周期分批清分，决策 10C）。
- **B 端批发**：渠道商向运营商付款（人天 / 设备月租 MO）；分销模式无 B2B 购卡。平台 1% 触发时点见 [渠道结算模式规则.md](./渠道结算模式规则.md)（人天池=确认消耗；卡差价=购卡支付；设备租赁=月租支付）。
- **监管冻结清分**：暂缓，待财务/法务专项会（决策 9C）；一期按架构 B + Mock 推进。
- **平台**：不参与骑手套餐/换电运营分成；技术服务费见 §5.5。

## 5.2 骑手套餐服务生命周期

| 能力 | 说明 |
|------|------|
| **中途完结** | 未到期主动结束服务；退还**未使用套餐费** + **电池押金**（已付押原路退；须还电） |
| **信用免押** | 满足门槛可用**芝麻信用** / **微信支付分**免押，无押金退款项 |
| **服务冻结** | 骑手中途申请冻结；**申请时须未持有电池**（仍持有则拒绝）；冻结期间不可换电、不持有电池 |
| **解冻续用** | **个人套餐**用户在套餐有效期内且处于冻结状态时，**满足条件即可自行解除冻结**（系统自动生效，无需运营商审核）；解冻后 `valid_to` 按冻结天数顺延；**首次服务为领取电池**（扫码出电，非换电），之后继续消耗原套餐额度 |

后台「套餐购买订单 / 服务冻结 / 中途完结」可查看冻结/解冻记录（只读）；**退款执行**统一在「退款管理」；套餐详情展示押金、冻结、解冻后首服、退款拆分。

**清分（1B）**：C 端支付成功后，平台 1%、运营商净额**实时清分、可提现**；流水≠收入，但资金侧不再经「待结转/可打款」周期。

**待退款（D36 / 5A）**：骑手发起退款 → 按运营商配置**自动原路退**或**人工确认后系统执行**；**出款主体为运营商收款子商户**；若运营商已提现后遇退款 → 标记**须垫付**；退款 ≤ 已付金额；**平台 1% 不退**。

完整定价 SKU、渠道兜底与自动退款规则见 **[个人套餐定价与退款.md](./个人套餐定价与退款.md)**。

### 5.2.1 套餐 SKU 与退款（摘要）

| SKU | 有效期 | 退款门槛 |
|-----|--------|----------|
| 包月 / 7天 | 按自然日服务期 | 剩余 ≥ 1 天、未持电池 → **自动退款** |
| 1天 / 单次 | **购买后 24h** | 未换电可退；已换电或不足 1 单位不可退 |

退款由**运营商子商户**原路退；清分/收入台账冲正；**平台 1% 不退**；详见 [合作模式与分账.md](./合作模式与分账.md) §3.5。

### 5.2.2 退款管理（运营商）

C 端个人用户**中途完结**及符合 §5.2.1 的**未使用退订**，统一进入运营商「退款管理」。

| 配置项 | 说明 |
|--------|------|
| **自动退款** | 开启后，申请满足 SKU 规则且已还电 → **系统自动原路退款**，无需人工点击 |
| **手动确认** | 关闭自动退款时，全部申请进入**待审核**；运营商「确认退款」后**系统从运营商子商户自动原路退**/垫付记账（非线下手工打款） |

| 字段（申请单） | 说明 |
|----------------|------|
| 退款单号 / 关联套餐单 / 服务变更单 | 唯一标识与追溯 |
| 套餐退 / 押金退 / 合计 | 应退拆分；免押用户押金退为 0 |
| 平台费不退 | 原支付平台 1% 不退还 |
| 须垫付 | 运营商已提现后遇退款，系统标记垫付（5A） |
| 处理状态 | 待审核 / 已退款 / 已驳回 |

**权限**：`refunds.view` 查看队列与设置；`refunds.audit` 切换模式、确认/驳回退款。

**与中途完结关系**：「中途完结」侧栏仅展示服务变更申请；退款执行、垫付、冲正均在「退款管理」完成。

### 5.3.2 渠道商权益（按结算模式）

| 模式 | 渠道商权益 |
|------|------------|
| **人天池** | 批发人天、额度池、团队登记与分配、渠道信用抵扣押金 |
| **渠道分销** | 授权 SKU 专享价+佣金、推广链接/二维码、佣金对账（无统管押金） |
| **设备租赁** | 签约统一月租、专属站、白名单维护、电池持有查询 |

### 5.3.3 多渠道分销 · 套餐与定价

同一运营商可签**多个分销渠道**（如骑士卡、闪送骑士卡），**各渠道独立**配置：

| 维度 | 规则 |
|------|------|
| **授权套餐** | 签约时勾选可售 SKU（如 30 天包月、7 天卡） |
| **价格** | 每渠道×SKU 维护**正式零售价**（基准）、**渠道专享价**、**佣金/单** |
| **配置入口** | 「渠道管理 → 签约渠道」编辑；「定价管理 → 渠道分销价」**汇总对比** |
| **渠道侧** | 渠道在「套餐与链接」基于授权 SKU 生成推广链接；**不可自改价** |
| **约束** | 专享价 ≤ 正式零售价；调价对新单生效 |

## 5.4 企业人天额度池（B 端）

**渠道商**向**一家签约运营商**签约，结算模式三选一：**人天池 / 卡差价 / 设备租赁**（D05；完整规则见 [渠道结算模式规则.md](./渠道结算模式规则.md)，支付一期 Mock）。**人天池 / 设备租赁**渠道用户默认免押，押金算在渠道，由 [渠道信用评估方案.md](./渠道信用评估方案.md) 抵扣。**设备租赁**用户须入**白名单**，免 C 端购套餐。**人天池**细则另见 [天数池.md](./天数池.md)。

### 5.4.1 渠道分销（骑士卡 · 推广链接）

| 维度 | 规则 |
|------|------|
| **链接形态** | 渠道在「套餐与链接」生成推广链接；每条链接可**生成二维码**（内容与链接一致） |
| **落地载体** | 链接/二维码**直达签约运营商小程序**（参数含 `op`、`ch`、`lnk`、可选 `sku`） |
| **用户路径** | 点击/扫码 → 打开运营商小程序 → **注册/登录** → **实名认证** → 购套餐 |
| **24h 归因** | 用户**首次点击**有效链接起 **24 小时内**，购买该渠道授权**全部 SKU** 均展示**渠道专享价** |
| **订单标记** | 归因期内成交写入 `channel_id`、首触 `link_code`；支付成功实时清分 1%；佣金线下结 |
| **过期** | 超过 24h 未购 → 恢复运营商**正式零售价** |

### 5.4.2 设备租赁（B 端 · 白名单）

| 维度 | 规则 |
|------|------|
| **运营商** | 签约时绑定专属站点；维护租赁设备 SN 与部署站点；与渠道约定**统一月租**（不按单台定价）；生成 **MO- 月租账单** |
| **渠道商** | **自行维护白名单**（扁平名单，**无团队**）；查看租赁设备、**电池持有**、月租账单 |
| **骑手** | 白名单内：免押、免购套餐换电；**无团队属性**；专属站仅白名单可见 |
| **渠道停服** | 运营商将渠道商主体状态设为**已停用**时：白名单用户**不可取电/换出**；扫码提示「当前站点已到期，请联系管理员续费」；**仍可还电入柜**（`E_LEASE_CHANNEL_SUSPENDED`） |
| **平台 1%** | 月租 B2B 支付成功时计提 |
| **与人天池差异** | 无预占/扣天；池不足时**无** C 端兜底 SKU，仅白名单准入 |

### 核心计费（首版默认）

| 步骤 | 规则 |
|------|------|
| 预占 | 每日 00:00 对有效配置内骑手预占 1 人天 |
| 确认 | 当日**有换电**或**持有电池** → 确认消耗 1 人天（每骑手每日最多 1 条消耗记录） |
| 释放 | 日终**无换电且未持有电池** → 释放预占；**同日多次换电仍只扣 1 人天** |
| 同步 | 每次换电成功**实时同步**至渠道商；消耗记录含当日换电次数与持有电池数 |
| 不足 | 不允许透支；余额不够覆盖全部用户 → **整批预占失败**；骑手可**在线支付**（默认 **1 天/单次** 兜底 SKU） |

### 渠道兜底零售（已确认）

渠道成员**无预占人天**时走 C 端在线支付；平台 **1%** 在支付成功时收取。

运营商在「定价管理」中**必须**为上架渠道服务保留 **1 天 + 单次** 各至少一款生效零售价（视为对人天额度池的兜底）；详见 [个人套餐定价与退款.md](./个人套餐定价与退款.md) §3.3。

### B2B 采购款与消耗结算（已拍板）

| 时点 | 资金流向 | 说明 |
|------|----------|------|
| **渠道采购** | 渠道商 → 运营商 | 在线支付 T+0/T+1 入账；线下对公由运营商确认到账后入账；**批发款在采购时已结清** |
| **骑手确认消耗** | 池余额 −1 人天 | **不向运营商二次打款**（收入已在采购时收到） |
| **骑手确认消耗** | 运营商 → 平台 | 按**平台标准人天价 × 1%** 计提技术服务费（代扣保证金/信用额度） |

> C 端个人套餐：**支付成功实时清分**（1B），不再使用「待结转 → 可打款」周期。**不适用于**渠道人天池消耗（B2B 采购时结清）。

### 合同级可配置（首版三项 + 一项第二期）

| 配置项 | 可选值 | 首版 |
|--------|--------|------|
| 扣天口径 `deduct_mode` | 自然日 / 有换电才扣天 | 固定自然日 |
| 计费批处理 `billing_mode` | 预占确认（首版写死）/ 后续扩展 | 固定预占确认 |
| 激活时机 `activation_mode` | 分配即开通 / 首换开通 | 分配即开通 |
| 池过期退款 `pool_expiry_refund_policy` | 不退 / 仅整池未分配可退 / 允许部分过期可退 | 允许部分过期可退 |
| 结算节奏 `settlement_mode` | 跟骑手个人 / 企业按月 | **非首版**（第二期；渠道池不按消耗向运营商打款） |

### 配置与权限

| 角色 | 能力 |
|------|------|
| 运营商 | 设定个人套餐零售价与人天批发价；向渠道商销售额度池；维护签约渠道（主体/批发价/合同，**不含可用站点**）；对已售池**额度调整**（充值/赠送/退款/修正/过期恢复） |
| 渠道商 | 向运营商购买人天额度；**创建团队并绑定消耗池**；登记骑手；分配/收回骑手额度；查看消耗与明细；**不可在线退池** |

## 5.6 跨运营商换电与保证金/信用额度清分

- **两类用户**：① 个人套餐（userOwner = 套餐售卖运营商）；② 渠道成员（userOwner = 人天额度售卖运营商）。
- **三元组**：每笔换电记录 userOwner / cabinetOwner / batteryOwner；允许电池跨柜、用户跨运营商柜机换电。
- **运营商间费用**（柜机 **¥0.5/次**、电池 **¥0.1/次**，暂定平台统价）：U→B 电池费；双方仅见**平台代收/代付**；**每日 23:59:59 日清**，周/月由日账单汇总。
- **保证金优先**：跨网设备服务费与 B 端平台费日清**平时划扣平台保证金**；**仅当保证金为 0** 才启用信用额度（允许欠费）。
- **停服**：信用额度也用光 → 关闭该运营商所属用户（个人+渠道）**跨网换电**。
- **运营商自配**：可关闭**跨网换电**（双向封闭）；**不设用户绑定站点**，同运营商任意站点可购可换；见 [换电范围策略.md](./换电范围策略.md)。
- **渠道成员跨网能力继承运营商换电范围**（4A）：U 关闭跨网则渠道骑手也不可跨网；`userOwner` = 签约运营商 U。
- 详见 [换电场景与运营商结算.md](./换电场景与运营商结算.md)。

### 5.3.3 收缴（仅手动缴纳 · 2026-06-16 拍板 D04 / 决策 2A）

**硬性规则**：租赁设备月租金由**运营商（承租方）主动缴纳**；系统**不发起**自动扣款、代扣、经营收入划扣或定期代付。

| 方式 | 流程 | 核销时机 |
|------|------|----------|
| **微信支付** | 运营商「月租金」扫码/在线支付 | 支付成功即时核销 |
| **支付宝** | 运营商「月租金」扫码/在线支付 | 支付成功即时核销 |
| **线下对公打款** | 运营商转账至资方对公账户后**提交工单**（金额、流水号、转账日） | **资方在「租金收缴」审核确认**后核销 |

## 5.8 运营商总览 · 站点繁忙度分析

**入口**：运营商登录 → **总览** → 「站点繁忙度分析」表格（位于「数据下钻」下方、「跨运营商待结算」上方）。

**筛选联动**：随总览顶部 **站点**、**统计范围**（今日 / 近 7 日 / 近 30 日）即时刷新（等待队列以**实时**为准；格口/电池为快照）。

**原型实现**：`prototype/index.html` → `siteBusinessStats()` / `renderOverviewSiteStats()`。

### 5.8.1 评估原则（拍板）

| 原则 | 说明 |
|------|------|
| **不用收入评估站点** | 总览表**不展示**站点收入、可分配经营收入、购/换站归因金额 |
| **繁忙度三要素** | **格口占用**、**柜内电池数**、**等待中**（排队换电骑手数） |
| **不设用户绑定站** | 用户可在同运营商**任意站点**购套餐、换电；后台**不对用户标注**主绑定站或购站 |
| **收入在运营商主体** | 套餐摊销、清分、退款均在 `device_owner_id` 维度；与站点繁忙度**解耦** |

### 5.8.2 表格字段定义

| 列名 | 字段键（建议） | 定义 | 计算公式 / 聚合口径 |
|------|----------------|------|---------------------|
| 站点 | `site_name` + `site_id` | 至少有一台归属当前运营商柜机的站点 | `DISTINCT cabinets.site` WHERE `device_owner_id = 当前运营商` |
| 地址 | `site_address` | 站点详细地址 | `sites.address`，空则回退 `sites.city` |
| 柜机 | `cabinet_total` / online / offline | 该站归属本运营商的换电柜 | COUNT(cabinets)；在线 = `online=true` |
| 格口占用 | `slot_used` / `slot_total` | 换电柜格口已用/总数 | `slot_total=Σslots`；`slot_used`≈柜内电池数（IoT 上报为准） |
| 柜内电池 | `batteries_in_cab` | 当前在柜电池块数（实时） | COUNT(batteries) WHERE `in_cab` 属于本站柜机 SN |
| 等待中 | `waiting_count` | 当前排队待换电骑手数 | 队列服务 / IoT 待处理请求，按 `site_id` 聚合 |
| 繁忙度 | `busy_level` | 综合繁忙标签 | `高`：等待≥3 或格口占用率≥85%；`中`：等待≥1 或≥60%；否则 `低` |
| 站点状态 | `site_status` | 站点营业状态 | `sites.status` |

**合计行**：对数值列求和；繁忙度不合计；站点/地址列合并为「合计（N 个站点）」。

Mock 字段清单见 [站点经营分析-Mock字段清单.md](./站点经营分析-Mock字段清单.md)。

### 5.8.3 与租赁月租金的关系

月租金账单**不再引用**站点收入覆盖或 `site_allocatable`。承租方按协议**在线扫码/对公工单**缴纳租金；是否补缴仅看应付与实还，与站点繁忙度无关。

### 5.8.4 原型演示（非生产口径）

站点 `waitingCount` 为 Mock 静态值；上线须接实时队列。设备类（柜机、格口、电池）为**实时快照**。

- **订单/流水**：仅 `device_owner_id` 与当前主体一致的套餐、换电、资金、应分记录。
- **员工**：各经营主体可添加**运营员工**并配置功能权限。
- **员工登录**：独立账号登录后台；侧栏仅展示权限内模块；数据限定所属主体。

## 5.4.1 人天额度池 · 首版默认配置（2026-06-12 拍板）

| 配置项 | 首版默认 | 说明 |
|--------|----------|------|
| 扣天口径 | `calendar_day` 自然日 | 服务期按自然日计；冻结日不计 |
| 计费批处理 | `reserve_confirm` 预占确认 | 00:00 预占 1 人天；**换电或持电池**确认消耗；无换电且未持电池则日终释放；同日不重复扣 |
| 激活时机 | `on_allocate` 分配即开通 | 首换开通后续版本支持 |
| 池过期退款 | `partial_allowed` | 与顺丰浦东池 Mock 一致 |
| B2B 资金 | 采购时结清 | 运营商在渠道采购/到账时已收款；消耗仅扣池余额 + 触发平台 1% |
| 池切分 | 一运营商一池 | 渠道 × 运营商 = 唯一 `DayPool` |

> 完整算例与边界 FAQ 见 [天数池.md](./天数池.md) §9。拍板闭合记录见 §11。

## 9. 订单模型（摘要）

| 类型 | 说明 |
|------|------|
| `PackageOrder` | 须绑定主柜机 `cabinet_id` → 归属主体 |
| `SwapOrder` | 须关联套餐或额度池；记录 **userOwner / cabinetOwner / batteryOwner** 三元组；含 跨网柜机费/电池服务费（U≠C/B 时产生）；**过程日志** |
| `SwapOrderLog` | 按时间序；`kind`=user/req/dev；用于客诉与故障追溯 |
| `LeaseContract` | 出租方、承租方、**deviceListId**、租期类型、月租金、押金；**不含站点**；变更次月 1 日生效 |
| `LeaseDeviceList` | 资方独立维护的设备组合；可导入；支持清单内设备替换（`replacements` 记录） |
| `LeaseRepayment` | 按期应还/实还；状态：已还/待还/未到期 |
| `PaymentAccount` | 主体进件账户；用途：骑手收款、分账、租金收付 |
| `LeaseRentBill` | 按月账单（**按租赁协议**出账）；应还/实还；`payChannel`：微信支付/支付宝/对公打款（**不含**站点收入覆盖） |
| `LeaseRentOfflineTicket` | 运营商对公打款工单；资方审核确认/驳回 |
| `ServiceChange` | 中途完结 / 冻结 / 解冻申请及处理状态 |
| `PackageDeposit` | 电池押金；可关联信用免押凭证 |
| `DayPool` | 企业天数池；含四项合同配置与余额 |
| `DayPoolAllocation` | 向骑手分配天数；可 pending（首换开通） |
| `DayPoolDayLedger` | 按日扣天流水（口径随 `deduct_mode`） |
| `DayPoolSettleBatch` | **第二期**可选；企业按月内部对账批次（渠道池不向运营商按消耗打款） |
