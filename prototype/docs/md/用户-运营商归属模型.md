# 用户 · 运营商归属模型

> 版本 1.0 · 2026-07-02  
> 关联：[PRD.md](./PRD.md) §5.10 · [骑手端PRD.md](./骑手端PRD.md) · [换电范围策略.md](./换电范围策略.md) · [合作模式与分账.md](./合作模式与分账.md)（架构 B）

---

## 1. 核心原则

| 原则 | 说明 |
|------|------|
| **不绑站点** | 用户表无 `home_site_id`；同运营商内任意站点可购、可换 |
| **绑运营商** | 商业归属 `userOwner` = **售卖/收款运营商**（`seller_operator_id`） |
| **付款定归属** | C 端支付成功写入订单；支付单绑该运营商**子商户**（架构 B） |
| **路径 A 不单独定 OP** | 「服务购买」复用会话 `context_operator_id`，不在 SKU 页手选运营商 |

---

## 2. 三层模型

```text
L1 平台身份     user_id（注册/登录）
L2 会话上下文   context_operator_id（短期，用于展示价格、创建订单）
L3 商业归属     seller_operator_id / userOwner（支付成功或渠道/激活码权益）
```

---

## 3. `context_operator_id` 解析优先级

创建套餐订单 / 调起支付**之前**执行：

| 优先级 | 条件 | 结果 | `context_source` |
|--------|------|------|------------------|
| **P0** | 已有生效个人套餐（续费） | `seller = 原套餐 OP` | `active_pkg` |
| **P1** | 启动参数 `op=`（渠道链接等） | `seller = op` | `launch` |
| **P2** | 本会话扫柜码 `cabinet_sn` | `seller = cabinet.device_owner_id` | `scan` |
| **P3** | **定位匹配（方案乙）** | 见 §4 | `geo_auto` / `geo_manual` |
| **P4** | 无法解析 | **阻断购套餐**，引导扫柜或开启定位 | — |

> 渠道成员、设备租赁白名单（购**渠道套餐**）、激活码用户：路径 A 定位规则不适用或收款方非运营商 U，见各模式专文。

---

## 4. 方案乙 · 定位匹配规则

### 4.1 算法（首期）

1. 获取用户经纬度（需授权；拒绝则不可走 P3，提示扫柜或去设置开启定位）。
2. 查询半径 **R = 15 km** 内、状态「在营」、且对公众开放的站点（`public_open=true`；专属站不参与）。
3. 按站点到用户距离升序；取最近站点所属 `device_owner_id` 作为候选 OP。
4. **唯一候选**：最近站距离 ≤ **5 km** 且次近站所属 OP 不同或次近距离差 ≥ **2 km** → 自动写入 `context_operator_id`，`context_source=geo_auto`。
5. **多 OP 重叠**：Top2 候选 OP 距离差 < 2 km，或两 OP 在 R 内均有在营站且距离接近 → 弹层**选择服务运营商**（仅列 OP 名称 + 最近站点距离，不列站点绑定）。
6. **无候选**：R 内无在营公开站 → 阻断购套餐，文案「附近暂无换电服务，请移步换电柜扫码或切换城市」。

### 4.2 会话持久

| 项 | 规则 |
|----|------|
| TTL | `context_operator_id` 会话缓存 **7 天**（或至退出登录） |
| 刷新 | 用户点「重新定位」可重跑 §4.1；手动改 OP 后 `context_source=geo_manual` |
| 覆盖 | P1/P2 优先于 P3；链接 `op=` 与扫柜会覆盖定位结果 |

### 4.3 与站点关系

- 定位用的是**最近站点**反推 OP，**不**把该站点写入用户档案。
- 仅可选字段：`PackageOrder.nearest_site_id_at_purchase`（分析用）。

## 5. 支付关联运营商（架构 B）

```text
resolveSellerOperator(user, session) → seller_operator_id
sub_mch_id = operatorPaymentAccount(seller_operator_id)
createPackageOrder({ seller_operator_id, sub_mch_id, ... })
wx.requestPayment({ sub_mch_id, ... })
onSuccess → userOwner = seller_operator_id
```

| 字段 | 说明 |
|------|------|
| `seller_operator_id` | 订单售卖方；= 清分与退款主体 |
| `sub_mch_id` | 微信/支付宝进件子商户 |
| `context_source` | 归因来源（launch/scan/geo_auto/geo_manual/active_pkg） |
| `channel_id` / `link_code` | 可选；渠道佣金，不改变 userOwner |

**禁止**：在支付页让用户切换运营商（防串单）。

## 6. 购套餐入口与 context 关系

| 入口 | 路径 | context 来源 |
|------|------|--------------|
| 我的 · 服务购买 | A | P0→P3 解析链 |
| 宫格 · 购买套餐 | A | 同上 |
| 渠道链接/二维码 | B | P1 `op=` |
| 扫柜机码（未开通引导购） | C | P2 柜机 owner |
| 渠道兜底 1天/单次 | A | 固定为额度售卖方 U（P0 类） |

## 7. 骑手端 UI

| 场景 | UI |
|------|-----|
| 已确定 OP | 首页/我的顶部条：「当前服务：绿色出行 · 距您最近 浦东骑手驿站 1.2km」 |
| 多 OP 重叠 | 底部 sheet「选择服务运营商」列表（OP 名 + 距离） |
| 无定位权限 | 购套餐前提示开启定位，或「扫描换电柜二维码」 |
| 续费 | 不弹 OP 选择；固定原套餐 OP |

## 8. 接口（建议）

### `POST /api/v1/context/resolve-operator`

**请求**

```json
{
  "user_id": "U1028",
  "lat": 31.22,
  "lng": 121.55,
  "launch_op": null,
  "cabinet_sn": null
}
```

**响应 · 自动命中**

```json
{
  "seller_operator_id": "OP-SX",
  "seller_operator_name": "绿色出行",
  "context_source": "geo_auto",
  "nearest_site_id": "ST-SH-01",
  "nearest_distance_m": 1200,
  "need_user_pick": false
}
```

**响应 · 需用户选择**

```json
{
  "need_user_pick": true,
  "candidates": [
    { "operator_id": "OP-SX", "name": "绿色出行", "nearest_site_id": "ST-SH-01", "distance_m": 800 },
    { "operator_id": "OP-LJZ", "name": "陆家嘴联营", "nearest_site_id": "ST-SH-04", "distance_m": 950 }
  ]
}
```

### `POST /api/v1/context/confirm-operator`

用户手动选择后写入会话。

## 10. 路径 D · 平台营销（立减券 · 购时锁 OP）

> 专文：[平台营销-PRD.md](./平台营销-PRD.md) v1.1 · [decision-012](../decisions/decision-012.md)

| 原则 | 说明 |
|------|------|
| **购时必锁 OP** | 经 `ch=PLATFORM` 购套餐须带有效 `op=`（已签约）；解析为 `seller_operator_id` |
| **支付定 OP** | 支付成功写入 `userOwner`=锁定 OP；款进该运营商子商户 |
| **链接须 op** | 与路径 B 同形；优惠层为立减券，而非改写价签唯一路径 |

购套餐入口：

| 入口 | 路径 | context / seller |
|------|------|------------------|
| 平台营销链接 | D | 支付时 seller=`op`；userOwner 同步；无待激活 |
