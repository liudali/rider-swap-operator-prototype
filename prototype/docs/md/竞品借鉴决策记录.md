## C-01 总览摘要 + 下钻

### 口径

| 层级 | 内容 | 所在位置 |
|------|------|----------|
| **摘要** | 核心 KPI（换电次数、活跃用户、订单量、收入/消耗等）+ 近 7 日迷你趋势 | 外卖后台 · 各角色「总览」 |
| **下钻** | 按日/周/月趋势、维度筛选、明细表格、CSV 导出 | 数据面板 · 趋势分析 / 明细数据 |

### 角色与指标（初版）

| 角色 | 总览摘要（示例） | 下钻入口 |
|------|------------------|----------|
| 平台管理员 | 全平台换电/订单/运营商数 | 数据面板 · 按运营商 |
| 运营商 | 本主体换电、套餐单、跨网待结算 | 数据面板 · 按站点 |
| 渠道商 | 池余额、消耗、团队规模 | 数据面板 · 按团队（人天池） |
| 资金方 | 在租设备、应收 | 暂不纳入数据面板一期 |

### 不做

- 在外卖后台复制竞品独立「换电统计 / 租赁报表」整页（避免双份口径）
- 数据面板一期不做多租户权限（沿用数据面板 PRD 原型简化）

## C-03 柜级告警 + 未正常弹电池 + ICCID 管理

### 范围

| 能力 | 说明 | 主要角色 |
|------|------|----------|
| **柜级告警** | 换电柜离线、过温、门禁异常等；列表 + 按柜筛选 | 运营商 · 我的设备 |
| **未正常弹电池** | IoT 识别应弹未弹/弹错格口；告警列表 + 关联换电单 | 运营商 · 换电柜详情 / 告警中心 |
| **ICCID 管理** | 物联网卡台账：卡号、绑定设备、套餐/到期、状态；变更历史 | 平台 · 设备管理；运营商 · 只读或本主体卡 |

### 原则

- **IoT 上报为准**，后台不第二套采集
- ICCID **变更留痕**（对标竞品 ICCID 变动记录）
- 与「设备须平台绑定后才可见」规则一致

### 待详设

- [ ] 告警等级与推送（站内 / 短信）
- [ ] ICCID 逾期是否联动换电准入（竞品有 overdue 计数接口）
- [ ] 平台 vs 运营商 ICCID 可见范围
