| 原技术默认 | 产品决策 | 影响 |
|------------|----------|------|
| 1A（运营商包月仍结转） | **1B 全部实时** | 废除 C 端「待结转/可打款」主路径；支付成功即平台 1% + 运营商清分；**退款由运营商子商户原路退** + D36 须垫付 |
| 3A（仅人天池真逻辑） | **3B 三模式全 PRD** | 卡差价、设备租金池规则与 API 已闭合；原型三账号 `CH-SF` / `CH-CARD` / `CH-RENT` 可演示 |
| 6A（Mock 预警不阻断） | **单独方案** | 评估体系、押金抵扣公式、不足处置须方案闭合后方可开发信用模块 |
| 9A（一期架构 B） | **9C 暂缓** | 支付通道选型阻塞于财务/法务会；一期按 Mock 推进，不提前锁通道 |
