# 渠道结算模式规则

> **状态**：已确认（2026-06-17，§3 修订 2026-06-12）  
> **决策来源**：[需求评审决策记录.md](./需求评审决策记录.md) #3B、C-06–C-08  
> **关联**：D03 单运营商签约、D05 三种模式、D16/D38 渠道信用与免押

---

## 1. 共性规则

| 规则 | 说明 |
|------|------|
| **签约** | 一期每渠道**仅签一家运营商** |
| **userOwner** | 经渠道链接购卡的用户 `userOwner` = 签约运营商 U |
| **跨网** | **继承运营商换电范围**（4A） |
| **渠道免押** | **人天池 / 设备租赁**：押金算渠道 → [渠道信用评估方案.md](./渠道信用评估方案.md)。**渠道分销**（骑士卡等）：用户直购，按个人用户押金/免押规则，**不适用**渠道信用 |
| **C 端兜底** | 三种模式下渠道兜底 C 端支付均为 **平台 1% + 运营商 99%** |
| **清分** | C 端实收：**支付成功实时清分**（1B） |

## 2. 模式一：人天额度池（人天池）

> 详规见 [天数池.md](./天数池.md)

| 维度 | 规则 |
|------|------|
| **采购** | 渠道向 U 批量购买人天；B2B 款采购时结清 |
| **消耗** | 预占确认：换电或持电池扣 1 人天/日 |
| **平台 1%** | **确认消耗时**按标准人天价 × 1% 向 U 计提 |
| **兜底** | 池不足时骑手 C 端在线支付（1天/单次 SKU） |

## 3. 模式二：渠道分销（骑士卡）

> **2026-06-12 修订**：骑士卡渠道为**销售渠道**，非 B2B 批发入库再分配。用户通过渠道**推广链接**直购套餐；系统标记渠道来源；运营商与渠道**佣金结算**。

### 3.1 商业模式

渠道商（如骑士卡渠道）是运营商的**分销/推广渠道**：

- 运营商维护套餐**正式零售价**（定价管理）。
- 运营商与渠道协商**渠道专享价**（低于正式价，按 SKU）。
- 渠道获得**专属推广链接**；用户点击链接**直接购买**，无需渠道后台分配卡片。
- 订单与用户打上**渠道来源标记**（`channel_id` / `link_code`）。
- 渠道收入 = 运营商按约定支付的**佣金**（按单固定金额或比例，签约时配置）。

**不存在**：渠道批发购卡、卡库存、持卡骑手分配、CO 批发订单。

### 3.2 计费与收银（C-06 / C-08 · 已确认）

| 时点 | 规则 |
|------|------|
| **用户购卡** | 经渠道链接以**渠道专享价**支付；**平台代收**（款进运营商子商户）；**支付成功实时清分** 1% → 平台，余款 → 运营商 |
| **渠道标记** | 订单、用户档案写入渠道 ID；换电权益随套餐订单即时开通 |
| **平台 1%** | **购卡支付成功时**按实付金额 × 1% 计提（C-06A） |
| **渠道佣金** | 按签约规则计算（如 ¥25/单 或 实付 × N%）；**运营商与渠道线下/周期结算**；平台提供**佣金对账明细** |
| **换电权益** | 与个人套餐相同，有效期内按 SKU 规则换电；**不扣人天池** |
| **跨网费** | 同其他用户，U 承担日清跨网设备服务费 |
| **C 端兜底** | 平台 1% + 运营商 99% |

### 3.2.1 渠道差异化定价（2026-06-12）

同一运营商下，**不同分销渠道**可配置不同的渠道专享价与佣金。

| 维度 | 规则 |
|------|------|
| **定价粒度** | 签约渠道 × 套餐 SKU |
| **正式零售价** | 运营商「定价管理 → 个人套餐价」；全渠道基准价 |
| **渠道专享价** | 运营商与渠道协商；**须 ≤ 正式零售价**；用户经链接按此价支付 |
| **佣金** | 按单固定金额或实付比例；签约时配置，对账按单汇总 |
| **配置入口** | 运营商「**渠道管理 → 签约渠道**」；「**定价管理 → 渠道分销价**」汇总对比 |

### 3.2.2 多渠道分销 · 套餐授权与定价

同一运营商下可并存**多个分销渠道**（演示：`CH-CARD` 骑士卡、`CH-DELIV` 闪送骑士卡），管理原则：

| 步骤 | 操作 |
|------|------|
| 1. 签约 | 「渠道管理 → 签约渠道」为每个渠道商创建独立合同，结算模式 = 渠道分销 |
| 2. 授权 SKU | 编辑签约时为该渠道勾选可售套餐（如 30 天包月、7 天卡） |
| 3. 定价 | 每个渠道×SKU 分别配置**正式零售价**（只读基准）、**渠道专享价**、**佣金/单** |
| 4. 汇总 | 「定价管理 → 渠道分销价」Tab **横向对比**各渠道同 SKU 价格 |
| 5. 渠道侧 | 渠道登录后在「套餐与链接」基于授权 SKU 生成链接/二维码；**不可改价** |
| 6. 成交 | 用户经链接 24h 内购授权 SKU 享该渠道专享价；佣金按渠道规则结算 |

**渠道商权益（分销）**：专享价销售权、推广链接/二维码、成交与佣金对账；**不含**统管骑手押金/团队（与人天池区分）。

| **约束** | 渠道专享价 ≤ 正式零售价；调价对新单生效 |

**示例**（同一运营商 OP-SX）：

| 渠道 | SKU | 正式零售价 | 渠道专享价 | 佣金/单 |
|------|-----|------------|------------|---------|
| 骑士卡渠道 | 包月 30 天 | ¥299 | ¥279 | ¥25 |
| 闪送骑士卡 | 包月 30 天 | ¥299 | ¥269 | ¥30 |

### 3.3 推广链接、二维码与小程序落地

渠道在「**套餐与链接**」管理可售套餐，并为同一套餐生成**多条推广链接**（如包月 ¥279 可同时有 App Banner、短信、社群等链接）。每条链接可**一键生成二维码**，用于地推海报、社群、短信等场景。

| 字段 | 说明 |
|------|------|
| `package_id` / `sku_id` | 所属可售套餐（可选预选 SKU） |
| `link_code` | 渠道 + SKU + 链接唯一短码 |
| `purpose` | **链接用途**（渠道自定义，如「App 首页 Banner」） |
| `link_url` | 运营商**小程序路径**，带 `op`、`ch`、`lnk`、可选 `sku` |
| `qr_code` | 与 `link_url` 内容一致的二维码图片（PNG/SVG，可下载） |
| 统计 | **每条链接**独立统计点击、成交（链接购卡成功笔数） |
| 权限 | 渠道可新建/停用链接、下载二维码；**不可改价**（专享价由运营商签约配置） |

**用户路径（C 端）**：

1. 用户点击推广链接或**扫描二维码** → **直达签约运营商小程序**（非 H5 中转页）。
2. 完成**微信授权登录/注册** → **实名认证**（购套餐前置条件）。
3. 套餐列表展示价格：若处于渠道归因窗口内，展示**渠道专享价**；否则展示运营商**正式零售价**。
4. 支付成功 → 订单与用户写入渠道来源标记；换电权益即时开通。

**链接 URL 示例**（演示）：

```
wxmp://OP-SX/pages/landing/index?op=OP-SX&ch=CH-CARD&sku=SKU-30D&lnk=qsk-30d-home
```

### 3.3.1 24 小时渠道归因窗口

| 维度 | 规则 |
|------|------|
| **触发** | 用户**首次点击/扫码**某条**有效**推广链接时，记录 `attribution_start` |
| **窗口** | 自点击时刻起 **24 小时**（自然时间） |
| **价格范围** | 窗口内用户购买该渠道签约授权的**全部 SKU**，均按**渠道专享价**展示与结算（不限于链接绑定的单一 SKU） |
| **订单归属** | 归因期内成交订单写入 `channel_id`；`link_code` 取**首触链接**（用于链接级转化统计） |
| **窗口续期** | 归因期内再次点击**同一渠道**其他链接 → **不重置** 24h 窗口（首版默认） |
| **渠道切换** | 窗口内点击**其他渠道**链接 → 以**最后触达渠道**为准，**重置** 24h 窗口 |
| **过期** | 超过 24h 且未支付 → 恢复**正式零售价**；已支付订单不受影响 |

> **与非归因用户差异**：未点击渠道链接的用户，在运营商小程序内按**正式零售价**购套餐，订单**无**渠道标记、**不计**渠道佣金。

### 3.3.2 购卡记录

| 字段 | 说明 |
|------|------|
| 范围 | **仅**经本渠道推广链接成交的套餐购买 |
| 筛选 | 支付时间起止、套餐、链接用途、用户手机 |
| 关联 | 订单写入 `channel_id`、`link_code`、`link_purpose` |

### 3.4 佣金对账（平台提供）

| 字段 | 说明 |
|------|------|
| 周期 | **自然月**（非按单滚动） |
| 链接购卡笔数 | 当月带渠道标记的套餐订单 |
| 实付合计 | 当月用户经链接支付总额 |
| **应结佣金** | 按签约规则汇总 |
| 平台 1% | 已清分明细 |
| 明细 | 可选某月展开订单明细（渠道侧） |

### 3.5 后台菜单

总览、结算说明、**套餐与链接**、**购卡记录**、**佣金对账**、变更记录

> **不含**「渠道信用额度」：分销渠道无统管骑手/设备押金，不适用渠道信用评估。  
> **不含**「员工」：分销渠道无后台运营分工，仅渠道主体账号登录查看链接与对账。

> **说明**：原「CO 批发购卡 → 卡库存 → 持卡分配」路径**已废弃**。第三方 App SDK 对接（原美团骑士卡）**已取消**。一期仅走 **推广链接 → 用户直购 → 渠道标记 → 佣金结算**。

## 4. 模式三：设备租赁

> **2026-07-01 修订**：原「设备租金池」更名为**设备租赁**。取消「预缴租金池余额 + 骑手兜底 SKU」模型；改为**运营商维护租赁设备清单与月租**，渠道**白名单用户**免 C 端购套餐、免按次计费。

### 4.1 商业模式

渠道向签约运营商**租赁换电柜/电池**，按设备清单缴纳**月租**（B2B）。运营商为渠道维护：

- **租赁设备清单**（柜机/电池 SN、部署站点；**不含单台月租**）
- **签约统一月租**（渠道与运营商 B2B 约定的一口价，MO 账单按此金额）
- **渠道专属站点**（专用载体：不对公众开放，仅白名单可见可用）
- **白名单用户**（由**渠道商自行维护**；仅名单内骑手可换电）

渠道 B2B 向运营商付月租；**白名单骑手不向平台/运营商购买套餐**，换电**不扣人天、不按 C 端 SKU 计费**。服务通过**专属站点 + 白名单**闭环，与普通 C 端开放站点隔离。

### 4.2 运营商设置（签约与资产）

| 能力 | 规则 |
|------|------|
| **签约** | 结算模式 = 设备租赁；合同有效期、主体信息 |
| **设备清单** | 运营商维护：绑定 SN、类型、部署站点；**不对单台设备定价** |
| **月租** | 签约时与渠道约定**统一月租**（元/月）；MO 账单按签约价；**增删设备不改变月租**（除非重新签约） |
| **专属站点** | 签约时**新建**或绑定站点，标记 `channel_dedicated=true`、`public_open=false` |
| **欠费策略** | 月租未到账：**停服**（白名单骑手不可换电）；**不走** C 端兜底 SKU |
| **渠道停服** | 运营商将渠道商主体 `status=已停用`：**白名单不可取电/不出电**；扫码提示「当前站点已到期，请联系管理员续费」（`E_LEASE_CHANNEL_SUSPENDED`）；**仍允许还电入柜** |

### 4.2.1 渠道专属站点（专用 · 白名单可见）

| 维度 | 规则 |
|------|------|
| **定位** | 为签约渠道提供换电服务的**专用站点**；与普通「开放站点」区分 |
| **公众开放** | `public_open=false`：**不对普通 C 端用户开放**；不在公众地图/站点列表展示 |
| **小程序可见性** | 骑手端地图、附近站点、定位检索：**仅对该渠道白名单用户**返回专属站 POI |
| **非白名单** | 地图**不可见**；若通过柜机码等方式触达，扫码拦截 `E_WHITELIST` |
| **运营商** | 创建/绑定专属站、维护设备与月租；**不维护**渠道白名单 |
| **渠道商** | 查看专属站与租赁设备；**自行维护**白名单（增删骑手） |

### 4.3 白名单用户

| 维度 | 规则 |
|------|------|
| **准入** | 仅渠道**添加**的手机号/骑手可换电；**由渠道商自行维护** |
| **组织** | **无团队/车队属性**（与人天池的团队登记不同）；扁平白名单 |
| **权益** | 免押（押金算渠道信用）、**免 C 端购套餐**、换电不限次（合同期内）；可在小程序看到并导航至**本渠道专属站** |
| **非白名单** | 专属站地图不可见；扫码拦截：`E_WHITELIST`；**不提供**在线购套餐入口 |
| **维护方** | **渠道商**在「白名单用户」增删；运营商**仅查看** |
| **离职/移除** | 移出白名单即停服；若持有电池须先归还；专属站对其不可见 |
| **渠道已停用** | 运营商停用渠道商主体后，名单内用户**暂停取电**；**还电入柜不受限**；恢复「在营」后自动恢复换电 |

### 4.3.1 电池持有（渠道可见）

| 维度 | 规则 |
|------|------|
| **数据来源** | 换电成功后 IoT/订单同步：当前**用户 ↔ 电池 SN** 绑定关系 |
| **展示** | 渠道商「**电池持有**」模块：白名单用户、手机号、当前持有电池 SN、SOC、取电时间、所在站点 |
| **未持有** | 显示「未持有」；仍可在白名单内换电取电 |
| **权限** | 渠道**只读**；不可手工改绑（以换电记录为准） |
| **运营商** | 可在渠道权益/订单侧查看同一关系，**不替代**渠道侧模块 |

### 4.4 计费与平台 1%

| 时点 | 规则 |
|------|------|
| **月租支付** | 渠道 → U 支付当月设备月租；到账后账期生效 |
| **换电** | 白名单用户：不扣人天、无 C 端实收 |
| **平台 1%** | **月租支付成功时** × 1%（B2B） |
| **跨网费** | 同其他用户：U 承担日清跨网设备服务费（若开启跨网） |

### 4.5 后台菜单

**运营商**：渠道管理（签约 · MO 月租单 · 租赁设备 · 专属站点）

**渠道商**：总览 / 结算说明 / **月租账单** / **租赁设备** / **电池持有** / **白名单用户** / 变更记录 / 渠道信用 / 员工

> **废弃**（相对旧「设备租金池」）：租金池余额、预缴充值、到期兜底 SKU（`fallback_sku`）、C-07 到期策略。

## 5. 三模式对照表

| 维度 | 人天池 | 渠道分销（骑士卡） | 设备租赁 |
|------|--------|-------------------|----------|
| 渠道角色 | 额度采购方 | **推广销售渠道** | **设备承租方** |
| 用户获权益 | 登记 + 分配人天 | **链接/二维码 → 小程序购套餐** | **白名单添加** |
| 平台 1% 触发 | 确认消耗 | **购卡支付成功** | **月租支付成功** |
| 骑手付款 | 池不足→兜底 SKU | **链接专享价 · 平台代收** | **无**（白名单免付费） |
| 非授权用户 | 可自费兜底 | 正常 C 端购套餐 | **地图不可见 · 扫码拦截** |
| 运营商维护 | 批发价、额度池 | 专享价/佣金 | **设备清单、月租、专属站点** |

## 7. 决策记录（C-06–C-09）

| ID | 决议 |
