## 决策问题

押金信息散落在套餐单押金列与退款管理中，运营商如何集中查看用户已付/免押押金？

## 调用的大师视角

- **贝佐斯（逆向）**：3 年后运营日常要「按手机/状态查在押押金」→ 独立台账页优于在套餐列表堆字段。
- **乔布斯**：列表只做押金台账；退押动作仍归「退款管理」，避免双入口审批。

## 已确认规则

| # | 规则 |
|---|------|
| 1 | 菜单位置：订单与服务 → **用户押金**（一期） |
| 2 | 数据源：本运营商套餐单衍生；含 **实付押金** 与 **信用免押**；**不含**渠道担保、无需押金 |
| 3 | 状态：实付在押 / 退押中 / 已退押 / 信用免押 |
| 4 | 筛选：套餐单号、用户手机、支付日起止、押金状态 |
| 5 | 分页：默认 8 条/页；改筛选回第 1 页 |
| 6 | 操作：套餐详情；退押中可跳转退款管理 |

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 与骑手电池押金口径一致；扫在押/退押更快 |
| 反对 | 与「用户」列表押金列部分重叠 |
| 失效 | 若押金改为独立支付单（非同笔套餐）需改数据源 |
