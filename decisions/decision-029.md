# 决策卡片 029 · 平台服务 L1：保证金账户 + 平台服务费

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-16 |
| 阶段 | MVP / 信息架构 |
| 结论 | **已锁定**：运营商侧栏将「服务保证金账户」「平台服务费」收拢为一级菜单 **平台服务** 下的二级 |

## 支持理由

- **乔布斯减法**：两页同属「对平台缴存/缴费」心智，合并减少一级菜单噪音
- **俞军价值公式**：找保证金充值 / 看服务费账单时少一次「在哪个一级」的认知成本

## 反对理由

- 权限粒度若需「只开保证金不开服务费」→ 仍用 `deposit.view` / `platform_fee.view`，员工可见 L1 后按权进页（`[假设]` 两权任一即可见 L1）

## 失效条件

- 商务要求两能力完全拆开运营团队 → 可再拆一级

## 结构

```
平台服务（L1）
├── 服务保证金账户（L2，默认）
└── 平台服务费（L2）
```

## 兼容

- 旧 `data-view-jump=depositAccount|platformFee` 自动落到对应 L2
- 员工权限映射至 `platformService`
