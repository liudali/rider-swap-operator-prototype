# 决策卡片 049 · 人天额度耗尽仍持电池

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-21 |
| 阶段 | MVP |
| 结论 | **已锁定**：D 日有额度并持电池 → D+1 人天不可用且未还电时进入 **「待还电」**（仅可还电、禁止换电；不透支）。原因必须区分 **个人无额度** / **预占失败**。渠道商顶栏在「余额不足」旁增加 **「骑手零额度」** 预警（在职且剩余人天=0），点击进骑手登记名单 |

## 场景定义

| 时点 | 状态 |
|------|------|
| D 日 | 有今日资格；持有电池未归还 |
| D+1 日 | ①个人剩余人天=0，或②当日预占失败；电池仍在手 → **两者均进待还电**，原因字段区分 |

## 调用的大师视角

- **第一性原理**：人天=当日服务权，非电池物权；服务权到期须可还电。
- **俞军**：仅还电，替换成本清晰；自费旁路已废止（054）。
- **乔布斯**：顶栏只加「骑手零额度」一条硬预警，与余额不足并列，不另开模块。

## 已确认规则

| # | 规则 | 说明 |
|---|------|------|
| 1 | **状态** | 持电池且今日无可用人天 → `待还电` |
| 2 | **原因枚举** | `gateReason`：`个人无额度` \| `预占失败`（预占失败可附 `failReason` 如余额不足） |
| 3 | **门禁** | `allowed_swap=false`，`allowed_return=true`；错误码 E006/E007 |
| 4 | **计费** | 无预占则不扣池、不欠费 |
| 5 | **自费兜底** | **已取消**（见 decision-054）；零额度不可自费换电 |
| 6 | **渠道预警** | 顶栏「骑手零额度」紧挨「余额不足」；额度池页同款横幅；落地骑手登记筛选在职·剩余0 |
| 7 | **还电后** | 未持电池；无额度则权益仍不可用 |

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 与离职/停用仅还电一致；保护资产；不破不透支 |
| 反对 | 「电没用完却要还」需文案解释自然日占用 |
| 失效 | 改为透支持电计费 / 远程锁电一期上线 |

## Non-goals

- 一期不做违约金、远程锁电
- 运营商顶栏不汇总他渠道路骑手零额度
