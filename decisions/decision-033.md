# 决策卡片 033 · 平台服务费区分 C 端分账与 B 端计提

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-16 |
| 阶段 | MVP |
| 结论 | **已锁定**：运营商「平台服务费」补齐 C 端支付分账；与 B 端消耗计提分 Tab 展示 |

## 支持理由

- **俞军价值公式**：财务对账需同时看见「支付瞬间分账」与「消耗计提」，只看 B 端会漏掉主要 C 端收入路径
- **乔布斯减法**：不新增一级菜单；页内 Tab 拆两类任务，总览保留 KPI + 月结

## 反对理由

- C 端明细与「套餐购买订单」有交叉；接受以「平台费视角」再列一层 `[假设]` 对账价值 > 重复成本

## 失效条件

- C/B 合并为单一流水且产品强制统一状态机 → 可改回单表 + 类型筛选

## 结构

```
平台服务费（L2）
├── Tab：费用总览（比例、C/B KPI、月结账单）
├── Tab：C 端支付分账（套餐实付 × C 费率 · 通道分账）
└── Tab：B 端消耗计提（确认消耗/激活码 · 保证金代扣）
```

## 数据

- C：由本运营商 `packageOrders` 派生
- B：`platformFeeAccruals` 中确认消耗 / 激活码核销
