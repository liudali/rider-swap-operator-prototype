# 决策卡片 013 · 平台营销立减券成本默认由运营商承担

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-16 |
| 阶段 | MVP |
| 结论 | **已确认**：平台营销立减券的面价差（`coupon_amount`）**默认由锁定运营商承担**（让利），平台不做营销预算补贴出账。**交付排期见 [decision-014](./decision-014.md)（整块二期）**。 |

## 决策问题

decision-012 留下的 `[待确认]`：券成本谁出——平台补贴 vs 运营商让利？

## 调用的大师视角

- **俞军**：用户已按实付成交；成本分摊不改变 C 端体验，但改变 B 端参与激励。运营商承担 = 参与活动即接受让利，价值公式对运营商为「获客增量 − 让利 − 营销服务费」。
- **王慧文（双边）**：平台侧零补贴降低获客烧钱；运营商侧需用 opt-in 过滤，避免强制让利导致退出。
- **贝佐斯（逆向）**：与「款进运营商、不代收」一致——少一条平台补贴清算链路。

## 核心判断

| 项 | 口径 |
|----|------|
| 用户实付 | `paid = list − coupon` → 进锁定 OP 子商户 |
| 券成本 | **运营商让利** = 少收 `coupon_amount`；**无**平台补贴流水 |
| 1% | 仍按 **实付** `paid` 分账 |
| 营销服务费 | 另按协议元/单应付平台（与券成本独立） |
| 对账展示 | 立减合计 = 运营商本月让利额（只读记账，非应付平台） |

## 反方最强论据

- 小运营商可能因让利 + 营销费双重负担而拒签，活动覆盖面下降。
- 平台统一投放却不贴补，运营商议价权弱时易感知不公。

## 失效条件

- 总部强制要求平台出资拉新（品牌投放合同）；或监管要求优惠成本由平台列示。

## Non-goals

- 不一期做「平台补贴 / 运营商共担」双模式切换（可二期协议字段扩展）。
- 不改变用户实付与收款主体。

## 下一步

- 签约文案明示「参与即接受立减面价差由本运营商承担」。
- 退款：用户退 `paid`；无平台补贴冲回；营销服务费冲正规则不变。
