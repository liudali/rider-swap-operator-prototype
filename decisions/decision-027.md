# 决策卡片 027 · 收款账户须绑定对公结算账户

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-16 |
| 阶段 | MVP |
| 结论 | **已锁定**：收款账户（进件子商户）必须绑定**对公结算账户**；平台审核提现后打款至该对公户；未绑定不可提现 |

## 支持理由

- **贝佐斯逆向工作法**：3 年后对账/打款仍须法人对公户，今日即落绑定闸门，避免「只进件不绑卡」脏数据
- **俞军价值公式**：相对「口头登记结算卡」，结构化绑定降低财务差错与客服成本，替换成本低（一次表单）

## 反对理由

- 进件尚未完成对公材料时阻塞提现 → 接受：合规优先；可先开通子商户再补绑对公
- 个人独资等非对公场景 → `[假设]` 一期主体均为公司对公；个人卡不在本期

## 失效条件

- 监管要求结算至个人卡或存管专户 → 需扩展账户类型枚举，而非取消绑定

## 字段（对公结算账户）

| 字段 | 必填 | 说明 |
|------|------|------|
| 对公户名 | 是 | 一般与主体营业执照一致 |
| 开户行 | 是 | 含支行 |
| 对公账号 | 是 | 可脱敏展示 |
| 联行号 | 否 | 便于跨行打款 |

## 规则

1. 微信/支付宝/对公转账等进件行均可绑定；**预存户**无需对公
2. 默认 C 端子商户未绑定 → 收款账户警示 + 提现拦截
3. 变更对公不影响历史提现申请；新申请用新账户
4. 渠道即时到付 / 设备租赁收款账户同样须绑定对公

## Non-goals

- 不接银行四要素实时鉴权（原型 Mock）
- 不做多对公轮询分账
