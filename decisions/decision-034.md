| 结论 | **已锁定（建议）**：日清账单、周月汇总 **不是** 保证金账户变动的派生账；**不宜**并入「服务保证金账户」。若要减 L2，应并入「运营商往来」页内 Tab |

## 事实关系（数据血缘）

```
跨网换电 → 跨网服务费明细（事实明细）
         → 日清账单（按日轧差代收/代付）
         → 周月汇总（日账单聚合）
         ↓ 结算动作
    划扣/入账 → 保证金变动明细（钱包流水，含日清划扣、平台费代扣、充值…）
```

- **日清/周月的真相源**：跨网往来（平台代收/代付），不是保证金流水
- **保证金变动**：结算结果之一；同一账本还有充值、平台服务费代扣等，口径更宽

## 支持理由（不宜并入保证金）

- **唐·诺曼 · 心智模型**：保证金页 =「我的清分钱包」（充值/余额/流水）；日清 =「跨网结算单据」。并入会混「账户」与「往来结算」两套任务
- **乔布斯减法**：保证金已有 概况/充值/变动 三 Tab；再塞日清/周月 → 钱包页过载，且切断与「运营商往来 → 明细」的下钻链
- **俞军价值公式**：对账同学找「某日跨网轧差」时，从往来进日清成本更低；从保证金进易误当成「只看扣款流水」

## 反对理由（为何有人想并）

- 日清优先划扣保证金，结果会写进保证金变动 → 表象「都是钱从保证金走」
- 侧栏 L2 仍偏多，想继续收拢

## 失效条件

- 产品把日清重新定义为「仅保证金日终对账单」且不再展示代收/代付业务口径 → 可再议并入保证金

## 更好的收拢（若要减侧栏）

```
运营商往来（L2）
├── Tab：概况
├── Tab：跨网服务费明细

└── Tab：周月汇总      ← 建议迁入此处

服务保证金账户（L2）仍只做钱包：概况 / 充值 / 变动明细
```
