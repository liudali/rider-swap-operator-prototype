# 决策卡片 011 · 平台统价拆二级 + 服务费归运营商管理

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-15 |
| 阶段 | MVP / 信息架构 |
| 结论 | **已确认**：原「平台统价」单页拆为二级：**跨网服务费**（含城市覆盖）/ **人天标准日值**（新增城市覆盖）/ **预警短信**。「运营商平台服务费」迁至 **运营商管理** 二级。 |

## 为何改

- **乔布斯减法 + IA**：统价杂糅四块能力，按业务语义拆栏更易找。
- **俞军**：运营商费率与运营商主体同屏，降低「改费率时还要切统价页」的替换成本。

## 规则

1. 跨网：全网默认柜机/电池费 + 城市覆盖；改价不追溯。
2. 日值：全网默认 + **城市覆盖**；B 端计提优先城市价 `[假设]`。
3. 运营商平台服务费：C/B 比例在运营商管理维护。
4. 预警短信：模板 + 发送记录独立页。

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 导航语义清晰；费率与运营商治理同域 |
| 反对 | 侧栏 L2 变多，培训需一次性说明 |
| 失效 | 日值若永久全国一价且无城市诉求，可再合并回单页 |
