# 决策卡片 032 · 服务保证金账户页内 Tab

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-16 |
| 阶段 | MVP / 信息架构 |
| 结论 | **已锁定**：服务保证金账户长页拆为页内 Tab：账户概况 / 充值申请 / 变动明细 |

## 支持理由

- **乔布斯减法**：概况、充值动作、账本三类任务拆开，首屏不再堆叠
- **俞军价值公式**：财务提交充值时直达「充值申请」，不必滚过档位与 KPI

## 反对理由

- 专户信息在概况与充值各露一次，略重复；充值 Tab 保留完整专户便于转账对照

## 失效条件

- 充值与概况强绑定须同屏对照（如边看余额边填单）→ 可改回单页或分栏

## 结构

```
服务保证金账户（平台服务 L2）
├── Tab：账户概况（准入档位、KPI、专户摘要）
├── Tab：充值申请（专户 + 提交表单 + 申请列表；待确认角标）
└── Tab：变动明细
```

## 兼容

- `data-view-jump=depositAccount` → 落到「充值申请」Tab
- state：`depositAccountTab` = `overview|recharge|ledger`
