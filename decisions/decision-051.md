# 决策卡片 051 · 人天池「骑手团队」并入「骑手登记」

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-21 |
| 阶段 | MVP / 信息架构 |
| 结论 | **已锁定**：顶层 Tab 去掉独立「骑手团队」；并入「骑手登记」页内 Tab。登记骑手操作列：在职/离职均可**加入 / 移除 / 变更团队** |

## 结论

1. **导航**：人天额度池顶层 Tab = 额度池 / **骑手登记** / 额度分配 / 额度使用规则 / 消耗明细 /（二期零售价·异常）——**无**独立「骑手团队」。
2. **骑手登记**页内 Tab：
   - **登记骑手**：原登记列表 + 登记/批量导入
   - **骑手团队**：原团队列表（创建团队、绑定消耗池）
3. **操作列（登记骑手）**——在职、离职均支持：
   | 操作 | 规则 |
   |------|------|
   | **加入团队** | 无团队，或离职复职入队；选团队后 `status=在职`，池随团队绑定 |
   | **变更团队** | 已有团队时换队；**不**改变在职/离职；池随新团队；`[假设]` 剩余人天仍挂骑手，扣减池切换为新团队池 |
   | **移除团队** | 未用人天回池；`status=离职`；保留末次团队展示便于追溯 |

## 支持理由

- **乔布斯**：团队是登记的从属编排，独立 L2 增加跳转成本。
- **俞军**：离职后仍要改队/复职是真实运营路径；操作死角（离职仅「—」）替换成本高。

## 反对理由

- 页内双 Tab 略增登记页复杂度；团队管理员权限边界需仍只读消耗。

## 失效条件

- 团队成为跨池编排一级实体、需独立审批/合同页时，可再拆出 L2。
