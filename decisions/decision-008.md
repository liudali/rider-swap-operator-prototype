# 决策卡片 008 · C 端退款：平台 1% 按比例冲正退还

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-15 |
| 阶段 | MVP / 规则更正 |
| 结论 | **已确认**：**支付渠道按比例原路退** → 原支付清分出的**平台 C 端 1% 须按同比例退还/冲正**。废止「平台 1% 不退」（D24 旧口径）在 **C 端微信/支付宝原路退** 场景下的适用。 |

## 为何改

- **第一性原理**：通道侧退款是「按付款比例退回各分账方」，不能只把用户款从运营商退出而不冲平台已分得的 1%。  
- **俞军**：旧规则让运营商在退全款时净亏平台费，替换成本高、公平性差；新规则与通道一致，降低对账摩擦。

## 规则

1. **适用范围**：个人套餐 / 渠道 C 端直购等**经支付渠道收款并分账**的订单退款（冷静期、中途完结、SKU 退订等）。  
2. **计算** [假设]：`平台费退还 = round(退套餐费 × 该单 C 端费率, 2)`；也可等价为 `原平台费 × (退套餐费 / 原套餐实付)`。  
3. **押金**：押金本不参与 1% 清分 → **退押不产生平台费冲正**。  
4. **出款**：用户仍看到「原路退」；平台侧在清分明细记**平台费冲正（退还）**，保障运营商子商户可组齐应退用户金额。  
5. **已提现 / 须垫付**：仍适用；垫付口径含「用户应退合计」，平台费冲正另记账。[待确认] 平台费退还不完是否阻塞用户原路退。

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 与微信/支付宝比例退款机制一致；账务可解释 |
| 反对 | 平台营收随退款波动；部分退款比例计算需工程严密 |
| 失效 | 支付通道改为「仅从收款方扣、平台费一律不回退」的产品能力，或监管另有强制 |

## 废止关系

- **废止适用**：D24「平台 1% 不退」→ **仅不再用于 C 端支付分账退款**  
- **仍有效**：B 端人天/激活码**确认消耗后计提**的 1% 是否可退 → **[待确认]** 本期不展开（非通道原路比例退场景）

## 原型 / 文档

- 退款管理字段：`platformFeeRefund`（原 `platformFeeNonRefund` 语义翻转）  
- PRD / 合作模式与分账 / 个人套餐定价与退款 / 验收：同步「按比例退还」
