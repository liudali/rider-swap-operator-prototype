| 结论 | **已确认并原型化**：运营商「定价管理」新增 **押金设置**；配置个人套餐用户在**信用不足不免押**时须缴纳的电池押金数额 |

## 已确认

1. **落位**：定价管理 → 押金设置（与个人套餐价并列，非退款/用户台账）  
2. **对象**：仅**个人套餐**用户；渠道人天 / 设备租赁白名单不适用  
3. **触发**：信用（芝麻/支付分等）**不足以免押**时实缴本数额；达标则实收 ¥0  
4. **金额维度** [假设]：**运营商级**统一数额（不按 SKU / 城市分档），与套餐零售价独立  

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 俞军：押金是换电启用的「价」类参数，挂定价管理符合心智；乔布斯：不另开菜单，避免与「平台保证金」混淆 |
| 反对 | 高损耗城市可能要差异化押金；按城市分档后置可扩 |
| 失效 | 监管强制统一押金、或改由平台统一定价 |

## 边界

- 关闭「启用实缴」= 本运营商个人用户亦不收押（演示开关）  
- [假设] 新设立即对新购生效；在途单沿用下单时数额  
- 退押模式仍在「退款管理」；本页只定**收多少**

## 原型

- Mock：`operatorPersonalDepositSettings`（OP-SX=¥99 / OP-LJZ=¥199）  
- Tab：`pricingTab=deposit`；可保存数额与启用状态
