# 决策卡片 028 · 换电订单不展示个人套餐应分；激活码按天/次消耗

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-16 |
| 阶段 | MVP / 二期（激活码） |
| 结论 | **已锁定**：换电订单「消耗」列仅服务 B 端记账；个人套餐应分在支付时已清分，本表不展示；二期激活码按天/次确认消耗（类人天） |

## 支持理由

- **乔布斯减法**：个人套餐在换电单重复展示 ¥应分/已清分，造成「每笔换电又清分一次」误解，砍掉
- **俞军价值公式**：财务对账以支付清分为真源；换电页只回答「这次换电耗了什么额度」

## 反对理由

- 运营习惯在换电单看「这笔单赚多少」→ 引导至套餐购买订单 / 清分明细
- 激活码若坚持「仅核销时一次计提、换电不记消耗」→ 与「类人天按天/次」冲突；本决策以用户口径为准

## 失效条件

- 监管要求次卡按次实时分账至多方 → 需单独产品线，不回退个人包月应分到换电单

## 展示规则

| 权益来源 | 消耗列 | 额度状态 |
|----------|--------|----------|
| 个人套餐（在线支付） | — | — |
| 渠道人天 | N 人天 | 已确认消耗 / … |
| 激活码（二期） | N 天 或 N 次 | 已确认消耗 |

## 口径说明

- **个人套餐**：支付成功 → 实时清分；换电=履约，不产生新的应分流水
- **激活码**：渠道线下批发；骑手输码开通；换电按天/次记消耗，用于向运营商侧结算/对账（类人天）
- **平台 1%**（激活码）：`[假设]` 仍可在核销时按「标准人天价×服务人天」计提；与换电消耗展示并行，不在换电单展示 C 端应分金额

## Non-goals

- 不把个人套餐次卡改成「按次实时清分」
- 不删除激活码核销计提入口
