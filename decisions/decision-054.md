# 决策卡片 054 · 取消运营商兜底 SKU · 零额度仅还电

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-22 |
| 阶段 | MVP |
| 结论 | **已锁定**：移除全部「运营商渠道兜底 SKU / 自费换电」能力；渠道骑手**无人天额度**时**禁止换电**，持电池**仅可还电** |

## 结论

1. **取消兜底**：运营商定价不再标记「渠道兜底必选」；骑手端 C06 不再提供 1天/单次自费换电。  
2. **门禁**：`allowed_swap=false`；持电池 `allowed_return=true`（待还电）。  
3. **出路**：联系渠道续配 / 池恢复后次日预占成功；不开放 C 端自费旁路。  
4. **个人套餐**：1天/单次仍可作为**个人用户**可选 SKU，与渠道无关。

## 支持理由

- **乔布斯**：砍掉「额度失败还能付钱继续」旁路，逼渠道把额度经营做对。  
- **俞军**：自费兜底让骑手感知「还能用」，替换成本低但**掩盖渠道欠配**；仅还电成本清晰。  
- **第一性原理**：人天=渠道 B2B 权益，不是 C 端零售应急包。

## 反对理由

- 骑手当日可能无法继续跑单，需渠道侧及时续配/客服话术。

## 失效条件

- 产品明确恢复「渠道应急零售」且与人天池账本隔离计费。

## 交付

- 骑手端 C06 / 扫码；定价列表去「渠道兜底」列；`渠道骑手可换电校验` / 天数池 / AC / decision-049 修订
