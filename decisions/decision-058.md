# 决策卡片 058 · 套餐购买订单列表去掉期内使用 / 累计应分

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-22 |
| 阶段 | MVP / 列表收敛 |
| 结论 | **已锁定**：运营商「订单与服务 → 套餐购买订单」列表**删除**「期内使用」「累计应分」两列 |

## 决策问题

套餐列表是否需要展示期内换电次数与累计应分金额？

## 调用的大师视角

- **乔布斯**：列表只保留核单所需字段；用量与分账明细进详情/流水，避免双入口。
- **俞军**：列表上的应分数与「我的流水」重复，替换成本低但认知负担高。

## 已确认规则

| # | 规则 |
|---|------|
| 1 | 运营商套餐购买订单列表列：单号、用户、站点、套餐、押金、服务状态、有效期、分账、操作 |
| 2 | **不含**期内使用、累计应分 |
| 3 | 换电次数 / 应分明细仍可在订单详情或流水侧查看（若已有） |

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 列表更短、扫单更快；财务字段归流水 |
| 反对 | 运营习惯在列表一眼看换电次数 |
| 失效 | 若一期不做详情内用量汇总且运营强依赖列表扫数 |
