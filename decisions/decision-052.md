# 决策卡片 052 · 后台登录分流与短信改密

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-21 |
| 阶段 | MVP |
| 结论 | **已锁定**：B 端登录页区分**运营商 / 渠道商**；账号为手机号；允许用户通过**短信验证码**修改密码 |

## 结论

1. **登录分流**：登录页 Tab「运营商登录」「渠道商登录」；凭手机号 + 密码进入对应后台。  
2. **渠道商账号**：与「新增渠道商」一致，登录账号为手机号，默认密码 `123456`。  
3. **运营商账号**：与「新增运营商」一致，登录账号为手机号，默认密码 `123456`（见 decision-053）；平台/资金方等演示身份登录后可在侧栏切换。  
4. **修改密码**：登录页与登录后账号菜单均可进入；流程为 **手机号 → 获取验证码 → 新密码（至少 6 位）**；支持「返回登录」。演示验证码固定 `888888`。

## 支持理由

- **乔布斯**：运营商与渠道商门户分离，避免错登主体。  
- **俞军**：短信改密降低「默认密码忘记」替换成本，不依赖运营人工重置。

## 反对理由

- 短信成本与风控（频控、盗号）需研发落地；原型仅 Mock。

## 失效条件

- 统一 SSO / 企业微信登录替代手机号密码。

## 交付

- `prototype/index.html` 登录门禁 + 改密弹层；`app.js` / `app.css`；AC
