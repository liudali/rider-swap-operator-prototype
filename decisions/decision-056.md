# 决策卡片 056 · 运营商设备租赁并入融资（二期）

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-22 |
| 阶段 | MVP / 范围收敛 |
| 结论 | **已锁定**：运营商侧**移除「设备租赁」产品需求面**（我的设备不再打开租赁协议/还款计划抽屉）；原设备权属「租赁」改为展示 **「融资」**，业务并入 **融资管理（二期）** |

## 决策问题

运营商「我的设备」上的租赁协议（LC-*、还款计划、确认变更）是否保留？与融资管理如何分工？

## 调用的大师视角

- **乔布斯**：设备列表只保留运营所需权属标签；协议/还款不在设备页堆第二套产品。
- **俞军**：融资台账已是唯一还款协作入口，再开租赁协议抽屉 = 替换成本高、价值重复。

## 已确认规则

| # | 规则 |
|---|------|
| 1 | 运营商「我的设备」权属：自有 / **融资（二期）**；不再出现可点开的「租赁协议」 |
| 2 | 融资设备点击 → 融资管理（二期提示）；一期不交付 |
| 3 | 资方（设备租赁公司）侧协议/放款仍属融资协作链路，标二期（既有 decision-003） |
| 4 | **不波及**：渠道商结算模式「设备租赁 / 白名单」（CH-RENT）——仍为独立二期渠道能力 |

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 与 I6-5「移除直租、保留融资」一致；减少双轨口径 |
| 反对 | 历史 Mock 仍用 ownership=`租赁` 字段名，展示层映射为融资 |
| 失效 | 若商务坚持「经营性租赁」与「融资租赁」两套独立后台产品 |
