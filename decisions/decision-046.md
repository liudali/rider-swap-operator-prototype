# 决策卡片 046 · 押金设置补齐信用分门槛

| 项 | 内容 |

| 阶段 | MVP |
| 结论 | **已锁定**：运营商「押金设置」除押金数额外，须配置 **微信支付分免押门槛** 与 **芝麻信用免押门槛**；任一路 ≥ 门槛可免押，均低于门槛则按配置数额实缴 |

## 调用的大师视角

- **第一性原理（马斯克）**：免押判定本质是「分数 ≥ 阈值」，没有阈值就无法产品化「不足须缴」。
- **俞军（价值公式）**：可配置阈值降低运营用文案含糊沟通的成本，替换成本低于事后客诉。

## 支持理由

- 补齐「未达标」的可运营定义，与购套餐报价/骑手端展示一致。
- 支付分与芝麻分列，符合双通道免押现状。

## 反对理由

- 两套门槛增加配置项；城市差异化未做。
- 真实分值接口与风控策略可能由平台统一，运营商自定义幅度需合规确认。

## 失效条件

- 平台强制全国统一支付分门槛 → 改为平台统价只读下发。
- 仅保留微信支付分、废弃芝麻 → 隐藏芝麻字段。

## Non-goals

- 不接真实微信/芝麻查询接口；不按城市覆盖门槛；不在本页配置风控黑名单。
