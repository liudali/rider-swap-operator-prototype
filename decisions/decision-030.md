# 决策卡片 030 · 运营商往来并入平台服务 L2

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-16 |
| 阶段 | MVP / 信息架构 |
| 结论 | **已锁定**：取消独立一级「运营商往来账」，其四个二级页并入 **平台服务** |

## 支持理由

- **乔布斯减法**：保证金划扣、平台服务费、跨网往来同属「对平台清分账户」心智，再拆一个一级增加找页成本
- **俞军价值公式**：财务同学查「日清 / 保证金 / 服务费」时在同一 L1 内切换，替换成本低于记两个一级菜单位置

## 反对理由

- L2 增至 6 项，侧栏二级略长；若后续再扩「发票/对账」可能需分组或三级 `[假设]`

## 失效条件

- 往来账独立运营团队且与保证金/服务费权限完全隔离 → 可再拆一级

## 结构

```
平台服务（L1）
├── 服务保证金账户（L2，默认）
├── 平台服务费（L2）
├── 往来概览（L2）
├── 服务费明细（L2）
├── 日清账单（L2）
└── 周月汇总（L2）
```
