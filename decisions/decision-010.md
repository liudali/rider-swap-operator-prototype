# 决策卡片 010 · 设备批量导入 SN+运营商，废除待绑定

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-15 |
| 阶段 | MVP / 信息架构 + 录入简化 |
| 结论 | **已确认**：平台「批量导入」仅填 **SN,运营商ID**；设备类型与参数由系统按 SN 调 **IoT 平台**回填；导入即完成归属并进台账。**移除**设备管理「待绑定」及二次手工绑定。 |

## 为何改

- **乔布斯减法**：人填类型/规格易错且与 IoT 真相源重复；待绑定是中间态，可砍。
- **俞军**：录入从「多字段 + 再绑一次」→「两字段一步到位」，替换成本显著下降。

## 规则

1. 模板：每行 `SN,运营商ID`（如 `CAB-NEW-003,OP-SX`）。
2. 导入时按 SN 查 IoT → type/城市/规格；运营商须在营。
3. 成功后写入台账，站点默认「未分配站点」；失败行汇总提示（已入库 / IoT 无 / 运营商无效）。
4. 侧栏仅：**设备台账** / **批量导入**。

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 与 IoT 主数据一致；运营路径最短 |
| 反对 | IoT 延迟/缺失时导入失败率上升，需失败行可修 |
| 失效 | IoT 无法按 SN 返回类型参数，或监管要求强制人工复核再入账 |

## 假设 / 待确认
