# 决策卡片 064 · 骑士卡收款账户划为一期

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-23 |
| 阶段 | 发布前 |
| 结论 | **已锁定**：渠道商·骑士卡（卡差价）「**收款账户**」从二期改为一期；覆盖两种佣金结算：**即时到付**与**线下结算** |

## 调用的大师视角

- **俞军（价值公式）**：无收款账户则即时分账无法落地、线下结佣无打款对象，新体验成立成本为 0 → 必须一期。
- **贝佐斯（逆向工作法）**：3 年后两种结算并存是常态；账户能力前置避免二次改造进件。

## 范围

| # | 规则 |
|---|------|
| 1 | 骑士卡侧栏「收款账户」**一期**；去掉 `isPhase2View` 对该菜单的二期标 |
| 2 | **即时到付**：微信/支付宝子商户进件 + 对公绑定（既有） |
| 3 | **线下结算**：仍须维护对公结算账户，供运营商线下打佣；购卡款仍进运营商子商户 |
| 4 | 修订 decision-041 骑士卡一期清单：套餐与链接、购卡记录、佣金对账、**收款账户** |

## 支持理由

- 结算类型双轨决定账户是主路径，不是锦上添花。
- 与运营商「收款账户」心智一致。

## 反对理由

- 线下结算若仅靠合同填账户号、不做系统进件，一期账户页价值偏弱。

## 失效条件

- 商务取消线下结算、仅保留即时到付且进件改由运营商代配 → 可再评估渠道侧账户是否必做。

## Non-goals

- 不改人天池（无 C 端收款账户）；不改设备租赁/激活码二期边界。

## 下一步验证

- CH-CARD 登录：侧栏「收款账户」无二期徽章；即时/关闭即时均可进入并见账户表。
