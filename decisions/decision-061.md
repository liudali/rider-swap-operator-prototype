# 决策卡片 061 · 提现「融资待还」划二期

| 项 | 内容 |

| 阶段 | MVP |
| 结论 | **已锁定**：提现可提现余额**一期不扣**本月融资待还；融资待还预留与融资管理同属**二期** |

## 决策问题

运营商提现是否在一期就从可提现中预留「本月融资待还」？

## 调用的大师视角

- **乔布斯（减法）**：融资整链已是二期；提现再嵌融资预留等于把未交付能力塞进一期主路径，增加对账与审核心智。
- **贝佐斯（逆向工作法）**：三年后融资与还款日历成熟时，再让提现「必须预留待还」才闭环；一期先保证清分→审核→对公到账可用。

## 已确认规则

| # | 规则 |
|---|------|
| 1 | **一期**可提现 = 已清分 − 已提现 − 待审/处理中 |
| 2 | **二期**可提现 = 一期可提现 − 本月融资待还（还款日历当月未结清） |
| 3 | 提现申请/审核列表**一期不展示**「待还预留」列；新申请 `monthDueReserved=0` |
| 4 | 提现页可展示「本月融资待还」KPI，须标**二期**，并注明「一期不扣可提现」 |
| 5 | 与 `decision-003` / `decision-056` 一致：融资管理整链二期 |

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 一期提现可独立验收；避免融资未上线时扣减口径悬空 |
| 反对 | 有融资欠款的运营商一期可能提空，二期再上预留有短期资金风险 |
| 失效 | 若融资与还款日历提前并入一期交付 |
