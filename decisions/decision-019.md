| # | 问题 | 结论 |
|---|------|------|
| 1 | 便宜买完后能否全市换电 | **A** 能（接受套利，视为分区/偏远补贴） |
| 2 | 配置粒度 | **B** 价格分区；站点挂到区 |
| 3 | 特价范围 | **A′** 任意 SKU；**价格不限**（可高于/低于城市底价） |

## 调用的大师视角

- **俞军**：分区价降低偏远站购入门槛；权益仍全市可用，替换成本低。
- **乔布斯**：不维护全站价表；城市底价 + 少数分区即可。
- **贝佐斯**：站点数增长时「站→区」可水平扩展，避免逐站价爆炸。

## 核心模型

```
售价 = 站点所属分区的区价(运营商×分区×SKU) ?? 城市底价(运营商×城市×SKU)
购价上下文：扫柜机 / 选站 → 站点 → 分区
履约：同运营商任意站换电（不因购价分区限制）
已购：price_snapshot；改价不追溯
```

| 实体 | 说明 |
|------|------|
| 城市底价 | 现「个人套餐价」主表 |
| 价格分区 | 运营商×城市下命名分区（如「偏远站区」） |
| 站点挂区 | 一站至多属一个启用分区；未挂区用城市底价 |
| 区价行 | 分区×SKU 的零售价（可高于/低于底价）；未配行继承底价 |

## Non-goals

- 按单站独立完整价表（无分区）
- 购分区价后限制换电站点
- 渠道分销价 / 人天批发价跟分区走

## 失效条件

- 多数站点都要独立价且无法归区 → 再评估逐站覆盖
- 合规要求禁止跨区套利 → 需改换电范围（与现行策略冲突）
