## 决策问题

人天池「换电或持电池」均确认消耗 1 人天/日，平台提成流水如何区分来源？

## 调用的大师视角

- **马斯克（第一性原理）**：扣人天的事实只有两类触发——发生换电 / 仅持电池；账本场景应与事实同构。
- **俞军**：运营对账时「有单可点」与「无单但合法计提」体验差大，合并成笼统「确认消耗」抬高核对成本。

## 已确认规则

| # | 规则 |
|---|------|
| 1 | 场景枚举：**确认消耗-换电** / **确认消耗-持有电池**（兼容筛选「确认消耗」= 全部子类型） |
| 2 | **确认消耗-换电**：关联单 = 换电订单号（SW-*） |
| 3 | **确认消耗-持有电池**：关联单展示 **—**（当日无换电、因持电池确认） |
| 4 | 计提基数/费率不变：平台标准人天价 × 运营商 B 端费率；主体仍为额度售卖方 U |
| 5 | 落点：平台「流水管理 → 平台提成」；运营商「平台服务费 → B 端」同步展示 |

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 与天数池预占确认规则一致；对账可区分有单/无单 |
| 反对 | 筛选项变多；历史数据需映射旧「确认消耗」 |
| 失效 | 若改为「仅首次换电确认」而不再对持电池确认 |
