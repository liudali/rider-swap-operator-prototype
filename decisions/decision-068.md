## 决策问题

套餐购买订单出现「无需押金」，与用户台账四值口径不一致，客服/对账易误判为独立业务类型。

## 调用的大师视角

- **乔布斯**：砍掉「无需」标签；本单不收押 = 无记录 = ——。
- **俞军**：统一枚举降低跨页替换成本（用户列表 / 用户押金 / 套餐单 / 平台用户信息）。

## 已确认规则

| 取值 | 含义 |
|------|------|
| **实付** | 用户实缴电池押金 |
| **信用免押** | 支付分 / 芝麻达标免实付 |
| **渠道担保** | B 端渠道担保（如租赁白名单等） |
| **——** | 无押金记录、本单应缴为 0、已退清等 |

> **映射**：原「无需押金」（`batteryDeposit === 0`）→ **——**；收款状态仍为 ——。

## 支持 / 反对 / 失效

- **支持**：与 decision-066 用户信息口径一致。
- **反对**：详情里少一句「本单不收」说明；可用备注/空态文案补。
- **失效**：产品单独定义「零押金 SKU」为第五种方式。
