# 决策卡片 006 · 押金退还纳入退款管理（自动/手动独立配置）

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-15 |
| 阶段 | MVP / 发布前补漏 |
| 结论 | **已确认**：骑手「仅退电池押金」统一进运营商「退款管理」；运营商自行设置 **自动退款 / 手动确认**；与套餐退款模式 **独立配置** |

## 已确认规则

1. **入口统一**：押金退还申请与冷静期/中途完结等同属「退款管理」队列（类型=`押金退还`）。  
2. **模式自设**：运营商在「退款设置」配置 `depositRefundMode`=`auto|manual`，与套餐 `mode` 互不影响。  
3. **资金路径**：退押 = 运营商子商户原路退实收押金；**不影响**套餐有效期；押金本就不切分，无平台分润冲正。  
4. **主单押金子项**：冷静期/中途完结中的押金随主单审核，不走独立押金模式开关。

## 决策依据（交叉验证）

- **乔布斯减法**：不单独开「押金管理」菜单，复用退款管理减少认知与工单分裂。  
- **俞军价值公式**：运营商可按风险偏好选自动/手动（替换成本≈零，价值=控损+效率可选）。

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 与套餐退款运营动线一致；手动可控电池损耗争议；自动降低低风险运营商积压 |
| 反对 | 两套开关增加配置理解成本；误配自动可能导致有争议仓促退押 |
| 失效 | 监管要求押金托管账户、或强制人工复核所有退押 |
