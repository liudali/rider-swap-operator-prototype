## 决策问题

运营商自有电池的「位置」如何向运营人员表达？是否需要按位置筛选与分页？

## 调用的大师视角

- **俞军**：相对「本电池运营主体」的位置分类，降低跨柜排查成本；筛选降低替换成本。
- **乔布斯**：位置只保留四档枚举，柜号/用户细节作副文案，不堆第五种模糊状态。

## 已确认规则

| # | 规则 |
|---|------|
| 1 | 位置枚举（相对本电池 `deviceOwnerId`）：**自有电柜** / **其他运营商电柜** / **柜外** / **柜外-用户** |
| 2 | `inCab` 为 `CAB-*`：柜主=电池主 → 自有电柜；否则 → 其他运营商电柜 |
| 3 | 用户持有（`柜外-用户` / `heldByUser`）→ 柜外-用户；其余非柜内（柜外充电、待入柜、资方库存等）→ **柜外** |
| 4 | 列表展示：主文案=四档之一，副文案=柜 SN / 用户 / 柜外细节 |
| 5 | 顶栏筛选项「位置」；默认每页 5 条，筛选/切 Tab 重置页码 |

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 与跨网电池场景一致；运维可快速找「在别人柜里 / 在用户手里」的电池 |
| 反对 | 「资方库存」并入柜外，融资侧若要独立台账需另开菜单 |
| 失效 | 若 IoT 只能报柜 SN、无法区分用户持有与柜外充电 |

## 文档 / 原型

- `batteryLocationKind` / `batteryLocationCell`；`devices_battery.location`；`deviceBatteryPage`
- AC · 角色清单 · 产品变更记录
