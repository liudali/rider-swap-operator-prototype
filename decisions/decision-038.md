# 决策卡片 038 · 平台管理员复用员工模块并作为一期例外

| 项 | 内容 |
|----|------|
| 日期 | 2026-07-17 |
| 阶段 | MVP |
| 结论 | **已锁定**：平台侧复用 `employees` 视图显示「管理员」，纳入一期；其他主体员工仍为二期 |

## 调用的大师视角

- **乔布斯（减法）**：复用已有人员列表、表单和权限机制，不建设第二套管理组件。
- **俞军（价值公式）**：平台内部权限配置是一期治理刚需；按角色改变名称和范围的替换成本低于新增独立系统。

## 支持理由

- 复用员工模块可快速形成管理员人员、模板和菜单权限闭环。
- 平台“管理员”和经营主体“员工”在入口、登录分组、二期标记上清晰区分。
- 保留未来拆分空间，不影响现有员工二期演示。

## 反对理由

- 同一 `employees` 视图按角色改变名称和交付范围，存在串标风险。
- 平台权限 ID 与经营主体权限 ID 并存，需要持续回归验证。

## 失效条件

- 平台管理员需要独立账号域、审批流、权限版本或数据行级权限时，应拆分为独立模块。

## Non-goals

- 不接真实密码、邀请、短信和找回密码。
- 不做数据行级权限。
- 不做权限审批、版本历史或登录审计详情。

## 下一步验证

- 平台侧栏「管理员」无二期标记，其他主体「员工」仍为二期。
- 平台管理员登录无二期前缀，只能看到授权菜单。
- 超级管理员受保护，普通管理员可配置模板和菜单权限。
