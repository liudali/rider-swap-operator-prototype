## 决策问题

渠道商骑士卡 **即时到付 ↔ 线下结算** 切换时，如何处理新单、旧单与对账展示？

## 调用的大师视角

- **贝佐斯（逆向工作法）**：账本不可改写；审计按订单还原当时结算方式。
- **俞军（价值公式）**：「新单跟新规则」替换成本最低。
- **乔布斯（减法）**：不做整月重算 / 追回已分账；一期无审批流。

## 已确认规则

### 1. 生效口径

| 项 | 规则 |
|----|------|
| 谁改 | 运营商「渠道管理 → 签约渠道」改即时开关（及比例） |
| 何时生效 | **次日 00:00**（业务日）；变更保存时**必须提示**生效日与「历史不回溯」 |
| 当日 | 变更当天新成交仍按**变更前**结算方式 |
| 历史订单 | 结算类型、佣金额、分账结果**冻结** |
| 留痕 | 变更记录：谁、何时、旧→新、生效日 |

### 2. 即时 → 线下

- 允许关闭；确认文案含：次日生效、已分账不追回。
- 生效后新单：线下待结；已即时分账订单不变。
- 子商户可保留；对公继续用于线下打佣（decision-064）。

### 3. 线下 → 即时

- 闸门：微信/支付宝进件已开通 + 对公已绑定 + 已配 `commission_rate`。
- 生效后新单即时分账；历史待线下仍线下结清。

### 4. 对账展示

- 订单级带结算方式；同月可混态。
- **月度汇总按结算方式拆成行**（同月可两行：即时分账 / 线下结算）。

### 5. 退款

- 按原订单结算方式冲正。

## 支持理由 / 反对理由 / 失效条件

- 支持：次日生效给运营缓冲；拆行对账清晰。
- 反对：同月两行略繁；渠道或误解「关即时为何不追回」。
- 失效：监管要求整月重算 → 另立。

## Non-goals

- 无审批流；不批量补分账/追回；不关即时就注销子商户。
