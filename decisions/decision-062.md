## 决策问题

渠道商人天池是否需要「额度使用规则」为团队设置周期额度上限？

## 调用的大师视角

- **乔布斯（减法）**：团队已有「绑定消耗池 + 骑手个人分配」两层控制；再加团队周期上限抬高配置成本，且与池余额/个人剩余三者叠加易冲突。
- **俞军**：新体验（精细控团队）相对旧体验（池+个人）增量小，替换成本（理解规则命中、超额预占失败）偏高，净值不足。

## 已确认规则

| # | 规则 |
|---|------|
| 1 | 侧栏/顶层 Tab **删除**「额度使用规则」；旧 `dayPoolTab=rules` 进入「额度池」 |
| 2 | **无**团队周期额度上限；额度约束仅：**池可用余额** + **骑手个人已分配剩余** |
| 3 | 团队仍为编排层（登记、绑定消耗池、消耗汇总）；池级扣天/激活口径仍见「额度池」详情 |
| 4 | 错误码 `E003`（无匹配额度使用规则）**废止** |
| 5 | 顶层 Tab 计数按 6（含二期零售价/异常）更新文档 |

## 支持 / 反对 / 失效

| | 内容 |
|--|------|
| 支持 | 配置面更简；预占失败原因只剩池空/个人无额度，对账清晰 |
| 反对 | 无法防止单团队透支整池；大渠道多车队场景可能需人工约定 |
| 失效 | 若后续明确要求「车队预算包」强管控且不可用个人分配替代 |
